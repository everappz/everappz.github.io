이 화면에서는 음악이 저장된 모든 소스를 연결할 수 있습니다. Dropbox, Google Drive, iCloud Drive, OneDrive, MEGA, Box, pCloud, Yandex Disk, Synology Drive 등 인기 있는 클라우드 서비스와 표준 프로토콜을 통해 Mac, PC 또는 NAS를 통합할 수 있습니다. 컬렉션이 Dropbox와 같은 스트리밍 친화적인 서비스에 있든, Synology, QNAP, Buffalo, Apple Time Capsule, WD My Cloud Home과 같은 개인 NAS에 있든, Flacbox는 단일 화면에서 모두 연결합니다.

## 클라우드 스토리지 연결

- **연결하기** 탭을 엽니다.
- 메뉴에서 **클라우드 스토리지 연결**을 선택합니다.
- 목록에서 클라우드 스토리지 서비스를 선택합니다.
- 클라우드 제공업체가 제공하는 공식 인증 페이지에서 자격 증명을 입력한 다음 **완료됨**을 탭합니다.

문제가 발생하면 인터넷 연결과 로그인 / 비밀번호를 확인하세요. 앱의 프리미엄 버전에서는 무제한 서비스를 추가할 수 있으며, 무료 버전은 최대 3개까지 지원합니다.

## 지원되는 클라우드 스토리지 서비스, 미디어 서버 및 프로토콜

Flacbox는 매우 다양한 음악 소스를 지원합니다. 아래의 모든 것은 단일 **클라우드 스토리지 연결** 화면에서 작동합니다.

**개인 클라우드 스토리지:** iCloud Drive · Dropbox · OneDrive · Google Drive · MEGA · Box · pCloud · Yandex Disk · WD My Cloud Home · MediaFire · TeraCLOUD (InfiniCLOUD) · HiDrive · IceDrive · Koofr · OpenDrive · MyDrive · Put.io · Cloud Mail.ru · Internxt · Proton Drive · AliDrive (阿里云盘) · Baidu Pan (百度网盘).

**자체 호스팅 서버:** Plex · Jellyfin · Emby · Subsonic (및 모든 Subsonic 호환 서버 — Airsonic, Funkwhale, Gonic, Logitech Media Server, Ampache) · Navidrome · Nextcloud (및 공유 API를 통한 ownCloud) · Synology Drive · QNAP.

**NAS 및 파일 공유 프로토콜:** SMB (SMB1, SMB2, Auto) · WebDAV (HTTP / HTTPS) · FTP · FTPS · SFTP (SSH 파일 전송 프로토콜, 비밀번호 또는 공개키 인증) · NFS · DLNA / UPnP (재생 및 다운로드).

**S3 호환 오브젝트 스토리지:** AWS S3, Backblaze B2, Wasabi, Cloudflare R2, MinIO, DigitalOcean Spaces 및 기타 S3 API 엔드포인트.

**로컬 네트워크 검색:** 사용 가능한 기기 섹션이 Wi-Fi 네트워크의 모든 Bonjour / mDNS 서비스를 자동으로 나열하므로 IP 주소를 입력하지 않고도 탭하여 연결할 수 있습니다.

각 연결은 지원되는 경우 OAuth 기반 인증으로 서비스의 **공식 SDK 또는 개방형 프로토콜**을 사용합니다. 동일한 서비스의 여러 계정을 연결할 수 있으며(예: Google Drive 계정 두 개, 개인 Dropbox와 업무용 Dropbox, 또는 Plex와 Jellyfin 서버 모두) 연결하기 화면에서 나란히 탐색할 수 있습니다.

## 보안 및 개인 정보

연결된 클라우드 서비스와 상호 작용할 때 공식 SDK와 보안 연결만 사용합니다. 로그인과 비밀번호는 애플리케이션에서 액세스할 수 없으며 모든 전송은 TLS 암호화됩니다.

로그인과 비밀번호를 입력하면 애플리케이션은 클라우드 서비스 제공업체가 제공하는 공식 인증 페이지를 표시하며, 전체 인증 프로세스는 애플리케이션 외부에서 이루어집니다. 클라우드 서비스 제공업체는 성공적인 인증 후 애플리케이션에 인증 토큰을 보내고, 해당 토큰은 API 호출에 사용됩니다.

인증 토큰은 제3자 애플리케이션이 사용자를 대신하여 클라우드 스토리지와 상호 작용할 수 있도록 하는 디지털 키입니다. 토큰은 Apple의 보안 시스템 스토리지(Keychain)에 기기에 저장되며, 이는 저장 시 암호화되고 기기 비밀번호 또는 생체 잠금으로 보호됩니다. 연결된 클라우드 서비스에서 파일을 기기로 다운로드할 수 있으며, 해당 파일은 앱의 Documents 디렉토리에 저장되고 내장 파일 관리자를 사용하여 언제든지 제거할 수 있습니다.

애플리케이션은 연결된 클라우드 계정의 정보를 Everappz, 광고주 또는 제3자와 공유하지 않습니다. 웹 브라우저에서 계정 설정 페이지를 열어 언제든지 클라우드 계정에 대한 액세스를 취소할 수 있습니다.

## 인증 토큰 거부

인증 토큰을 취소하려면 웹 브라우저에서 클라우드 계정에 로그인하고 보안 또는 연결된 앱 페이지로 이동하세요. 거기서 클라우드 계정에 연결된 모든 제3자 앱을 찾아 더 이상 사용하고 싶지 않은 앱을 제거할 수 있습니다. Google 계정에 대한 자세한 지침은 [여기](/docs/howto/how-to-disconnect-third-party-app-from-your-google-account)에서 확인할 수 있습니다.

애플리케이션 내에서도 클라우드 계정의 연결을 끊을 수 있으며, 그렇게 하면 인증 토큰이 기기에서 즉시 삭제됩니다. 기기에서 애플리케이션을 제거하면 다운로드된 모든 데이터와 액세스 토큰이 자동으로 함께 제거됩니다.

## 클라우드 스토리지 연결 끊기 또는 설정 변경

- **클라우드 스토리지 옵션 액세스** — **연결하기** 화면에서 연결된 서비스를 찾습니다.
- **서비스 제목 옆의 "..." 버튼을 탭**하여 추가 옵션을 엽니다:
  - **이름 바꾸기** — 목록에 표시되는 클라우드 서비스 이름을 변경합니다.
  - **설정** — 설정 또는 인증 데이터를 수정합니다. 인증 토큰이 만료된 경우 연결된 클라우드 서비스를 다시 인증해야 할 수 있습니다.
  - **연결 끊기** — 앱과 클라우드 서비스 사이의 연결을 완전히 끊습니다. 해당 서비스와 관련된 모든 곡이 앱의 음악 라이브러리에서 제거되지만, 서버에서는 그대로 유지됩니다.

## 컴퓨터 또는 NAS 연결

SMB, DLNA 또는 WebDAV 프로토콜을 사용하여 컴퓨터, 개인 NAS 또는 기타 네트워크 기기를 연결할 수도 있습니다. Mac, Windows PC, Synology 박스 또는 NAS에 있는 기존 홈 음악 라이브러리를 아무것도 복사하지 않고 Flacbox로 가져오는 가장 쉬운 방법입니다.

## SMB를 사용하여 컴퓨터 연결

- **클라우드 스토리지 연결 → SMB**를 탭합니다.
- URL 필드에 `smb://computer-ip-address/shared-folder-name` 형식으로 컴퓨터의 IP 주소와 공유 폴더 이름을 입력합니다.
- 프로토콜 버전 선택: **Auto**, **SMB1** 또는 **SMB2**.
- 로그인과 비밀번호를 입력합니다(필요한 경우).
- **완료됨**을 탭합니다.

연결에 성공하면 **클라우드 스토리지** 섹션에서 연결된 스토리지를 볼 수 있습니다. SMB를 사용하여 Mac 또는 PC를 연결하는 방법에 대한 전체 튜토리얼은 [여기](/docs/howto/stream-your-music-from-mac-or-pc-to-iphone-using-smb/)에서 확인할 수 있습니다.

## WebDAV를 사용하여 NAS 연결

모든 단계는 SMB와 동일하며 URL 필드만 다릅니다. URL은 서버가 SSL을 지원하는 경우 `http://server-name` 또는 `https://server-name` 형식이어야 합니다. WebDAV는 **Synology, QNAP, Nextcloud, ownCloud** 및 WebDAV 엔드포인트가 있는 다른 서버와 함께 작동합니다.

WebDAV를 사용하여 NAS를 연결하는 방법에 대한 전체 튜토리얼은 [여기](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac)에서 확인할 수 있습니다.

## DLNA를 사용하여 컴퓨터 또는 NAS 연결

[여기](/docs/howto/how-to-enable-dlna-media-server-on-windows-10-and-play-your-music-on-iphone)에서 설명한 대로 DLNA / UPnP 프로토콜을 사용하여 Windows PC 또는 개인 NAS에 있는 음악 라이브러리를 공유하고 앱에서 해당 라이브러리에 액세스할 수도 있습니다. DLNA는 인기 있고 널리 지원되는 프로토콜이지만 음악을 재생하거나 다운로드할 수만 있습니다 — DLNA 서버에 파일을 업로드하거나 새 폴더를 만들 수 없습니다.

## FTP, FTPS 또는 SFTP를 사용하여 NAS 또는 서버 연결

Flacbox는 클래식 파일 전송 프로토콜도 지원합니다. FTP 또는 FTPS를 통해 서버를 연결하려면 **클라우드 스토리지 연결 → FTP**를 탭하고, `ftp://server-name` 형식(암호화 연결의 경우 `ftps://server-name`)으로 호스트 URL을 입력하고, 로그인과 비밀번호를 제공한 다음 **완료됨**을 탭합니다. **SFTP**(SSH 파일 전송 프로토콜)의 경우 대신 **SFTP**를 선택하고 비밀번호 또는 SSH 키 쌍을 제공합니다. 둘 다 NAS 기기, Linux 호스트 및 FTP / FTPS / SSH 데몬이 있는 모든 서버와 함께 작동합니다.

## NFS 공유 연결

Flacbox에는 **NFS**(네트워크 파일 시스템) 지원이 포함되어 있어 SMB 대신 NFS를 통해 음악 라이브러리를 노출하는 Linux 호스트, BSD 서버 및 NAS 기기에 편리합니다. **클라우드 스토리지 연결** 메뉴에서 **NFS**를 선택하고, 서버 주소와 내보낸 경로를 입력한 다음 **완료됨**을 탭합니다.

## Plex 미디어 서버 연결

Flacbox는 Plex 미디어 서버에 직접 연결하여 아티스트, 앨범, 장르, 재생 목록별로 음악 라이브러리를 탐색할 수 있습니다. **클라우드 스토리지 연결 → Plex**를 탭하고, Plex 계정으로 로그인하고, 서버를 선택하면 라이브러리가 클라우드 서비스와 함께 나타납니다. 동일한 로컬 네트워크의 Plex 서버는 **사용 가능한 기기** 섹션에서 자동으로 검색됩니다.

## Jellyfin 또는 Emby 서버 연결

**Jellyfin**(오픈 소스)과 **Emby**(상업용) 미디어 서버 모두 Flacbox에서 기본적으로 작동합니다. **클라우드 스토리지 연결 → Jellyfin** 또는 **Emby**를 탭하고, 서버 URL(예: `http://server-ip:8096`)과 자격 증명을 입력하면 음악 라이브러리를 스트리밍할 준비가 됩니다. Plex와 마찬가지로 로컬 네트워크의 라이브러리는 **사용 가능한 기기**에 자동으로 나열됩니다.

## Subsonic 또는 Navidrome 서버 연결

Flacbox는 Subsonic API를 지원하므로 **Subsonic** 자체, **Navidrome** 및 Airsonic, Funkwhale, Gonic, Logitech Media Server (LMS), Ampache를 포함한 모든 Subsonic 호환 서버와 함께 작동합니다. **클라우드 스토리지 연결 → Subsonic**을 탭하고, 서버 URL과 자격 증명을 입력하면 연결하기 화면에 라이브러리가 나타납니다.

## S3 호환 오브젝트 스토리지 연결

클라우드 오브젝트 스토리지를 사용하는 자체 호스터 및 오디오파일을 위해 Flacbox에는 S3 호환 커넥터가 포함되어 있습니다. **클라우드 스토리지 연결 → S3 스토리지**를 탭하고, 엔드포인트 URL, 리전, 액세스 키, 비밀 키, 버킷 이름을 입력합니다.

## 사용 가능한 기기

이 섹션에는 Bonjour 검색을 통해 Flacbox에서 연결할 수 있는 로컬 네트워크의 모든 기기가 표시됩니다. 연결을 설정하려면 다음 단계를 따르세요:

- 앱을 열고 연결하기 아래의 **사용 가능한 기기** 섹션으로 이동합니다.
- 연결하려는 기기를 탭합니다.
- 필요한 경우 로그인 세부 정보를 입력하여 연결을 완료합니다.

이것은 IP 주소를 수동으로 입력하지 않고 홈 네트워크에서 SMB, WebDAV, DLNA 공유를 검색하는 가장 빠른 방법입니다.

## Wi-Fi Drive

Wi-Fi Drive는 모든 데스크톱 브라우저를 통해 컴퓨터에서 iOS 기기로 무선 파일 전송을 가능하게 하는 편리한 기술입니다. 이 기능을 효과적으로 사용하려면 기기와 컴퓨터가 동일한 Wi-Fi 네트워크에 연결되어 있는지 확인하세요.

### Wi-Fi Drive 활성화

- 애플리케이션을 열고 **연결하기** 탭으로 이동합니다.
- **Wi-Fi를 통해 연결**을 선택하여 Wi-Fi Drive 메인 화면에 액세스합니다.
- (선택 사항) 액세스를 보호하기 위해 임베디드 웹 서버의 사용자 이름과 비밀번호를 설정합니다.
- **Wi-Fi Drive 시작**을 탭하여 Wi-Fi Drive를 활성화합니다.

### 컴퓨터에서 Wi-Fi Drive 액세스

- 컴퓨터(데스크톱 또는 노트북)에서 웹 브라우저(Chrome, Firefox 또는 Safari 등)를 엽니다.
- 브라우저의 주소 표시줄에 애플리케이션에서 제공한 URL을 입력합니다.

### 무선으로 파일 전송

브라우저에서 iOS 기기에 해당하는 웹 페이지가 열리면 컴퓨터에서 웹 페이지로 파일을 쉽게 드래그 앤 드롭할 수 있습니다. 드롭한 파일이 iOS 기기로 전송되기 시작하며 Flacbox 내에서 액세스할 수 있게 됩니다.

macOS의 Finder(이동 → 서버에 연결…) 또는 Windows의 파일 탐색기(네트워크 드라이브 연결…)에서 Wi-Fi Drive를 직접 마운트하여 iPhone 또는 iPad를 일반 네트워크 드라이브로 사용할 수도 있습니다.

Wi-Fi Drive를 사용하여 무선으로 파일을 전송하는 방법에 대한 자세한 지침은 [여기](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive)에서 확인할 수 있습니다.

## iTunes / Finder 파일 공유

iTunes 파일 공유(macOS Catalina 이상에서는 Finder 파일 공유)는 Lightning 또는 USB-C 케이블을 사용하여 컴퓨터에서 기기로 파일을 전송하는 또 다른 방법입니다.

- 케이블을 사용하여 기기를 컴퓨터에 연결하고 Mac에서 **Finder**(또는 Windows에서 **iTunes**)를 실행합니다.
- **위치 → 연결된 기기 → 파일**을 열고 Flacbox 앱을 찾습니다.
- 앱 아이콘을 탭하여 공유된 모든 폴더를 확인합니다.
- 드래그 앤 드롭을 사용하여 컴퓨터에서 기기의 공유 폴더로 파일을 복사합니다.

Finder 파일 공유 사용 방법에 대한 자세한 지침은 [여기](/docs/howto/how-to-play-local-itunes-files-on-my-iphone/)에서 확인할 수 있습니다.

## USB 플래시 드라이브 연결

SD 카드 또는 USB 드라이브가 있는 경우 Lightning-USB / SD 카드 리더기 또는 USB-C 드라이브(iPad 및 iPhone 15 / 16 / 17 / Pro)를 사용하여 연결할 수 있습니다. 앱은 Apple 인증 카드 리더기와 iXpand 플래시 드라이브를 지원합니다. iXpand 플래시 드라이브의 경우 Lightning 포트에 삽입하고 애플리케이션을 열면 외부 기기 연결됨 메시지와 기기 정보가 표시됩니다. 플래시 드라이브 아이콘을 탭하여 음악 폴더에 액세스하고 오디오 파일을 탭하여 재생을 시작합니다.

iPhone에 USB 플래시 드라이브를 연결하고 그 안에 있는 음악을 듣거나 파일을 관리하는 방법에 대한 자세한 지침은 [여기](/docs/howto/how-to-connect-a-usb-flashcard-to-the-iphone-and-listen-to-music-or-manage-files-located-on-it)에서 확인할 수 있습니다.

## 파일 관리자

클라우드 스토리지 서비스를 연결한 후 해당 아이콘을 탭하면 사용 가능한 모든 파일과 폴더를 볼 수 있습니다. 내장 파일 관리자를 사용하여 이러한 파일을 다운로드, 이름 바꾸기, 이동, 업로드, 삭제 등으로 작업할 수 있습니다. 다운로드를 시작하면 파일이 전송 큐에 나타납니다. 전송 큐를 열려면 로컬 파일 탭으로 이동하여 왼쪽 상단 모서리의 회전 화살표 아이콘을 탭합니다. 다운로드된 모든 파일과 폴더는 로컬 파일 섹션에서 사용할 수 있습니다.

## 상단 도구 모음

탐색 바 아래에 편리하게 위치한 상단 도구 모음은 쉽게 액세스할 수 있는 몇 가지 유용한 작업을 제공합니다. 아래로 스와이프하여 표시하거나 숨길 수 있습니다.

- **검색** — 현재 디렉토리 내에서 빠른 검색을 수행하여 특정 파일이나 폴더를 쉽게 찾습니다.
- **계속 재생** — 애플리케이션 설정에서 활성화된 경우 현재 폴더의 오디오 플레이어 큐와 마지막 미디어 위치를 복원합니다.
- **모두 재생** — 현재 폴더와 하위 폴더를 스캔한 다음 찾은 모든 오디오 파일을 새 플레이어 큐에 추가합니다.
- **모두 셔플** — 모두 재생과 비슷하지만 파일을 오디오 플레이어 큐에 추가하기 전에 섞습니다.

## 폴더 옵션

폴더를 열면 오른쪽 상단 모서리의 **"..."** 버튼을 탭하여 사용 가능한 편리한 작업 세트를 찾을 수 있습니다.

- **선택하다** — 파일 선택 모드를 활성화합니다. 폴더 내에서 하나 이상의 파일을 선택하고 전체 선택에 대한 작업을 수행할 수 있습니다.
- **새 폴더** — 현재 폴더에 새 폴더를 만듭니다.
- **오프라인 모드 활성화** — 현재 폴더에 대한 오프라인 모드를 켭니다. 오프라인 모드는 단순한 다운로드를 넘어 폴더의 변경 사항을 적극적으로 모니터링합니다.
- **파일 업로드** — 기기에서 온라인 폴더로 파일을 업로드합니다.
- **검색** — 현재 폴더 내에서 특정 파일을 검색합니다.
- **정렬** — 이름, 크기, 수정 날짜 또는 메타데이터별로 파일을 정렬합니다.
- **그리드 / 목록 보기** — 테이블 보기와 썸네일 보기 사이를 전환합니다.

## 온라인 파일 편집

클라우드 스토리지의 여러 파일을 관리해야 하는 경우 **선택 모드**를 사용하여 작업을 간소화하세요:

- **선택 모드 활성화** — 오른쪽 상단 모서리의 **"..."** 버튼을 탭하고 **선택하다**를 선택합니다.
- **파일 선택** — 모든 파일과 폴더 옆에 체크박스가 나타납니다. 하나 또는 여러 항목을 선택합니다.
- **작업 수행** — 파일이나 폴더를 선택하면 다음 재생, 나중에 재생, 음악 라이브러리에 추가, 재생 목록에 추가, 복사, 업로드, 이동, 이름 바꾸기, 삭제에 액세스할 수 있습니다.

연결된 클라우드 스토리지를 읽기 전용으로 처리하려면(실수로 인한 삭제 방지) 설정 → 파일 관리자 → 온라인 파일 편집 → 끄기를 활성화하여 UI에서 모든 파괴적인 작업을 숨기세요.

## 파일 작업

파일 제목 근처의 **"..."** 아이콘을 탭하여 작업 메뉴를 표시합니다:

- **다음 재생** — 현재 트랙 바로 다음에 재생되도록 플레이어 큐의 상단에 파일을 삽입합니다.
- **나중에 재생** — 플레이어 큐의 하단에 파일을 추가합니다.
- **음악 라이브러리에 추가** — 아티스트, 앨범, 장르 또는 작곡가별로 정리된 음악 라이브러리에 파일을 통합합니다.
- **재생 목록에 추가** — 기존 재생 목록에 파일을 추가하거나 새 재생 목록을 만듭니다.
- **오디오 태그 편집** — 내장 태그 편집기를 열어 메타데이터를 수정합니다. 온라인 파일의 경우 트랙이 임시로 다운로드되고, 편집된 후 확인하면 다시 업로드됩니다.
- **즐겨찾기에 추가** — 빠른 액세스를 위해 파일을 즐겨찾기 목록에 추가합니다.
- **다운로드** — 오프라인 사용을 위해 파일이나 폴더를 기기로 다운로드합니다.
- **이름 바꾸기** — 원격 스토리지에서 직접 파일 이름을 바꿉니다.
- **이동** — 클라우드 스토리지 내의 다른 폴더로 파일을 이동합니다.
- **다른 앱으로 열기** — 파일을 다른 호환 앱으로 내보냅니다. 파일이 기기로 다운로드된 다음 시스템 공유 시트가 나타납니다.
- **삭제하기** — 클라우드 스토리지에서 파일을 영구적으로 제거합니다. **이 작업은 취소할 수 없습니다.**

## 폴더 작업

클라우드 스토리지의 각 폴더에 대해 폴더 제목 옆의 **"..."** 아이콘을 탭하여 다양한 작업을 사용할 수 있습니다.

- **모두 재생** — 선택한 폴더의 모든 항목으로 현재 플레이어 큐를 교체합니다.
- **다음 재생** — 전체 폴더를 플레이어 큐의 상단에 추가합니다.
- **나중에 재생** — 폴더 내용을 플레이어 큐의 하단에 추가합니다.
- **음악 라이브러리에 추가** — 폴더 내용을 음악 라이브러리에 통합합니다.
- **재생 목록에 추가** — 전체 폴더를 재생 목록에 추가합니다. 새 재생 목록 이름은 폴더 이름에서 자동으로 채워집니다.
- **즐겨찾기에 추가** — 빠른 액세스를 위해 폴더를 즐겨찾기에 추가합니다.
- **오프라인 모드 활성화** — 단순한 다운로드를 넘어 새 파일이 온라인에 나타날 때 자동으로 다운로드합니다.
- **다운로드** — 오프라인 액세스를 위해 폴더의 모든 내용을 기기로 다운로드합니다.
- **이름 바꾸기** — 원격 스토리지에서 직접 폴더 이름을 바꿉니다.
- **이동** — 클라우드 스토리지 내의 다른 위치로 폴더를 이동합니다.
- **아카이브 (ZIP)** — 폴더 내용을 단일 ZIP 파일로 묶습니다(프리미엄 기능).
- **삭제하기** — 클라우드 스토리지에서 폴더와 내용을 영구적으로 제거합니다. **이 작업은 취소할 수 없습니다.**

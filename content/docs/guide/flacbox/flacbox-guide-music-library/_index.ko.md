Flacbox로 음악 라이브러리를 관리하는 것은 간편합니다. 로컬 FLAC, ALAC, DSD, MP3, M4A, OGG, WMA, APE 및 수십 가지 다른 형식의 모든 트랙을 단일하고 검색 가능한 컬렉션으로 쉽게 구성할 수 있습니다. 음악 라이브러리를 구성하는 방법에는 두 가지가 있습니다: 수동 추가(추가될 항목을 직접 선택) 또는 자동 동기화(Flacbox가 지정된 클라우드 폴더를 스캔하여 새 파일이 나타나면 자동으로 추가).

## 수동 추가

트랙을 수동으로 추가하려면 왼쪽 상단 모서리에 있는 **음악 추가** 아이콘을 탭하고 연결된 클라우드 스토리지 서비스나 기기에 있는 파일에서 폴더나 파일을 선택합니다. 트랙을 라이브러리에 추가하면 해당 트랙에 대한 링크만 생성됩니다 — 실제 파일은 원래 위치에 그대로 저장되어 소중한 디스크 공간을 절약합니다. 트랙을 오프라인에서 사용할 수 있도록 하려면 옵션 메뉴에서 다운로드 작업을 사용하거나 재생 목록 및 트랙 컬렉션에 대해 오프라인 모드를 활성화할 수 있습니다.

Mac 버전에서는 파일을 라이브러리로 드래그 앤 드롭하거나, iPhone 및 iPad에서는 시스템 파일 선택기의 **파일 열기…** / **폴더 열기…**를 사용할 수도 있습니다.

## 계속 재생

이 기능이 애플리케이션 설정에서 활성화된 경우 마지막으로 저장된 위치에서 오디오 플레이어 큐를 복원합니다. 최상의 경험을 위해 설정 → 오디오 플레이어 → 일반에서 **오디오 플레이어 상태 저장**과 **재생 위치 저장** 모두 활성화하세요. 활성화되면 모든 폴더, 앨범, 아티스트, 장르 및 재생 목록 화면 상단에 **계속 재생** 버튼이 나타납니다 — 이를 탭하면 마지막으로 떠난 정확한 트랙과 위치로 즉시 돌아갑니다.

## 위치

라이브러리의 모든 트랙은 소스 유형과 음악 태그별로 신중하게 그룹화됩니다. 오른쪽 상단 모서리의 **추가 작업** 버튼을 사용하고 **필터**를 선택하여 위치별로 곡을 필터링할 수 있습니다.

### 온라인 음악

이 섹션에는 연결된 클라우드 스토리지 서비스의 음악이 표시됩니다. 여기 있는 트랙은 요청 시 스트리밍됩니다; 명시적으로 다운로드하거나 오프라인 모드를 활성화하지 않는 한 로컬 스토리지를 사용하지 않습니다.

### 이 애플리케이션의 파일

여기서 오프라인 재생이 가능한 음악을 찾을 수 있으며, 로컬 파일에서 소싱됩니다. 다운로드한 파일, Wi-Fi Drive를 통해 전송한 파일, 또는 Finder 파일 공유를 통해 가져온 파일이 포함됩니다.

### 이 iPhone / iPad / Mac의 파일

이 카테고리에는 **파일 열기…** 시스템 대화 상자를 통해 기기에서 애플리케이션으로 가져온 음악이 포함됩니다. 원본 파일은 원래 위치에 그대로 유지됩니다; Flacbox는 파일에 대한 링크만 유지합니다.

## 카테고리

음악 라이브러리에 트랙을 추가하면 앱이 자동으로 오디오 태그를 읽고 곡, 미재생 곡, 앨범, 앨범 아티스트, 아티스트, 장르, 작곡가와 같은 카테고리로 구성합니다.

## 태그 그룹화

이러한 카테고리는 음악 태그별로 트랙을 구성하는 데 도움이 됩니다. 음악 라이브러리에 트랙을 추가하면 앱이 메타데이터를 읽고 이러한 카테고리별로 그룹화합니다. 모든 앨범이 표시되지 않는 경우 앱이 모든 트랙을 스캔했는지 확인하세요. 설정 → 음악 라이브러리 → 메타데이터 읽기 → 음악 라이브러리에서 처리된 파일 수에서 스캔 진행 상황을 확인할 수 있습니다. 로컬 파일의 경우 설정 → 음악 라이브러리 → 오프라인 폴더 동기화 → 로컬 폴더 스캔 시작에서 오프라인 폴더를 다시 스캔할 수도 있습니다. 메타데이터 리더가 모든 작업을 완료하면 음악 라이브러리에서 다음 카테고리를 볼 수 있습니다:

- **곡** — TRACK_TITLE 태그별로 그룹화된 모든 곡. 오른쪽 상단 모서리의 추가 작업 메뉴를 사용하여 정렬 순서를 확인할 수 있습니다.
- **미재생 곡** — 한 번도 재생되지 않은 모든 곡.
- **앨범** — 아티스트, 앨범 아티스트, 작곡가 태그를 무시하고 ALBUM_NAME 태그별로 그룹화된 곡. 같은 이름이지만 다른 아티스트의 앨범이 여러 개 있는 경우 아래에 설명된 독점 앨범 정렬을 사용하는 것이 좋습니다.
- **앨범 아티스트** — ALBUM_ARTIST_TAG로만 그룹화된 곡. 컴필레이션과 협업을 솔로 작업에서 명확하게 분리하는 데 유용합니다.
- **아티스트** — ARTIST_TAG로만 그룹화된 곡.
- **장르** — GENRE 태그별로 그룹화된 곡.
- **작곡가** — COMPOSER 태그별로 그룹화된 곡 — '작곡가'가 기본 탐색 축인 클래식 음악 라이브러리에 매우 유용합니다.

## 즐겨찾기

오디오 플레이어 화면이나 옵션 메뉴에서 곡을 즐겨찾기로 표시할 수 있습니다. 즐겨찾기는 한 번의 탭으로 찾을 수 있는 자체 섹션에 표시됩니다.

## 최근 항목

이 섹션에는 최근에 재생된 모든 트랙이 표시됩니다. 설정 → 라이브러리 → 최근 항목 → 목록 크기 변경에서 최근 항목 목록이 유지하는 항목 수를 맞춤 설정하고, 청취 기록을 백업하기 위해 목록을 M3U / CSV / TXT로 내보낼 수 있습니다.

## 북마크

곡이 재생되는 동안 오디오 북마크를 만들고 이 화면에서 관리할 수 있습니다 — 오디오북, 긴 믹스, 강의, 또는 좋아하는 트랙의 코러스 표시에 완벽합니다. 오디오 북마크 작업에 대한 자세한 지침은 [여기](/docs/guide/evermusic/evermusic-guide-music-library)에서 확인할 수 있습니다.

## 상단 도구 모음

탐색 바 바로 아래에 있는 상단 도구 모음은 검색, 모두 재생, 모두 셔플, 계속 재생과 같은 여러 편리한 작업을 제공합니다. 아래로 스와이프하여 이 도구 모음을 표시하거나 숨길 수 있습니다.

## 검색

검색 기능을 사용하면 음악 라이브러리에서 특정 트랙, 아티스트, 앨범 또는 장르를 찾을 수 있습니다. 검색 화면에서 정렬, 필터 및 그리드 / 목록 보기 작업에 액세스할 수 있습니다. 검색은 음악 라이브러리 데이터베이스에 대해 로컬로 실행되므로 완전히 오프라인으로 작동하고 입력하는 동안 결과를 반환합니다.

## 옵션 메뉴

음악 라이브러리의 각 곡에는 곡 제목 근처의 점 세 개 버튼을 탭하여 액세스할 수 있는 더 많은 작업 메뉴가 있습니다.

### 개별 곡의 경우

- **다음 재생** — 플레이어 큐의 상단에 곡을 추가합니다.
- **나중에 재생** — 플레이어 큐의 하단에 곡을 추가합니다.
- **재생 목록에 추가** — 곡을 재생 목록에 추가합니다.
- **즐겨찾기에 추가** — 빠른 액세스를 위해 곡을 즐겨찾기로 표시합니다.
- **다운로드** — 로컬 파일에 곡을 저장합니다. **로컬 파일** 탭과 **오프라인 음악** 섹션에 나타납니다.
- **오디오 태그 편집** — 누락된 메타데이터를 수정하기 위해 내장 오디오 태그 편집기를 엽니다; 이것은 스토리지의 곡을 변경합니다.
- **폴더에서 보기** — 오디오 파일이 저장된 폴더를 표시합니다.
- **Finder에서 보기** — Mac에서 가져온 파일의 경우 Mac에서 오디오 파일이 있는 폴더를 표시합니다.
- **다른 앱으로 열기** — 오디오 파일을 다른 앱으로 내보냅니다.
- **클라우드 서비스에서 삭제** — 음악 라이브러리와 클라우드 스토리지 모두에서 파일을 제거합니다. **이 작업은 되돌릴 수 없습니다.**
- **음악 라이브러리에서 삭제** — 음악 라이브러리에서 곡을 삭제하지만 파일은 스토리지에 그대로 남습니다. 자동 동기화가 활성화되어 있고 파일이 원격 스토리지에 있는 경우 동기화 작업 후 라이브러리에 다시 나타납니다.

### 곡 컬렉션의 경우

앨범, 아티스트, 장르 또는 작곡가와 같은 곡 컬렉션의 경우 옵션 메뉴에는 다음 작업이 포함됩니다.

- **모두 재생** — 선택한 컬렉션의 곡으로 플레이어 큐를 교체합니다.
- **다음 재생** — 이 컬렉션의 곡을 플레이어 큐의 상단에 추가합니다.
- **나중에 재생** — 이 컬렉션의 곡을 플레이어 큐의 하단에 추가합니다.
- **재생 목록에 추가** — 새 재생 목록을 만들 수 있는 옵션으로 이 컬렉션의 곡을 재생 목록에 포함합니다.
- **오프라인 모드 활성화** — 로컬 파일로 이 컬렉션의 곡을 다운로드합니다. 파일은 **로컬 파일** 탭과 **오프라인 음악** 섹션에 나타납니다. 서버의 컬렉션에 새 항목이 추가되면 자동으로 다운로드됩니다.
- **이미지 편집** — 곡 컬렉션의 앨범 커버를 변경합니다.
- **아카이브에 추가** — 쉬운 백업이나 전송을 위해 전체 컬렉션을 단일 ZIP 파일로 묶습니다(프리미엄 기능).
- **곡 목록 내보내기** — 다른 플레이어에서 사용하거나 보관을 위해 컬렉션을 M3U, CSV 또는 TXT로 내보냅니다.
- **음악 라이브러리에서 삭제** — 음악 라이브러리에서 곡 컬렉션을 제거합니다. 이 작업은 스토리지에서 실제 파일을 삭제하지 않습니다. 자동 동기화가 활성화되어 있고 파일이 원격 스토리지에 있는 경우 동기화 작업 후 라이브러리에 다시 나타납니다.

## 선택 모드

오른쪽 상단 모서리의 추가 작업 버튼을 사용하여 선택 모드를 활성화할 수 있습니다. 이 모드에서는 여러 트랙을 한 번에 선택하고 일괄 작업을 수행할 수 있습니다 — 재생 목록에 추가, 즐겨찾기로 표시, 라이브러리에서 삭제, 다운로드 등.

## 앨범 세부 정보

아티스트, 앨범 아티스트 또는 작곡가 섹션을 열면 곡 / 전체 앨범 / 독점 앨범 / 솔로 앨범에 대한 전환기를 볼 수 있습니다.

- **곡** — 오디오 태그에 이 아티스트 / 앨범 아티스트 / 작곡가가 있는 모든 곡을 표시합니다.
- **전체 앨범** — 아티스트가 전혀 있는 컴필레이션 앨범과 모든 앨범을 표시합니다.
- **독점 앨범** — 지정된 아티스트가 해당 앨범 이름을 가진 유일한 아티스트인 앨범을 표시합니다.
- **솔로 앨범** — 다른 아티스트가 같은 이름의 앨범을 가지고 있더라도 지정된 아티스트의 트랙만 나타나는 앨범을 표시합니다.

이것은 대형 라이브러리에서 혼잡한 '다양한 아티스트' 컴필레이션을 정리하는 데 특히 유용합니다.

## 설정

추가 작업 → 설정을 탭하여 음악 라이브러리 기본 설정을 구성합니다.

### 메타데이터 읽기

트랙을 라이브러리에 추가하면 메타데이터 리더가 작동을 시작합니다. 이 백그라운드 프로세스는 트랙에서 모든 메타데이터를 읽고 아티스트, 앨범, 장르, 작곡가별로 구성합니다. 데이터를 더 빠르게 로드하기 위해 메타데이터 읽기 속도를 조정할 수 있습니다 — 더 빠른 읽기는 더 많은 에너지를 사용한다는 점을 알아두세요. 메타데이터 리더를 완전히 비활성화하고 태그 정보 대신 파일 이름을 표시할 수도 있습니다.

중요하게도, 메타데이터 리더는 음악 라이브러리의 메타데이터만 업데이트하며 클라우드 계정이나 로컬 스토리지에 저장된 파일을 변경하지 않습니다.

### 메타데이터 리더의 사용 가능한 모드

- **비활성화됨** — 메타데이터 리더가 꺼져 있고 오디오 태그 데이터 대신 파일 이름이 표시됩니다.
- **현재 곡** — 애플리케이션은 현재 재생 중인 곡의 메타데이터만 읽습니다. 네트워크 연결이 느린 경우 이 옵션을 사용하여 메타데이터 리더가 클라우드 서버에 많은 요청을 보내는 것을 방지합니다.
- **재생 큐** — 앱은 오디오 플레이어 큐의 모든 곡에 대한 메타데이터를 읽습니다.
- **라이브러리** — 앱은 음악 라이브러리의 모든 곡에 대한 메타데이터를 읽습니다.

**백그라운드에서 메타데이터 읽기** 스위치가 켜져 있으면 메타데이터 리더가 백그라운드 모드에서 계속 작동합니다.

**메타데이터 인코딩 정규화** 옵션이 활성화되면 앱이 음악 라이브러리의 모든 곡에 대해 메타데이터 인코딩을 자동으로 정규화합니다.

**메타데이터 새로 고침** 작업은 음악 라이브러리의 모든 파일을 누락된 메타데이터가 있는 것으로 표시하여 메타데이터 리더가 모든 레코드를 새로 고치도록 트리거합니다.

**메타데이터 읽기 시작**을 탭하여 리더를 수동으로 트리거합니다.

### 온라인 동기화

자동 온라인 음악 동기화를 사용하면 연결된 클라우드 서비스의 트랙을 음악 라이브러리에 자동으로 추가할 수 있습니다. 이 기능을 활성화하려면 음악 라이브러리 설정으로 이동하여 동기화 폴더를 선택합니다.

온라인 음악 동기화는 앱이 전면에 있을 때만 작동하므로 대규모 라이브러리를 동기화하는 데 시간이 걸릴 수 있습니다. 프로세스를 빠르게 하려면 앱을 열어 두고, 전원 소스에 연결하고, 설정 → 화면 → 항상 활성화를 활성화하세요.

### 오프라인 동기화

여기서 오프라인 음악 동기화를 구성할 수 있습니다.

#### 동기화된 오프라인 폴더

클라우드 폴더를 오프라인으로 사용할 수 있도록 설정하면(추가 작업 메뉴를 통해) 로컬 파일 → 오프라인 폴더 섹션에 나타납니다. 앱은 내용을 기기로 다운로드합니다. 온라인 폴더에 변경 사항이 생기면(파일 추가, 제거 또는 업데이트) 앱이 변경 사항을 감지하고 로컬 복사본을 자동으로 업데이트합니다.

#### 시간 간격

앱이 오프라인 폴더의 수정 사항을 확인하는 시간 간격을 설정할 수 있습니다.

#### 로컬 폴더 스캔 시작

이 옵션은 애플리케이션의 Documents 디렉토리에 있는 모든 로컬 폴더를 스캔하여 지원되는 오디오 파일을 찾습니다.

#### 중요

음악 라이브러리를 로컬 파일로 최신 상태로 유지하기 위해 주기적으로 오프라인 음악 동기화를 시작하는 것이 좋습니다.

### 개인화

이 섹션에서 음악 라이브러리 화면 스타일을 기본 설정에 맞게 구성할 수 있습니다. 일반 메뉴, 그룹화 메뉴, 탭 메뉴의 세 가지 옵션이 있습니다. 앨범 세부 화면에서 앨범 커버 표시를 활성화하거나 비활성화할 수도 있습니다.

### 앨범 커버

여기서 애플리케이션이 앨범 아트워크를 로드하고 저장하는 방법을 구성할 수 있습니다.

- **네트워크 유형** — 커버 다운로드를 위해 Wi-Fi 또는 Wi-Fi & 셀룰러 데이터를 선택합니다.
- **온라인 파일의 앨범 커버 로드** — 활성화되면 클라우드 스토리지에 저장된 파일의 내장 앨범 커버가 로드됩니다.
- **폴더에서 검색** — 활성화되면 트랙에 내장 커버가 없는 경우 앱이 같은 폴더에서 JPEG 또는 PNG 이미지를 찾아 앨범 아트워크로 사용합니다.
- **커버 품질** — 기기에 저장된 앨범 커버의 품질을 선택합니다.
- **폴더에서 보기** — 앨범 커버가 캐시된 폴더를 엽니다.
- **모두 삭제** — 캐시된 앨범 커버를 삭제하여 스토리지 공간을 확보하고 앱이 요청 시 다시 로드하도록 합니다.

### 재생 목록

같은 곡을 재생 목록에 두 번 추가할 수 있는 옵션을 활성화할 수 있습니다. 기본적으로 이 옵션은 비활성화되어 있습니다.

### 최근 항목

최근에 재생된 곡 목록을 관리할 수 있습니다.

- **목록 삭제** — 최근에 재생된 곡의 전체 목록을 삭제합니다.
- **목록 크기 변경** — 목록에 나타나는 항목 수를 설정합니다.
- **곡 목록 내보내기** — 최근에 재생된 곡 목록을 M3U, CSV 또는 TXT로 내보냅니다. 자세한 지침은 [여기](/docs/howto/export-tracks-collection-from-evermusic-flacbox-to-m3u-csv-txt/)에서 확인할 수 있습니다.

### 즐겨찾기

즐겨찾는 곡 목록을 관리할 수 있습니다.

- **동시 편집** — 활성화되면 곡을 즐겨찾기에 추가할 때 음악 라이브러리와 파일 섹션 모두에 동시에 추가됩니다.
- **목록 삭제** — 즐겨찾는 곡의 전체 목록을 삭제합니다.
- **곡 목록 내보내기** — 최근 항목과 유사하게 즐겨찾기 목록을 M3U, CSV 또는 TXT로 내보냅니다.

### 라이브러리 삭제

이 작업은 음악 라이브러리 데이터베이스를 삭제하지만 스토리지의 음악 파일은 그대로 유지합니다.

### 콘텐츠 로딩 제한

기본적으로 애플리케이션은 콘텐츠 로딩 시간을 줄이고 대규모 라이브러리의 반응성을 유지하기 위해 페이지 매김을 사용합니다. 그러나 이 옵션을 비활성화하고 모든 사용 가능한 데이터를 한 번에 로드할 수 있습니다. 이를 위해 애플리케이션 설정을 열고 개인화 → 콘텐츠 로딩 제한으로 스크롤하고 비활성화됨을 선택합니다.

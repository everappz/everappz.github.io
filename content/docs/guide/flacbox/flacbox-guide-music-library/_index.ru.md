Управление музыкальной библиотекой — простая задача в Flacbox, где можно без труда организовать все свои треки — локальные FLAC, ALAC, DSD, MP3, M4A, OGG, WMA, APE и десятки других форматов — в единую, доступную для поиска коллекцию. Есть два варианта создания музыкальной библиотеки: ручное добавление (вы сами выбираете, что добавляется) или автоматическая синхронизация (Flacbox сканирует указанные папки в облаке и автоматически добавляет новые файлы по мере их появления).

## Ручное добавление

Для ручного добавления треков нажмите иконку **Добавить музыку** в верхнем левом углу и выберите папки или файлы из подключённого облачного сервиса или файлы на устройстве. При добавлении треков в библиотеку создаются только ссылки на них — реальные файлы остаются в исходных местах, чтобы сэкономить ценное место на диске. Если нужно сделать треки доступными офлайн, можно воспользоваться действием «Скачать» в меню параметров или включить Офлайн-режим для плейлистов и коллекций треков.

Также можно перетаскивать файлы в библиотеку в версии для Mac или использовать **Открыть файлы…** / **Открыть папку…** из системного выбора файлов на iPhone и iPad.

## Продолжить воспроизведение

Восстановить очередь аудиоплеера с последней сохранённой позиции, если эта функция включена в настройках приложения. Включите и **Сохранить состояние аудиоплеера**, и **Сохранить позицию воспроизведения** в Настройки → Аудиоплеер → Общие для лучшего опыта. После включения кнопка **Продолжить воспроизведение** появляется в верхней части каждого экрана папки, альбома, исполнителя, жанра и плейлиста — нажмите её, чтобы вернуться прямо к треку и позиции, где остановились.

## Местоположения

Все треки в библиотеке удобно сгруппированы по типу источника и музыкальным тегам. Можно фильтровать песни по местоположению с помощью кнопки **Другие действия** в правом верхнем углу и выбора пункта **Фильтр**.

### Онлайн-музыка

Этот раздел содержит музыку из подключённых облачных сервисов. Треки здесь транслируются по запросу; ничто не занимает место на устройстве до явного скачивания или включения офлайн-режима.

### Файлы в этом приложении

Здесь находится музыка, доступная для офлайн-воспроизведения, из локальных файлов. Это включает файлы в директории «Документы» приложения — всё, что загружено, перенесено через Wi-Fi Drive или импортировано через Общий доступ к файлам Finder.

### Файлы на этом iPhone / iPad / Mac

Эта категория включает музыку, импортированную в приложение с устройства, добавленную через системный диалог **Открыть файлы…**. Исходные файлы остаются в исходном месте; Flacbox хранит только ссылки на них.

## Категории

При добавлении треков в музыкальную библиотеку приложение автоматически считывает их аудиотеги и организует в категории: Песни, Непрослушанные песни, Альбомы, Исполнители альбомов, Исполнители, Жанры и Композиторы.

## Группировка по тегам

Эти категории помогают организовывать треки по музыкальным тегам. При добавлении треков в музыкальную библиотеку приложение считывает их метаданные и группирует по этим категориям. Если не все альбомы отображаются, убедитесь, что приложение просканировало каждый трек. Прогресс сканирования можно проверить в Настройки → Музыкальная библиотека → Чтение метаданных → Количество обработанных файлов в музыкальной библиотеке. Для локальных файлов также можно выполнить повторное сканирование офлайн-папок в Настройки → Музыкальная библиотека → Синхронизация офлайн-папок → Начать сканирование локальных папок. После завершения всех операций читателем метаданных в музыкальной библиотеке появятся следующие категории:

- **Песни** — все песни, сгруппированные по тегу TRACK_TITLE. Порядок сортировки можно проверить через меню «Другие действия» в правом верхнем углу.
- **Непрослушанные песни** — все песни, которые никогда не воспроизводились.
- **Альбомы** — песни, сгруппированные по тегу ALBUM_NAME, без учёта тегов исполнителя, исполнителя альбома и композитора. Если есть несколько альбомов с одинаковым именем, но разными исполнителями, рассмотрите использование сортировки «Эксклюзивные альбомы».
- **Исполнители альбомов** — песни, сгруппированные только по ALBUM_ARTIST_TAG. Удобно для чёткого разграничения компиляций и коллабораций от сольного творчества.
- **Исполнители** — песни, сгруппированные только по ARTIST_TAG.
- **Жанры** — песни, сгруппированные по тегу GENRE.
- **Композиторы** — песни, сгруппированные по тегу COMPOSER — бесценно для библиотек классической музыки, где «композитор» является основной осью навигации.

## Избранное

Песни можно отмечать как избранные на экране аудиоплеера или с помощью меню параметров. Избранные отображаются в отдельном разделе, позволяя найти их одним касанием.

## Недавние

В этом разделе отображаются все недавно воспроизведённые треки. Можно настроить количество элементов в списке «Недавние» в Настройки → Библиотека → Недавние → Изменить размер списка, а также экспортировать список в M3U / CSV / TXT для сохранения истории прослушивания.

## Закладки

Во время воспроизведения можно создавать аудиозакладки и управлять ими на этом экране — идеально для аудиокниг, длинных миксов, лекций или пометки припева любимого трека. Подробные инструкции по работе с аудиозакладками доступны [здесь](/docs/guide/evermusic/evermusic-guide-music-library).

## Верхняя панель инструментов

Верхняя панель инструментов, расположенная прямо под навигационной панелью, предлагает несколько удобных действий: Поиск, Воспроизвести всё, Перемешать всё и Продолжить воспроизведение. Эту панель инструментов можно показать или скрыть простым жестом смахивания вниз.

## Поиск

Функция поиска позволяет найти конкретный трек, исполнителя, альбом или жанр в музыкальной библиотеке. На экране поиска доступны действия «Сортировка», «Фильтр» и «Сетка / Список». Поиск выполняется локально по базе данных музыкальной библиотеки, поэтому работает полностью офлайн и возвращает результаты по мере набора текста.

## Меню параметров

У каждой песни в музыкальной библиотеке есть меню с дополнительными действиями, доступное через нажатие кнопки с тремя точками рядом с названием. Действия различаются в зависимости от того, является ли это отдельной песней или частью коллекции.

### Для отдельных песен

- **Воспроизвести следующим** — добавляет песню в начало очереди плеера.
- **Воспроизвести позже** — добавляет песню в конец очереди плеера.
- **Добавить в плейлист** — добавляет песню в плейлист.
- **Добавить в Избранное** — отмечает песню как избранную для быстрого доступа.
- **Скачать** — сохраняет песню в локальных файлах. Появляется на вкладке **Локальные файлы** и в разделе **Офлайн-музыка**.
- **Редактировать теги** — открывает встроенный редактор аудиотегов для исправления отсутствующих метаданных; обратите внимание, что это изменит файл в хранилище.
- **Показать в папке** — открывает папку, в которой хранится аудиофайл.
- **Показать в Finder** — для файлов, импортированных с Mac, это действие открывает папку, где находится аудиофайл на Mac.
- **Открыть в** — экспортирует аудиофайл в другое приложение.
- **Удалить из облачного сервиса** — удаляет файл как из музыкальной библиотеки, так и из облачного хранилища. **Это действие необратимо.**
- **Удалить из музыкальной библиотеки** — удаляет песню из музыкальной библиотеки, но файл остаётся в хранилище. Если включена автоматическая синхронизация и файл существует в удалённом хранилище, он снова появится в библиотеке после операции синхронизации.

### Для коллекций треков

Для коллекций треков, таких как Альбомы, Исполнители, Жанры или Композиторы, меню параметров включает следующие действия.

- **Воспроизвести всё** — заменяет очередь плеера треками из выбранной коллекции.
- **Воспроизвести следующим** — добавляет треки этой коллекции в начало очереди плеера.
- **Воспроизвести позже** — добавляет треки этой коллекции в конец очереди плеера.
- **Добавить в плейлист** — включает треки этой коллекции в плейлист с возможностью создания нового.
- **Включить офлайн режим** — загружает треки из этой коллекции в локальные файлы. Файлы появляются на вкладке **Локальные файлы** и в разделе **Офлайн-музыка**. Если на сервере к коллекции добавляются новые элементы, они загружаются автоматически.
- **Редактировать изображение** — изменить обложку альбома для коллекции треков.
- **Добавить в архив** — упаковать всю коллекцию в единый ZIP-файл для удобного резервного копирования или переноса (функция Premium).
- **Экспорт списка треков** — экспортировать коллекцию в M3U, CSV или TXT для использования в других плеерах или архивирования.
- **Удалить из музыкальной библиотеки** — удаляет коллекцию треков из музыкальной библиотеки. Это действие не удаляет реальные файлы из хранилища. Если включена автоматическая синхронизация и файлы существуют в удалённом хранилище, они снова появятся в библиотеке после операции синхронизации.

## Режим выбора

Режим выбора активируется с помощью кнопки «Другие действия» в правом верхнем углу. В этом режиме можно выбрать несколько треков одновременно и выполнять пакетные действия — добавление в плейлист, пометка как избранные, удаление из библиотеки, скачивание и многое другое.

## Детали альбома

При открытии разделов «Исполнитель», «Исполнитель альбома» или «Композитор» можно увидеть переключатель «Песни / Все альбомы / Эксклюзивные альбомы / Сольные альбомы».

- **Песни** — отображает все песни, где этот Исполнитель / Исполнитель альбома / Композитор указан в аудиотегах.
- **Все альбомы** — показывает компиляционные альбомы и все альбомы, в которых исполнитель присутствует.
- **Эксклюзивные альбомы** — показывает альбомы, где указанный исполнитель является единственным с таким именем альбома.
- **Сольные альбомы** — показывает альбомы, где присутствуют только треки указанного исполнителя, даже если другие исполнители имеют альбомы с таким же именем.

Это особенно полезно для упорядочивания загромождённых компиляций «Различных исполнителей» в больших библиотеках.

## Настройки

Нажмите «Другие действия» → «Настройки», чтобы настроить параметры музыкальной библиотеки.

### Чтение метаданных

При добавлении треков в библиотеку читатель метаданных приступает к работе. Этот фоновый процесс считывает все метаданные из треков и организует их по Исполнителю, Альбому, Жанру и Композитору. Можно настроить скорость чтения метаданных для более быстрой загрузки данных — более быстрое чтение потребляет больше энергии. Также можно полностью отключить читатель метаданных и отображать имена файлов вместо информации из тегов.

Важно: читатель метаданных обновляет только метаданные в музыкальной библиотеке и не изменяет файлы, хранящиеся в облачном аккаунте или локальном хранилище. Для редактирования метаданных аудиофайлов используйте встроенный редактор тегов, доступный через соответствующее действие в меню параметров.

### Доступные режимы читателя метаданных

- **Отключён** — читатель метаданных выключен, вместо данных аудиотегов отображаются имена файлов.
- **Текущий трек** — приложение считывает метаданные только для текущего воспроизводимого трека. Используйте эту опцию при медленном сетевом подключении, чтобы предотвратить отправку многих запросов к облачному серверу, которые могут вызывать прерывания воспроизведения.
- **Очередь воспроизведения** — приложение считывает метаданные для всех треков в очереди аудиоплеера.
- **Библиотека** — приложение считывает метаданные для всех треков в музыкальной библиотеке.

При включённом переключателе **Чтение метаданных в фоне** читатель метаданных продолжает работать в фоновом режиме. Обратите внимание: если приложение потребляет много энергии во время воспроизведения аудио, iOS может его приостановить.

Поэтому при большой музыкальной коллекции рекомендуется использовать десктопную версию приложения для синхронизации метаданных. Затем можно использовать функцию «Резервная копия и восстановление» в настройках приложения для переноса синхронизированной библиотеки с компьютера на мобильное устройство.

При включённой опции **Нормализовать кодировку метаданных** приложение автоматически нормализует кодировку метаданных для всех треков в музыкальной библиотеке. Это исправляет проблемы с повреждённой кодировкой аудиотегов (например, после редактирования файлов на Windows PC) и предотвращает отображение неправильных символов в информации о треках.

Действие **Перезагрузить метаданные** помечает каждый файл в музыкальной библиотеке как имеющий отсутствующие метаданные, заставляя читатель метаданных обновить каждую запись.

Нажмите **Начать чтение метаданных**, чтобы запустить читатель вручную. Прогресс операции отображается ниже.

### Онлайн-синхронизация

Автоматическая онлайн-синхронизация музыки позволяет автоматически добавлять треки из подключённых облачных сервисов в музыкальную библиотеку. Для активации этой функции перейдите в Настройки музыкальной библиотеки и выберите папки для синхронизации.

При включённой опции приложение сканирует все выбранные папки, определяет поддерживаемые аудиофайлы и плавно интегрирует их в библиотеку. Синхронизацию можно запустить или остановить через соответствующее действие в меню.

Онлайн-синхронизация музыки работает исключительно когда приложение находится на переднем плане, поэтому синхронизация большой библиотеки может занять некоторое время. Для ускорения процесса оставьте приложение открытым, подключите устройство к источнику питания и включите Экран → Всегда активен в настройках приложения.

В качестве альтернативы можно выполнить онлайн-синхронизацию музыки в десктопной версии приложения и перенести музыкальную библиотеку в версию iOS с помощью функции «Резервная копия и восстановление».

Также можно настроить частоту синхронизации онлайн-библиотеки. При установке интервала «Немедленно» онлайн-синхронизация будет запускаться каждый раз при открытии приложения.

### Офлайн-синхронизация

Здесь можно настроить синхронизацию офлайн-музыки.

#### Синхронизированные офлайн папки

При переводе папки в облаке в офлайн-режим (через меню «Другие действия») она появляется в разделе Локальные файлы → Офлайн папки. Приложение скачивает её содержимое на устройство. При изменении папки в облаке — добавлении, удалении или обновлении файлов — приложение обнаруживает эти изменения и автоматически обновляет локальную копию.

На этом экране можно вручную запустить синхронизацию офлайн-папки, показать её в содержащей папке и отключить офлайн-режим для папки. Отключение офлайн-режима удаляет все локальные копии файлов с устройства.

#### Интервал времени

Можно задать интервал времени, как часто приложение проверяет офлайн-папки на наличие изменений.

#### Начать сканирование локальных папок

Эта опция сканирует все локальные папки в директории «Документы» приложения для поиска поддерживаемых аудиофайлов. Все эти локальные файлы автоматически добавляются в музыкальную библиотеку. Локальные файлы на устройстве, но за пределами директории «Документы» этого приложения, необходимо добавлять в музыкальную библиотеку вручную, так как приложение не имеет доступа к ним из-за ограничений безопасности iOS / macOS.

#### Важно

Рекомендуется периодически инициировать офлайн-синхронизацию музыки для поддержания актуальности музыкальной библиотеки с локальными файлами.

### Персонализация

В этом разделе можно настроить стиль экрана музыкальной библиотеки по своим предпочтениям. Доступны три варианта: Обычное меню, Сгруппированное меню, Меню с вкладками. Также можно включить или отключить отображение обложек альбомов на экранах деталей альбома.

### Обложки альбомов

Здесь можно настроить загрузку и хранение обложек альбомов.

- **Тип сети** — выберите Wi-Fi или Wi-Fi и мобильные данные для загрузки обложек.
- **Загрузить обложки альбомов для онлайн-файлов** — при включении встроенные обложки альбомов загружаются для файлов в облачном хранилище. Может потребоваться дополнительный трафик.
- **Искать в папке** — при включении, если у трека нет встроенной обложки, приложение ищет изображения JPEG или PNG в той же папке и использует их как обложку альбома.
- **Качество обложки** — выберите качество обложек альбомов, хранящихся на устройстве.
- **Показать в папке** — открыть папку, где кэшируются обложки альбомов, для управления или резервного копирования.
- **Удалить все** — удалить кэшированные обложки альбомов для освобождения места и принудительной перезагрузки по требованию.

По умолчанию приложение проверяет встроенные обложки альбомов в треках и отображает их при наличии. Если встроенных обложек нет и включена опция **Искать в папке**, приложение проверяет содержащую папку на изображения JPEG или PNG и использует их как обложки альбомов для всех треков в этой папке.

### Плейлисты

Можно включить опцию добавления одной и той же песни в плейлист дважды. По умолчанию эта опция отключена.

### Недавние

Можно управлять списком недавно воспроизведённых песен.

- **Удалить список** — удалить весь список недавно воспроизведённых песен.
- **Изменить размер списка** — задать количество элементов в списке.
- **Экспорт списка треков** — экспортировать список недавно воспроизведённых песен в M3U, CSV или TXT. Подробные инструкции доступны [здесь](/docs/howto/export-tracks-collection-from-evermusic-flacbox-to-m3u-csv-txt/).

### Избранное

Можно управлять списком любимых песен.

- **Одновременное редактирование** — при включении добавление песни в избранное добавляет её одновременно в музыкальную библиотеку и в раздел файлов.
- **Удалить список** — удалить весь список любимых песен.
- **Экспорт списка треков** — аналогично «Недавним», экспортировать список избранного в M3U, CSV или TXT.

### Удалить библиотеку

Это действие сотрёт базу данных музыкальной библиотеки, но оставит музыкальные файлы нетронутыми в хранилище.

### Лимит загрузки контента

По умолчанию приложение использует пагинацию для сокращения времени загрузки контента и обеспечения отзывчивости больших библиотек. Однако эту опцию можно отключить, разрешив приложению загружать все доступные данные сразу. Для этого откройте настройки приложения, прокрутите вниз до Персонализация → Лимит загрузки контента и выберите «Отключён».

로컬 파일 섹션은 응용 프로그램의 "문서" 폴더에 있는 파일과 오프라인 재생에 사용할 수 있는 기기에서 추가된 파일을 관리하는 허브 역할을 합니다.

이 내장 파일 관리자를 사용하면 파일을 편집하고 다양한 방법으로 앱에 오디오 파일을 가져올 수 있습니다.

## 상단 도구 모음

탐색 바 아래에 있는 상단 도구 모음은 다음과 같은 몇 가지 작업을 제공합니다:

- **검색:** 현재 폴더 내에서 검색을 수행합니다.
- **재생 계속:** 응용 프로그램 설정에서 활성화된 경우, 이 기능은 현재 폴더의 오디오 플레이어 대기열과 마지막 미디어 위치를 복원하여 마지막 재생을 중단한 곳에서 재개할 수 있습니다.
- **모두 재생:** 현재 폴더와 하위 폴더를 스캔하고 현재 정렬 순서로 플레이어 대기열에 파일을 추가합니다.
- **모두 셔플:** "모두 재생"과 유사하지만, 오디오 플레이어 대기열에 추가하기 전에 파일을 셔플합니다.

아래로 스와이프하는 제스처를 사용하여 상단 도구 모음을 표시하거나 숨길 수 있습니다.

## 특수 폴더

"로컬 파일" 화면에서 여러 특수 폴더를 볼 수 있습니다.

## 다운로드

클라우드 스토리지 서비스에서 다운로드된 모든 파일은 기본적으로 여기에 표시됩니다. 응용 프로그램 설정에서 이 동작을 사용자 지정할 수 있습니다.

## 오디오 플레이어

이것은 오디오 플레이어 캐시 폴더입니다. 기본적으로 오디오 플레이어는 크로스페이드를 적용하고 재생을 최적화하기 위해 오디오 플레이어 대기열의 일부 노래를 다운로드합니다. 앱 설정에서 오디오 플레이어 캐시를 비활성화하거나 "로컬 파일"에서 이 폴더를 삭제할 수 있습니다.

## iCloud

이 폴더에 파일을 넣으면 동일한 iCloud 계정에 연결된 모든 기기에서 동기화됩니다.

## 오프라인 폴더

플레이리스트, 앨범, 아티스트, 또는 온라인 폴더를 오프라인에서 사용 가능하게 만들면 모든 파일이 이 폴더에 다운로드됩니다.  
특정 항목의 오프라인 모드를 비활성화해야 하는 경우, 추가 작업 메뉴를 사용하거나 "오프라인 폴더"에 있는 해당 폴더를 삭제할 수 있습니다.

## 두 개의 구별된 섹션

"로컬 파일" 화면은 두 개의 구별된 섹션으로 구성되어 있습니다.

## 이 응용 프로그램의 파일

앱의 문서 디렉토리와 iCloud Drive에 저장된 파일 및 폴더를 표시합니다.

## 이 iPhone/iPad/Mac의 파일

기기에 있지만 다른 응용 프로그램에 있는 파일을 표시합니다. 시스템 파일 선택기를 사용하여 이 응용 프로그램으로 가져올 수 있습니다. 선택기를 활성화하려면 파일을 선택하려면 "파일 열기..."를 선택하고 폴더를 선택하려면 "폴더 열기..."를 선택합니다. iPhone 또는 Mac에 저장된 로컬 음악을 가져오는 방법에 대한 자세한 지침은 [여기](/docs/howto/how-to-play-local-music-stored-on-your-iphone-or-mac)에서 확인할 수 있습니다. 기기에 있는 폴더를 연결하고 폴더 내용에 빠르게 접근할 수도 있습니다. "폴더 연결" 메뉴 항목을 사용하여 기기에 있는 폴더를 선택합니다. "완료됨"을 탭하면 앱이 읽기/쓰기 권한으로 해당 폴더에 대한 링크를 만들어 이 앱에서 직접 파일을 관리할 수 있습니다. 기기에 있는 폴더 연결을 끊으려면 "추가 작업" 버튼을 탭하고 "연결 끊기"를 선택합니다.

## 추가 작업

현재 열린 폴더의 오른쪽 상단 모서리에 있는 추가 작업 메뉴는 다양한 작업에 접근할 수 있습니다.

- **선택하다:** 파일 및 폴더에 대한 선택 모드로 전환합니다.
- **새 폴더:** 현재 폴더 내에 새 폴더를 만듭니다.
- **검색:** 특정 파일이나 폴더를 찾기 위해 검색 기능을 활성화합니다.
- **정렬:** 이름, 크기, 날짜, 또는 메타데이터 정보를 사용하여 파일을 정리합니다.
- **목록/격자:** 콘텐츠 표현을 테이블에서 격자 보기로 변경합니다.
- **도움말:** 도움말 지침에 접근합니다.

## 선택 모드

여러 파일을 편집해야 하는 경우, 탐색 바의 오른쪽 상단 모서리에 있는 추가 작업 버튼 "..."을 탭한 다음 "선택하다" 메뉴 항목을 선택하여 선택 모드를 활성화합니다. 이렇게 하면 각 파일 옆에 체크박스가 표시됩니다. 체크박스를 탭하여 원하는 파일을 선택합니다. 선택한 파일에 대해 다음 작업을 수행할 수 있습니다.

{{< cards cols="1">}}
  {{< card title="" subtitle="로컬 파일에 대한 선택 모드 작업" image="/docs/guide/evermusic/evermusic-guide-local-files/img/local-files-selection-mode.webp" >}}
{{< /cards >}}

- **다음에 재생:** 선택한 파일이나 폴더를 현재 정렬 순서로 플레이어 대기열의 상단에 추가합니다.
- **나중에 재생:** 선택한 파일이나 폴더를 현재 정렬 순서로 플레이어 대기열의 하단에 추가합니다.
- **음악 라이브러리에 추가:** 선택한 파일이나 폴더를 음악 라이브러리에 포함합니다.
- **플레이리스트에 추가:** 선택한 파일이나 폴더를 플레이리스트에 추가합니다.
- **복사:** 선택한 파일이나 폴더의 복사본을 만듭니다.
- **업로드:** 선택한 파일이나 폴더를 원격 스토리지에 업로드합니다.
- **이동:** 선택한 파일이나 폴더를 기기의 다른 위치로 이동합니다.
- **이름 바꾸기:** 선택한 파일이나 폴더의 이름을 변경합니다.
- **삭제하기:** 기기에서 선택한 파일이나 폴더를 제거합니다. 이 작업은 되돌릴 수 없습니다.

## 옵션 메뉴

앱의 모든 파일이나 폴더에 대해 추가 버튼 "..."을 탭하여 접근할 수 있는 여러 작업을 사용할 수 있습니다. 모든 작업이 표시되지 않으면 아래로 스크롤하여 확인하세요:

- **모두 재생:** 플레이어 대기열을 선택한 폴더의 항목으로 교체합니다.
- **다음에 재생:** 파일이나 폴더를 플레이어 대기열의 상단에 추가합니다.
- **나중에 재생:** 파일이나 폴더를 플레이어 대기열의 하단에 추가합니다.
- **음악 라이브러리에 추가:** 선택한 파일이나 폴더를 음악 라이브러리에 포함합니다.
- **플레이리스트에 추가:** 파일이나 폴더를 플레이리스트에 추가합니다. 폴더에서 플레이리스트를 만들면 플레이리스트 이름이 자동으로 할당됩니다.
- **즐겨찾기에 추가:** 선택한 파일이나 폴더를 즐겨찾기로 표시합니다.
- **오디오 태그 편집:** 선택한 파일의 오디오 태그를 변경하기 위해 내장 태그 편집기를 엽니다.
- **이름 바꾸기:** 선택한 파일이나 폴더의 이름을 수정합니다.
- **이동:** 선택한 파일이나 폴더를 기기의 다른 위치로 이동합니다.
- **복사:** 선택한 파일이나 폴더의 복사본을 만듭니다.
- **업로드:** 선택한 파일이나 폴더를 원격 스토리지에 업로드합니다.
- **Finder에서 표시:** Mac에서 가져온 파일의 경우 이 작업은 파일이 있는 Finder 디렉토리를 표시합니다.
- **다음에서 열기:** 파일을 다른 앱으로 내보냅니다.
- **삭제하기:** 기기에서 파일이나 폴더를 삭제합니다. 이 작업은 되돌릴 수 없으며 삭제된 파일을 복원할 수 없습니다.

## 오프라인 폴더

{{< cards cols="1">}}
  {{< card title="" subtitle="오프라인 폴더 추가 작업 메뉴" image="/docs/guide/evermusic/evermusic-guide-local-files/img/local-files-offline-folders.webp" >}}
{{< /cards >}}

오프라인 모드는 인터넷에 연결되지 않았을 때도 즐겨찾는 음악에 접근할 수 있게 해주는 편리한 기능입니다. 앨범, 아티스트, 플레이리스트, 장르, 또는 원격 폴더에 대해 오프라인 모드를 활성화하면 해당 컬렉션 내의 모든 파일이 자동으로 기기에 다운로드되어 오프라인 재생이 가능해집니다. 앱의 "오프라인 폴더" 섹션에서 이러한 파일에 편리하게 접근할 수 있습니다.

더욱 편리한 점은 오프라인 사용으로 표시한 폴더에 있는 원격 서버에 새 파일을 추가하면 이러한 새 추가 파일도 자동으로 기기에 다운로드된다는 것입니다. 이렇게 하면 항상 최신 음악을 오프라인으로 즐길 수 있습니다.

클라우드 서비스에서 오프라인 폴더를 수동으로 동기화해야 하는 경우: 앱의 오른쪽 상단 모서리에 있는 세 점을 탭하고 "동기화하다" 옵션을 선택하면 됩니다. 이렇게 하면 원할 때 클라우드 서비스의 최신 트랙으로 오프라인 컬렉션을 업데이트할 수 있습니다.

경험을 더 맞춤화하고 싶은 분들을 위해 응용 프로그램 설정에서 동기화 타임아웃 설정을 조정하여 선호에 맞출 수 있습니다. 이렇게 하면 오프라인 컬렉션이 언제 어떻게 업데이트되는지 완전히 제어할 수 있어 인터넷 연결 없이도 항상 즐겨찾는 곡을 들을 수 있습니다.

Evermusic 및 Flacbox에서 오프라인 음악을 재생하는 방법: 클라우드에서 로컬 파일로 다운로드 및 동기화에 대한 자세한 지침은 [여기](/docs/howto/play-offline-music-in-evermusic-flacbox-download-sync-from-cloud-to-local-files/)에서 확인할 수 있습니다.

## 파일 관리자 설정

로컬 파일 화면의 거의 모든 동작 — 네트워크 대역폭부터 다운로드 위치, 썸네일 캐시 방법까지 — 은 **설정 → 파일 관리자**에서 구성할 수 있습니다. 전송 속도를 세밀하게 조정하거나, 저장 공간을 절약하거나, 앱을 Wi-Fi 전용으로 제한하고 싶을 때마다 열어보세요.

{{< cards cols="1">}}
  {{< card title="" subtitle="파일 관리자 설정 화면" image="/docs/guide/evermusic/evermusic-guide-local-files/img/local-files-file-manager-settings.webp" >}}
{{< /cards >}}

화면에는 명확하게 레이블이 붙은 섹션으로 그룹화된 모든 옵션이 표시됩니다:

### 파일 전송 (네트워크 유형)

다운로드와 업로드가 네트워크를 사용하는 방법을 선택합니다:

- **Wi-Fi:** Wi-Fi 네트워크에 연결된 경우에만 전송이 실행됩니다. 종량제 셀룰러 요금제를 사용하거나 예측 가능한 모바일 데이터 사용을 원하는 경우 사용합니다.
- **Wi-Fi & 셀룰러 데이터:** Wi-Fi를 사용할 수 있으면 Wi-Fi로, 그렇지 않으면 셀룰러 연결로 전송이 실행됩니다. 모바일 데이터 요금제를 사용할 수 있습니다.

### 병렬 파일 전송 제한

동시에 전송할 수 있는 최대 파일 수를 설정합니다. 기본값은 균형 잡힌 값입니다. 큰 폴더의 다운로드 속도를 높이려면 높이고, 약한 네트워크나 오래된 기기에서 전송을 안정적으로 유지하려면 낮추세요. 높은 값은 더 많은 대역폭, CPU, 배터리를 사용합니다.

### 파일 전송 작업

진행 중인 모든 업로드/다운로드 작업을 진행 표시줄, 속도, 상태와 함께 표시하는 실시간 전송 대기열을 엽니다. 여기에서 개별 작업을 일시 중지, 재개, 취소할 수 있습니다. 동일한 대기열은 로컬 파일 화면의 왼쪽 상단 모서리에 있는 회전 화살표 아이콘에서도 접근할 수 있습니다.

### 백그라운드 전송

활성화되면 Evermusic를 백그라운드로 보내거나 기기를 잠글 때도 다운로드가 계속됩니다. 느린 네트워크에서 대용량 라이브러리에 유용합니다. 참고: 작업이 상당한 양의 에너지를 소비하는 경우 iOS가 배터리 수명을 보호하기 위해 앱을 일시 중단할 수 있습니다.

### 다운로드 파일 저장 위치

클라우드 다운로드의 기본 대상을 정의합니다:

- **다운로드:** *(기본값)* 모든 다운로드는 **로컬 파일 → 다운로드** 폴더에 저장됩니다.
- **항상 묻기:** Evermusic가 모든 다운로드에 대해 대상 폴더를 선택하도록 요청합니다. 기기에서 아티스트 또는 장르 폴더별로 라이브러리를 정리하는 경우 유용합니다.
- **사용자 정의 폴더:** 한 번 특정 폴더를 선택하면 설정을 변경할 때까지 모든 다운로드가 그곳에 저장됩니다.

### 동기화된 오프라인 폴더

"오프라인 사용 가능"으로 표시한 모든 폴더를 표시하고 관리할 수 있습니다. 폴더를 탭하여 수동으로 동기화를 트리거하거나, 위치를 표시하거나, **오프라인 모드 비활성화**를 탭하여 감시를 중지하고 로컬 복사본을 제거합니다. 오프라인 모드에 대한 자세한 내용은 [여기](/docs/howto/play-offline-music-in-evermusic-flacbox-download-sync-from-cloud-to-local-files/)에서 확인할 수 있습니다.

### 시간 간격 (오프라인 동기화)

앱이 오프라인 폴더의 변경 사항을 확인하는 빈도를 선택합니다:

- **즉시:** 앱을 열 때마다 오프라인 폴더가 동기화됩니다.
- **15 / 30 / 60분:** 앱이 활성화된 동안 주기적으로 확인합니다.
- **수동:** 폴더에서 **동기화하다** 작업을 탭할 때만 동기화합니다.

### 전체 파일 이름 표시

활성화되면 파일 관리자는 확장자를 포함한 전체 파일 이름을 표시합니다(예: `Song Title - Artist.flac`). 확장자를 숨기고 더 짧고 깔끔한 이름을 표시하려면 비활성화합니다. 혼합 오디오 형식이나 비오디오 파일을 작업할 때 유용합니다.

### 온라인 파일 편집

연결된 클라우드 계정에 대해 전체 앱을 읽기-쓰기 모드와 읽기 전용 모드 사이에서 전환합니다. **읽기 전용 모드**가 활성화되면 모든 파괴적인 작업(이름 바꾸기, 이동, 삭제, 업로드, 오디오 태그 편집)이 온라인 파일에 대한 사용자 인터페이스에서 숨겨집니다 — 공유 가족 라이브러리나 업무용 클라우드 드라이브에서 실수로 데이터를 손실할 걱정이 있을 때 좋은 안전망입니다. 이 설정에 관계없이 로컬 파일은 완전히 편집 가능합니다.

### 열기 시 파일 복사

다른 앱에서 오디오 파일을 열 때 어떻게 처리할지 제어합니다(예: 파일 앱, 메일 첨부 파일, AirDrop, 또는 공유 확장을 통해):

- **항상 묻기:** Evermusic가 파일을 앱의 문서 폴더로 복사할지 아니면 제자리에서 열지 매번 묻습니다.
- **항상 복사:** 파일은 로컬 파일 → 다운로드 섹션으로 가져와 영구적인 로컬 복사본이 됩니다.
- **제자리에서 열기:** 파일은 원래 위치에 그대로 있고 Evermusic는 참조로 접근합니다. 소스가 제거되면 링크가 사라집니다.

### 파일 썸네일

파일 옆에 표시되는 캐시된 썸네일(앨범 아트워크, 오디오 파형, 이미지 미리보기)을 관리합니다:

- **온라인 파일에 대한 썸네일 로드:** 클라우드의 파일에 대해 썸네일을 가져올지 여부를 전환합니다. 느린 연결에서 데이터 사용량을 줄이려면 비활성화합니다.
- **네트워크 유형 (썸네일):** Wi-Fi로 썸네일 로드를 제한하거나 셀룰러 데이터를 허용합니다.
- **폴더에 표시:** 저장된 내용을 검사할 수 있도록 썸네일 캐시 폴더를 표시합니다.
- **모두 삭제:** 저장 공간을 확보하기 위해 캐시된 모든 썸네일을 지웁니다. 앱은 폴더를 다시 열 때 썸네일을 재생성합니다.

### 임시 파일 삭제

응용 프로그램의 캐시 폴더(오디오 버퍼, 반쯤 완료된 다운로드, 이미지 미리보기, 디코딩된 아트워크)를 비웁니다. 버튼은 현재 캐시 크기를 표시하여 얼마나 확보될지 알 수 있습니다. 언제든지 실행해도 안전합니다 — Evermusic는 다음 재생 시 필요한 것을 재생성합니다.

## 개인화

응용 프로그램 설정을 열고 개인화로 이동하여 로컬 파일 화면 스타일을 구성할 수 있습니다.  
사용 가능한 옵션은 다음과 같습니다:

- **일반 메뉴:** 앱의 문서 디렉토리에 있는 폴더를 표시합니다.
- **그룹화된 메뉴:** 카테고리별로 그룹화된 화면 콘텐츠를 표시합니다.

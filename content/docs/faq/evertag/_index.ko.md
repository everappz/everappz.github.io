Evertag는 **iPhone, iPad, Mac** 전용 음악 메타데이터 편집기로, 오디오 태그를 수정하고, 앨범 커버를 교체하고, 가사를 편집하고, 라이브러리를 정리할 수 있습니다 — 로컬, iCloud Drive, Google Drive / Dropbox / OneDrive / MEGA / pCloud, 또는 SMB / WebDAV를 통한 개인 NAS에서 사용 가능합니다. 이 FAQ는 사용자들이 가장 자주 보내는 질문에 답합니다. 더 심층적인 안내가 필요하신가요? [사용자 가이드](/docs/guide/evertag/) 또는 [태그 필드 매핑](/docs/guide/evertag/evertag-tag-field-mappings/) 참조로 시작하세요.

{{% details title="Evertag란 무엇인가요?" closed="true" %}}
Evertag는 **iOS 및 macOS** 전용 음악 메타데이터 편집기 및 앨범 아트워크 관리자로, 태그를 수정하고, 커버 아트를 추가하고, 기기 또는 클라우드에 저장된 오디오 파일을 정리할 수 있습니다.<br><br>

이 앱은 MP3, FLAC, WAV, M4A, AIFF, OGG, OPUS, WMA, APE 등 다양한 인기 오디오 형식을 지원하며, **TITLE**, **ARTIST**, **ALBUM**, **GENRE**, **YEAR**, **TRACK NUMBER**와 같은 일반 태그와 함께 **BPM**, **DISC NUMBER**, **LYRICS**, **MUSICBRAINZ IDs**, **REPLAY-GAIN** 값, **PARENTAL ADVISORY** 등급과 같은 확장 필드도 편집할 수 있습니다. 한 번에 하나의 파일로 작업하거나 **일괄 모드**로 전환하여 여러 트랙을 동시에 편집할 수 있어 전체 앨범이나 재생 목록을 정리하는 데 완벽합니다.<br><br>

Evertag의 두드러진 기능 중 하나는 인터넷(MusicBrainz를 통해)에서 누락된 앨범 커버를 가져오거나 사진 라이브러리에서 직접 추가할 수 있다는 것입니다. 내장된 가사 편집기는 일반 가사와 **동기화(LRC)** 가사를 모두 지원하며, Lrclib, Genius, Lyricsify, Google 검색으로의 원탭 바로가기를 포함합니다. 이 앱은 **제자리 편집**을 지원하므로 파일을 복사하지 않고도 오디오 태그를 수정할 수 있습니다.<br><br>

기기나 클라우드(iCloud Drive, Google Drive, Dropbox, OneDrive, MEGA, Synology, pCloud 등)에서 음악을 관리하든, Evertag는 원활한 파일 접근 및 편집을 제공합니다 — 데스크탑 태그 편집기 없이 iPhone, iPad, Mac에서 깔끔하고 잘 정리된 라이브러리를 원하는 음악인, DJ, 팟캐스터, 수집가에게 이상적입니다. 스크린샷과 튜토리얼은 전체 [사용자 가이드](/docs/guide/evertag/)를 참조하세요.<br><br>
{{% /details %}}

{{% details title="Evertag는 iPhone, iPad, Mac에서 작동하나요?" closed="true" %}}
네 — Evertag는 **범용 앱**입니다. 동일한 구매로 iPhone, iPad, Mac(Intel 및 Apple Silicon, Mac Catalyst를 통해)을 모두 커버하며, 설정, 즐겨찾기, Premium 상태가 iCloud를 통해 기기 간에 동기화됩니다.<br><br>

레이아웃이 기기에 맞게 조정됩니다: iPhone에서는 하단 탭 바, iPad와 Mac에서는 사이드바가 있는 분할 화면, Mac에서는 전체 키보드 단축키 지원. 태그 편집기의 모든 기능 — 일괄 모드, 가사 검색, 확장 필드, 클라우드 통합 — 이 모든 플랫폼에서 사용 가능합니다.<br><br>
{{% /details %}}

{{% details title="Evertag는 무료인가요?" closed="true" %}}
네, Evertag는 제한을 제거하기 위한 선택적 인앱 구매와 함께 **무료로 다운로드**할 수 있습니다. 이 앱은 **일회성 평생** 구매와 두 가지 구독 옵션(월간 및 연간)을 제공하여 가장 적합한 플랜을 선택할 수 있습니다. 가격은 국가 또는 지역에 따라 다를 수 있으며, **Family Sharing이 활성화**되어 있어 모든 플랜에서 최대 5명의 가족 구성원과 Premium을 공유할 수 있습니다.<br><br>

평생 구매 및 구독은 iCloud를 통해 iOS와 Mac 간에 공유됩니다. iOS에서 이미 결제하셨다면 최신 앱 버전을 설치하고 iCloud가 활성화되어 있는지 확인하고 구매가 업로드될 때까지 약 1분 기다린 후 Mac에서 앱을 시작하세요(동일한 Apple ID + iCloud). 그러면 Premium이 자동으로 활성화됩니다. 언제든지 **설정 → 구매 → 구매 복원**을 탭할 수도 있습니다.<br><br>
{{% /details %}}

{{% details title="Evertag Free와 Evertag Premium의 차이점은 무엇인가요?" closed="true" %}}

**Evertag Free**는 일부 제한이 있는 핵심 태그 편집 기능에 액세스할 수 있습니다. 광고를 포함하며 제한된 방식으로 태그 편집기, 앨범 커버 편집기, 일괄 모드를 사용할 수 있습니다: 연결된 클라우드 계정 1개, 즐겨찾기 최대 10개, 일일 자동 태그 검색 최대 20회, 일일 앨범 커버 조회 최대 20회. 120개 이상의 오디오 태그를 편집하고 내장 파일 관리자를 통해 로컬 또는 파일을 관리할 수 있습니다.<br><br>

**Evertag Premium(월간, 연간 또는 평생)**은 광고를 제거하고 모든 것을 잠금 해제합니다: 무제한 120개 이상 태그 필드, 무제한 일괄 편집, 인코딩 수정, 무제한 즐겨찾기, **무제한 클라우드 계정**, 무제한 자동 태그 검색, 무제한 앨범 커버 조회, 완전한 개인화(대체 앱 아이콘, 테마, 원본 크기 앨범 아트워크). Premium은 여러 기기와 클라우드 플랫폼에서 대규모 라이브러리를 관리하는 음악 수집가, DJ, 팟캐스터 또는 누구에게나 이상적입니다.<br><br>

**가격 옵션**<br>
• Premium 월간 — $1.99/월<br>
• Premium 연간 — $12.99/년<br>
• Premium 평생 — $24.99 (일회성 구매)<br><br>

모든 Premium 옵션은 동일한 기능 세트를 잠금 해제합니다. [전용 비교 페이지](/docs/faq/evertag/what-is-the-difference-between-evertag-and-evertag-premium/)에서 자세한 차이점을 확인하세요.<br><br>
{{% /details %}}

{{% details title="새 iPhone, iPad 또는 Mac에서 Premium 구매를 복원하려면 어떻게 하나요?" closed="true" %}}
원래 구매에 사용했던 **동일한 Apple ID**로 로그인하고 Evertag에서 **설정 → 구매 → 구매 복원**을 탭하세요.<br><br>

복원이 Premium 상태를 되돌리지 않는 경우:<br>
• 새 기기에서 iCloud가 활성화되어 있는지 확인하세요 — Evertag는 iCloud를 통해 구매 데이터를 동기화합니다.<br>
• 이미 Premium이 있는 기기에서 앱을 열고 구매 메타데이터가 업로드되도록 약 1분 기다리세요.<br>
• 새 기기에서 **설정 → 구매**의 **앱 시작 시 구매 복원**을 활성화하고 앱을 다시 시작하세요.<br>
• 기기에 활성 인터넷 연결이 있는지, **iOS 설정 → Apple ID → 미디어 및 구매**의 App Store 계정이 업그레이드를 구매한 계정과 일치하는지 확인하세요.<br><br>

구독 및 평생 구매는 iOS와 Mac(Mac Catalyst) 간에 공유되며, **Family Sharing**을 통해 최대 5명의 가족 구성원이 동일한 Premium 플랜을 재사용할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="Evertag는 안전한가요?" closed="true" %}}
네. Evertag는 계정에 액세스하기 위해 **공식 클라우드 제공업체 SDK 및 암호화된 연결**만 사용하며, 비밀번호는 앱과 공유되지 않습니다. 클라우드 서비스를 연결하면 클라우드 제공업체가 제공하는 공식 인증 페이지가 표시됩니다 — 전체 로그인 흐름이 Evertag 외부에서 이루어지며, 제공업체는 앱이 API 호출에 사용하는 인증 토큰을 반환합니다.<br><br>

인증 토큰은 안전한 시스템 **Keychain**에 기기에 저장되며, 절대 서버로 전송되지 않습니다. 연결된 클라우드 서비스에서 다운로드한 파일은 앱의 "문서" 디렉토리에 저장되며, 내장 파일 관리자에서 언제든지 제거할 수 있습니다.<br><br>

Evertag는 연결된 클라우드 계정의 정보를 공유하지 않습니다. 언제든지 액세스를 취소할 수 있습니다:<br>
• Evertag 내부 — 클라우드 계정을 열고 **…**를 탭한 후 **연결 끊기**를 선택하세요. 인증 토큰이 기기에서 제거됩니다.<br>
• 클라우드 제공업체 웹사이트 — 로그인하여 타사 앱/연결된 앱 페이지로 이동하고 목록에서 Evertag를 제거하세요.<br><br>

기기에서 Evertag를 삭제하면 모든 다운로드된 데이터와 액세스 토큰이 함께 제거됩니다.<br><br>
{{% /details %}}

{{% details title="Evertag는 개인 데이터를 수집하거나 공유하나요?" closed="true" %}}
앱은 서버에 음악 라이브러리에 관한 어떤 것도 저장하지 않습니다; 편집하는 모든 것은 기기나 자신의 클라우드 계정에 남아 있습니다.<br><br>

Evertag는 분석 및 충돌 보고를 위해 최소한의 서비스(Firebase Analytics 및 Crashlytics)를 사용하며, 무료 버전에서는 AdMob을 통해 광고를 표시합니다. **설정 → 분석 및 데이터 수집**에서 이러한 서비스를 검토하고 끌 수 있습니다. Apple의 앱 추적 투명성 프롬프트를 통해 앱이 처음 시작될 때 추적을 거부할 수도 있습니다. 전체 내용은 [개인정보 처리방침](/legal/privacy-policy/)을 참조하세요.<br><br>
{{% /details %}}

{{% details title="비밀번호 또는 Face ID로 Evertag를 보호할 수 있나요?" closed="true" %}}
네. **설정 → 비밀번호**를 열고 4자리 또는 6자리 코드를 설정하세요. 활성화되면 앱이 포그라운드로 올 때마다 Evertag가 비밀번호(또는 선택한 경우 Face ID / Touch ID)를 요청합니다.<br><br>

비밀번호는 iOS Keychain에 로컬로 저장됩니다 — 온라인 복구가 없습니다. 더 이상 보호가 필요하지 않으면 동일한 설정 화면에서 비활성화하세요.<br><br>
{{% /details %}}

{{% details title="Evertag는 어떤 오디오 형식을 지원하나요?" closed="true" %}}
Evertag는 30개 이상의 오디오 형식에서 태그를 읽고 씁니다:<br><br>

**MP3, OGG, OGA, FLAC, MPC, WV, SPX, OPUS, TTA, M4A, M4R, M4B, M4P, MP4, 3G2, M4V, WMA, ASF, AIF, AIFF, AFC, AIFC, WAV, APE, MOD, MODULE, NTS, WOW, S3M, IT, XM.**<br><br>

이는 가장 일반적인 무손실 형식(FLAC, WAV, AIFF, M4A 내 ALAC, APE)부터 손실 형식(MP3, M4A 내 AAC, OGG Vorbis, OPUS, WMA)과 클래식 트래커 형식(MOD, S3M, IT, XM)까지 모든 것을 포괄합니다. 필요한 형식이 없다면 **설정 → 피드백 보내기**를 통해 알려주세요.<br><br>
{{% /details %}}

각 형식에서 사용되는 정확한 필드 이름은 [태그 필드 매핑](/docs/guide/evertag/evertag-tag-field-mappings/) 참조를 확인하세요.<br><br>

{{% details title="Evertag가 ID3v2.3 또는 ID3v2.4 형식으로 태그를 쓸 수 있나요?" closed="true" %}}
네. **설정 → 오디오 태그 편집기 → 태그 저장 옵션**을 열고 **ID3v2.4**를 켜거나 끄세요.<br><br>

• **ID3v2.4 켜기** (기본값) — 완전한 UTF-8 지원, 다중 값 프레임, 풍부한 메타데이터가 있는 최신 표준. 현재 데스크탑 플레이어, 음악 관리자, 대부분의 최신 차량용 카 스테레오에 이상적입니다.<br>
• **ID3v2.4 끄기** — 더 널리 지원되는 **ID3v2.3** 형식으로 태그를 저장합니다. 오래된 기기, 차량용 헤드 유닛, 또는 DJ 소프트웨어가 편집 후 빈 태그를 표시하는 경우 사용하세요.<br><br>

동일한 설정 페이지에는 **중복 태그** 옵션이 있어 레거시 플레이어와의 최대 호환성을 위해 파일의 두 ID3 섹션 모두에 공통 메타데이터를 씁니다.<br><br>
{{% /details %}}

{{% details title="Evertag가 오디오 태그의 깨진 문자(키릴, 중국어, 일본어 등)를 수정할 수 있나요?" closed="true" %}}
네. Evertag에는 잘못 인코딩된 텍스트(예: `Ðàñêîëüíèêîâ`와 같이 알아볼 수 없이 표시되는 키릴, 중국어, 일본어, 한국어 태그)를 읽기 가능한 UTF-8로 다시 변환하는 내장 **인코딩 정규화 도구**가 있습니다.<br><br>

사용 방법:<br>
1. 태그 편집기에서 파일을 여세요.<br>
2. 아트워크 아래의 **인코딩 정규화** 버튼을 탭하세요.<br>
3. 파일을 저장하세요.<br><br>

버튼이 보이지 않으면 **설정 → 오디오 태그 편집기 → 메인 화면의 버튼**에서 활성화하세요. 인코딩 수정은 태그 리더가 UTF-8 대신 ANSI / CP-1251 / Shift-JIS를 기본값으로 사용하는 Windows에서 원래 태그가 지정된 라이브러리에 특히 유용합니다.<br><br>
{{% /details %}}

{{% details title="어떤 클라우드 서비스가 지원되나요?" closed="true" %}}
Evertag는 다양한 인기 클라우드 서비스에 연결하여 원격 파일에서 직접 태그를 편집할 수 있습니다:<br><br>

**iCloud Drive, Google Drive, Dropbox, OneDrive, Box, MEGA, Yandex.Disk, pCloud, Synology Drive, MediaFire, WD My Cloud Home, InfiniCLOUD (TeraCLOUD), HiDrive, OpenDrive, MyDrive, Put.io, Cloud Mail.ru, Baidu Pan (百度网盘)**.<br><br>

**SMB** 또는 **WebDAV**를 사용하여 개인 NAS, Apple Time Capsule 또는 컴퓨터에 연결할 수도 있습니다 — 단계별 설정은 [연결 가이드](/docs/guide/evertag/evertag-guide-connections/)를 참조하세요. 무료 버전에서는 클라우드 계정 하나를 추가할 수 있으며, **Premium**을 사용하면 필요한 만큼 많은 계정을 연결할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="새 클라우드 계정을 추가하려면 어떻게 하나요?" closed="true" %}}
**연결하기** 탭을 열고 → **클라우드 스토리지에 연결**을 탭 → 목록에서 제공업체를 선택(iCloud Drive, Google Drive, Dropbox, OneDrive, Box, MEGA, Yandex.Disk, pCloud, Synology Drive, MediaFire, WD My Cloud Home, InfiniCLOUD, HiDrive, OpenDrive, MyDrive, Put.io, Cloud Mail.ru, 또는 Baidu Pan) → 제공업체의 공식 페이지에서 로그인 → **완료됨**을 탭하세요.<br><br>

연결이 실패하면 인터넷 연결과 자격 증명을 다시 확인하고, 해당 서비스에 대한 이중 인증이 올바르게 설정되어 있는지 확인하세요. **Premium** 버전에서는 무제한 수의 계정을 추가하고 깊이 중첩된 폴더를 연결 화면 상단의 **빠른 액세스** 목록에 고정하여 원탭 탐색할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="전체 라이브러리를 다운로드하지 않고 Google Drive, Dropbox, iCloud Drive에서 오디오 태그를 편집할 수 있나요?" closed="true" %}}
네. 클라우드 계정을 연결한 후, Evertag는 편집하려는 파일만 스트리밍합니다 — 전체 라이브러리를 로컬에 미러링할 필요가 없습니다.<br><br>

원격 파일을 탭하면:<br>
1. Evertag가 임시 폴더에 다운로드합니다.<br>
2. 평소처럼 태그를 편집합니다.<br>
3. **저장하다**를 탭하면 업데이트된 파일이 클라우드의 동일한 위치에 다시 업로드됩니다.<br>
4. 임시 복사본이 제거됩니다(또는 **설정 → 오디오 태그 편집기 → 온라인 파일 편집**에 따라 보관됩니다).<br><br>

**설정 → 오디오 태그 편집기 → 온라인 파일 업데이트**에서 저장 동작을 제어할 수 있습니다: 각 저장 전에 확인 요청, 항상 자동으로 업로드, 또는 업로드 완전히 건너뛰기(로컬 전용 편집).<br><br>
{{% /details %}}

{{% details title="Synology, QNAP, Asustor 또는 Apple Time Capsule과 같은 NAS와 함께 Evertag를 사용할 수 있나요?" closed="true" %}}
네. **SMB** 또는 **WebDAV**를 사용하여 NAS를 연결하세요 — 모든 주요 NAS 벤더(Synology, QNAP, Asustor, TerraMaster, Western Digital, Buffalo, Apple Time Capsule)가 기본으로 지원하는 두 가지 프로토콜입니다.<br><br>

• **SMB** — **클라우드 스토리지에 연결 → SMB**를 탭하고, `smb://nas-ip-address/shared-folder-name`을 입력하고, 프로토콜 버전(자동 / SMB1 / SMB2)을 선택하고, 로그인 + 비밀번호를 입력하고, **완료됨**을 탭하세요. 전체 튜토리얼: [SMB를 통해 Mac 또는 PC에서 음악 스트리밍](/docs/howto/stream-your-music-from-mac-or-pc-to-iphone-using-smb/).<br>
• **WebDAV** — 동일한 흐름이지만 SSL의 경우 `http://server-name` 또는 `https://server-name`을 입력하세요. 전체 튜토리얼: [WebDAV를 통해 NAS 스토리지 연결](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac).<br>
• **Synology Drive** — 일급 통합으로 지원됩니다(일반 WebDAV와 별도). 전체 튜토리얼: [Synology NAS를 연결하고 iPhone 또는 Mac에서 음악 듣기](/docs/howto/how-to-connect-synology-nas-and-listen-to-music-on-your-iphone-or-mac/).<br><br>

로컬 네트워크에서 브로드캐스팅하는 기기도 연결 화면의 **사용 가능한 장치** 섹션에 자동으로 나타납니다.<br><br>
{{% /details %}}

{{% details title="편집 후 클라우드 서비스에서 파일 메타데이터를 자동으로 업데이트하려면 어떻게 하나요?" closed="true" %}}
클라우드 파일의 저장 동작은 **설정 → 오디오 태그 편집기 → 온라인 파일 업데이트**에서 세 가지 옵션으로 제어됩니다:<br><br>

• **확인 메시지 표시** *(기본값)* — 편집된 파일을 클라우드에 다시 업로드하기 전에 앱이 확인을 요청합니다.<br>
• **파일의 메타데이터 자동 업데이트** — 태그 편집기에서 **저장하다**를 탭하면 앱이 즉시 조용히 파일을 다시 업로드합니다.<br>
• **파일의 메타데이터를 업데이트하지 않음** — 앱이 업로드를 완전히 건너뜁니다; 변경 사항은 로컬에 유지되고 클라우드 복사본은 그대로입니다.<br><br>

각 편집 후 로컬에 다운로드된 복사본이 삭제되는지(공간 절약) 또는 보관되는지(빠른 재열기) 결정하려면 **설정 → 오디오 태그 편집기 → 온라인 파일 편집**과 함께 사용하세요.<br><br>
{{% /details %}}

{{% details title="클라우드 파일 편집 후 변경 사항이 나타나지 않는 이유는 무엇인가요?" closed="true" %}}
먼저 **온라인 파일 업데이트** 기본 설정이 **파일의 메타데이터를 업데이트하지 않음**으로 설정되어 있지 않은지 확인하세요 — 그 모드는 변경 사항을 로컬로 유지하고 클라우드에 다시 업로드하지 않습니다. **설정 → 오디오 태그 편집기 → 온라인 파일 업데이트**를 열고 **확인 메시지 표시** 또는 **파일의 메타데이터 자동 업데이트**를 선택하세요.<br><br>

확인할 다른 사항들:<br>
• 클라우드 폴더에 대한 **쓰기 권한**이 있는지 확인하세요. 일부 공유/읽기 전용 WebDAV / SMB 마운트는 업로드를 자동으로 수락하지만 삭제합니다.<br>
• 클라우드 제공업체가 조용히 로그아웃하지 않았는지 확인하세요 — 계정을 열고 **연결하기 → … → 설정**에서 다시 인증하세요.<br>
• 클라우드 서비스의 데스크탑 또는 웹 클라이언트가 여전히 이전 버전을 동기화하고 있을 수 있습니다. 잠시 기다리고 새로 고침하세요.<br>
• **iCloud Drive**를 사용하는 경우 다른 기기에서 파일이 여전히 업로드 중일 수 있습니다. iOS 파일 앱에서 파일을 열고 동기화 표시기가 더 이상 회전하지 않는지 확인하세요.<br><br>

여전히 해결이 안 되나요? 파일 형식, 클라우드 서비스, 간단한 설명과 함께 **설정 → 피드백 보내기**를 사용하세요.<br><br>
{{% /details %}}

{{% details title="클라우드 스토리지에서 파일을 다운로드하려면 어떻게 하나요?" closed="true" %}}
먼저 연결된 클라우드 계정이 하나 이상 있어야 합니다 — **연결하기** 화면을 열고 제공업체를 추가하세요.<br><br>

파일 또는 폴더를 다운로드하려면:<br>
1. 연결된 클라우드 스토리지 서비스를 여세요.<br>
2. 다운로드할 폴더로 이동하세요.<br>
3. 오른쪽 상단의 **추가 작업** (`…`) 버튼을 탭하고 **선택하다**를 선택하세요.<br>
4. 필요한 파일이나 폴더를 선택한 후 **다운로드**를 탭하세요.<br><br>

다운로드된 파일은 **로컬 파일 → 다운로드** 폴더에 저장되며 **전송 대기열**에 나타납니다 — 로컬 파일 화면 왼쪽 상단의 회전 화살표 아이콘에서 열 수 있습니다. **설정 → 파일 관리자 → 다운로드한 파일 저장 위치**에서 기본 대상을 변경하고 같은 화면에서 백그라운드 전송을 전환할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="네트워크 스토리지에서 파일을 관리하려면 어떻게 하나요?" closed="true" %}}
클라우드 폴더를 열고, 오른쪽 상단의 **…** 버튼을 탭하고 **선택하다**를 선택하여 선택 모드로 진입하세요. 각 파일과 폴더 옆에 체크박스가 나타나 하나 또는 여러 항목을 선택할 수 있습니다.<br><br>

선택 항목에 사용 가능한 동작:<br>
• **오디오 태그 편집** — [태그 편집기](/docs/guide/evertag/evertag-guide-tag-editor/)에서 하나 또는 여러 파일을 엽니다(다중 선택의 경우 일괄 모드).<br>
• **다운로드** — 파일이나 폴더를 **로컬 파일** 섹션에 복사합니다.<br>
• **이동** — 동일한 클라우드 계정 내의 다른 폴더로 항목을 이동합니다.<br>
• **이름 변경** — 원격 서버에서 파일이나 폴더 이름을 변경합니다.<br>
• **삭제하기** — 클라우드에서 파일이나 폴더를 영구적으로 제거합니다(되돌릴 수 없음).<br>
• **정렬** — 이름, 크기 또는 편집 날짜순으로 재정렬합니다.<br>
• **그리드 / 목록** — 테이블 보기와 썸네일 보기 사이에서 전환합니다.<br><br>

모든 동작에 공간이 부족하면 도구 모음에 **추가 작업** 오버플로우 버튼이 표시됩니다 — 전체 목록을 보려면 탭하세요.<br><br>
{{% /details %}}

{{% details title="iPhone 또는 iPad에 로컬로 저장된 음악 파일의 메타데이터를 편집하려면 어떻게 하나요?" closed="true" %}}
**로컬 파일**을 열고 **이 iPhone의 파일**(또는 iPad / Mac) 섹션으로 스크롤하세요.<br><br>

• **파일 열기…**를 선택하여 기기 어디서나 하나 이상의 오디오 파일을 선택하거나, **폴더 열기…**를 선택하여 전체 폴더를 선택하세요.<br>
• **열기**로 확인하세요 — 파일이 즉시 편집기에 추가되고, **제자리에서 편집**되며, Evertag에 복사되지 않습니다.<br><br>

**기기 폴더에 빠른 액세스.** 매번 선택기를 건너뛰려면 섹션에 폴더를 추가하세요:<br>
1. **로컬 파일**을 여세요.<br>
2. **이 기기의 파일**로 스크롤하고 **폴더 연결**을 탭하세요.<br>
3. 음악 폴더를 선택하고 **열기**를 탭하세요.<br><br>

폴더가 읽기/쓰기 액세스로 **이 iPhone의 파일** 목록에 나타납니다 — 시스템 파일 앱 또는 다른 앱의 노출된 폴더에 저장된 음악 라이브러리에 이상적입니다. 나중에 연결을 해제하려면 폴더 옆의 **…**를 탭하고 **연결 끊기**를 선택하세요. 자세한 튜토리얼: [iPhone 또는 Mac에 저장된 로컬 음악 재생](/docs/howto/how-to-play-local-music-stored-on-your-iphone-or-mac).<br><br>
{{% /details %}}

{{% details title="컴퓨터에서 Evertag로 음악을 전송하려면 어떻게 하나요?" closed="true" %}}
**SMB**, **WebDAV**, **Wi-Fi Drive**, 또는 **Finder 파일 공유**(구 macOS의 iTunes 파일 공유)를 사용하여 컴퓨터나 NAS를 연결할 수 있습니다.<br><br>

**SMB** — **클라우드 스토리지에 연결 → SMB**를 탭하고, `smb://computer-ip-address/shared-folder-name`을 입력하고, 자격 증명을 설정하고, **완료됨**을 탭하세요. 전체 튜토리얼: [SMB 프로토콜을 사용하여 컴퓨터에서 iPhone으로 파일 전송](/docs/howto/transfer-your-files-from-the-computer-to-iphone-using-smb-protocol/).<br><br>

**WebDAV** — SSL의 경우 `http://server-name`(또는 `https://`)으로 동일한 흐름. 전체 튜토리얼: [WebDAV를 통해 NAS 스토리지 연결](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac).<br><br>

**Wi-Fi Drive** — 완전 무선. **연결하기 → 컴퓨터 → Wi-Fi를 통해 연결**을 열고 서버를 시작한 후 컴퓨터 브라우저에 표시된 URL을 열고 페이지에 파일을 **드래그 앤 드롭**하세요. 두 기기 모두 동일한 Wi-Fi 네트워크를 공유해야 합니다. 전체 튜토리얼: [Wi-Fi Drive를 사용하여 무선으로 파일 전송](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive).<br><br>

**Finder 파일 공유**(또는 구 macOS / Windows의 iTunes 파일 공유) — 케이블로 iPhone/iPad를 컴퓨터에 연결하고, **Finder → 기기 → 파일 → Evertag**를 열고, 공유 폴더에 파일을 드래그하세요. 전체 튜토리얼: [iPhone에서 로컬 iTunes 파일 재생](/docs/howto/how-to-play-local-itunes-files-on-my-iphone).<br><br>
{{% /details %}}

{{% details title="인터넷 연결 없이 오프라인으로 오디오 태그를 편집할 수 있나요?" closed="true" %}}
네. 로컬 파일 편집은 완전히 오프라인으로 작동합니다 — 태그 편집기, 일괄 모드, 앨범 커버 교체, 인코딩 수정, 가사 편집기는 인터넷 연결이 필요하지 않습니다.<br><br>

인터넷이 필요한 기능은 명확히 표시됩니다: MusicBrainz 자동 검색, 수동 MusicBrainz 태그 검색, 앨범 아트워크 웹 검색, 가사 검색 바로가기(Lrclib, Genius, Lyricsify, Google). 세션 중에 오프라인이 되면 해당 서비스가 단순히 비활성화됩니다 — 로컬 편집은 영향을 받지 않습니다.<br><br>
{{% /details %}}

{{% details title="여러 파일을 동시에 편집하려면 어떻게 하나요(일괄 모드)?" closed="true" %}}
**일괄 모드**를 사용하여 동시에 여러 트랙에 동일한 메타데이터 변경을 적용하세요. 시작하는 두 가지 방법이 있습니다:<br><br>

**파일 관리자에서**<br>
1. 파일이 포함된 폴더를 여세요.<br>
2. **추가 작업** (`…`) 버튼을 탭 → **선택하다**.<br>
3. 편집할 파일을 선택하세요.<br>
4. **오디오 태그 편집**을 탭하세요.<br><br>

**태그 편집기에서**<br>
1. 폴더에서 파일을 여세요.<br>
2. 태그 편집기 하단으로 스크롤하세요.<br>
3. **파일 동시 편집**을 탭하세요 — 동일한 폴더의 모든 오디오 파일이 함께 로드됩니다.<br><br>

변경 사항을 한 번 만들고 **저장하다**를 탭하세요 — 새 값이 선택된 모든 파일에 기록됩니다. 전체 앨범에 걸쳐 앨범 이름, 앨범 아티스트, 장르, 연도, 트랙 총수를 수정하는 데 이상적입니다.<br><br>
{{% /details %}}

{{% details title="여러 노래의 앨범 커버를 한 번에 변경하려면 어떻게 하나요?" closed="true" %}}
**일괄 모드**를 활성화하고(위 참조), 아트워크 캐러셀의 **카메라 아이콘**을 탭하고, 새 이미지를 선택하세요 — 동일한 아트워크가 한 번 저장으로 선택된 모든 파일에 기록됩니다.<br><br>

새 커버를 가져올 수 있는 곳:<br>
• **사진 라이브러리** — 이미 저장하거나 스캔한 커버에 유용합니다.<br>
• **로컬 파일 / 다운로드** — 기기에 이미 있는 이미지.<br>
• **연결된 클라우드 계정** — 클라우드의 JPG/PNG를 Evertag로 가리키세요.<br><br>

**설정 → 오디오 태그 편집기 → 앨범 커버 크기 조정**에서 저장된 이미지 품질을 제어하세요 — 소 / 중 / 대 / 초대, 또는 원본 크기를 유지하려면 "비활성화됨"(Premium).<br><br>
{{% /details %}}

{{% details title="누락된 앨범 아트워크를 자동으로 찾으려면 어떻게 하나요?" closed="true" %}}
태그 편집기의 아트워크 아래에 있는 **앨범 커버 검색** 버튼을 탭하세요. Evertag는 트랙의 현재 아티스트 및 앨범 메타데이터를 사용하여 온라인에서 일치하는 커버 아트를 조회합니다.<br><br>

검색 결과가 나타나면 마음에 드는 이미지를 길게 누르고 시스템 메뉴에서 **사진에 저장**을 선택하세요. Evertag로 돌아가 아트워크의 **카메라 아이콘**을 탭하고 **사진 라이브러리**를 선택하여 저장된 이미지를 선택하세요 — 커버가 파일에 적용됩니다. **앨범 아트워크 저장** 버튼(동일한 메뉴 아래)을 사용하면 현재 아트워크를 나중에 재사용하기 위해 Documents 폴더에 백업할 수도 있습니다.<br><br>

**앨범 커버 검색** 버튼이 보이지 않으면 **설정 → 오디오 태그 편집기 → 메인 화면의 버튼**에서 활성화하세요.<br><br>
{{% /details %}}

{{% details title="Evertag는 MusicBrainz를 사용하나요?" closed="true" %}}
네. Evertag의 자동 및 수동 태그 검색은 5천만 개 이상의 트랙을 보유한 가장 큰 오픈 음악 메타데이터 프로젝트 중 하나인 **MusicBrainz** 데이터베이스를 기반으로 합니다.<br><br>

태그 편집기 하단 도구 모음에서 두 가지 흐름을 사용할 수 있습니다:<br>
• **오디오 태그 자동 검색** — 기존 아티스트 / 앨범 / 타이틀 메타데이터를 사용한 최적 매치 조회.<br>
• **오디오 태그 수동 검색** — 앨범, 노래, 적용할 정확한 필드 세트를 선택하세요.<br><br>

MusicBrainz 식별자를 직접 저장할 수도 있습니다: Album ID, Album Artist ID, Artist ID, Disc ID, Release Group ID, Release Track ID, Track ID, Work ID 등([태그 필드 매핑](/docs/guide/evertag/evertag-tag-field-mappings/) 참조). 이러한 ID를 통해 다른 MusicBrainz 인식 앱(Picard, MusicBrainz Browser, Roon, Plex)이 트랙을 안정적으로 교차 참조할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="Evertag에서 노래에 가사를 추가하려면 어떻게 하나요?" closed="true" %}}
가사는 **확장 태그 편집기**에 있습니다. 빠른 경로는 다음과 같습니다:<br><br>

1. 파일을 탭하여 태그 편집기를 여세요.<br>
2. **확장 태그 표시**를 탭하여 전체 필드 세트를 활성화하세요.<br>
3. **동기화되지 않은 가사** 섹션으로 스크롤하고 필드를 탭하세요.<br>
4. **새 페이지 추가**를 탭하세요 — 각 가사 페이지에는 자체 **언어**와 **설명**이 있어 단일 트랙이 여러 번역을 보유할 수 있습니다.<br>
5. **텍스트** 필드에 일반 가사 또는 **LRC 형식(타임스탬프) 가사**를 붙여넣으세요 — 자리 표시자에 예시가 표시됩니다.<br>
6. 각 화면에서 **완료됨**을 탭한 후 **저장하다**를 탭하여 오디오 파일에 가사를 씁니다.<br><br>

**직접 입력하고 싶지 않으신가요?** 가사 편집기에는 가장 인기 있는 가사 데이터베이스로의 원탭 **검색 바로가기**가 있어 현재 트랙의 아티스트와 타이틀이 미리 채워져 있습니다:<br>
• **Lrclib** — **타이밍(LRC) 가사**의 최고 소스.<br>
• **Genius** — 정확한 일반 텍스트 가사의 대규모 카탈로그.<br>
• **Lyricsify** — 일반 및 동기화 가사가 있는 커뮤니티 데이터베이스.<br>
• **Google** — 대체 수단으로 사용하는 일반 웹 검색.<br><br>

각 버튼은 해당 서비스에 연결할 수 있을 때만 나타납니다. 전체 안내: [iPhone 또는 Mac에서 오디오 파일의 가사 편집하는 방법](/docs/howto/how-to-edit-lyrics-for-audio-files-on-iphone-mac/).<br><br>

**동기화(LRC) 가사.** 편집기는 일반 텍스트와 LRC 형식 가사(`[mm:ss.ms]` 줄 타임스탬프) 모두를 수락합니다 — LRC 텍스트를 필드에 붙여넣고 저장하면 오디오 파일의 가사 태그가 그대로 저장되어 모든 LRC 인식 플레이어에서 사용할 수 있습니다. 내장된 **Lrclib** 및 **Lyricsify** 바로가기는 기본적으로 LRC 문자열을 반환하므로 앱을 떠나지 않고도 완전히 동기화된 가사 라이브러리를 몇 초 만에 구축할 수 있습니다.<br><br>
{{% /details %}}

{{% details title="버그를 보고하거나 지원팀에 연락하려면 어떻게 하나요?" closed="true" %}}
**설정 → 피드백 보내기**를 열어 로그가 첨부된 지원팀 이메일을 보내세요. 파일 형식(예: FLAC), 클라우드 서비스(해당하는 경우), iOS / macOS 버전을 포함하면 진단이 훨씬 빠릅니다.<br><br>

셀프 서비스 답변은 [사용자 가이드](/docs/guide/evertag/), [방법 안내 문서](/docs/howto/), 또는 이 FAQ를 확인하세요. 문서에서 오타나 누락된 주제를 발견하시면 동일한 방법으로 알려주세요 — 가이드를 정기적으로 업데이트합니다.<br><br>
{{% /details %}}

{{% details title="Evertag를 제거하고 모든 데이터를 삭제하려면 어떻게 하나요?" closed="true" %}}
iOS, iPadOS 또는 macOS에서 앱을 삭제하면 Evertag가 생성한 **모든** 파일이 제거됩니다. 여기에는 다운로드된 노래, 캐시된 썸네일, 앨범 커버, 설정, 비밀번호, 연결된 클라우드 계정의 인증 토큰이 포함됩니다.<br><br>

앱을 유지하면서 제거 없이 공간을 확보하고 싶다면:<br>
• **설정 → 파일 관리자 → 파일 썸네일 → 모두 삭제** — 캐시된 썸네일을 삭제합니다.<br>
• **설정 → 파일 관리자 → 임시 파일 삭제** — 캐시 폴더를 지웁니다.<br>
• **로컬 파일 → 다운로드**를 열고 더 이상 필요하지 않은 개별 파일이나 폴더를 삭제하세요.<br>
• **연결하기** 화면에서 클라우드 계정을 연결 해제하여 인증 토큰을 제거하세요.<br><br>

Evertag를 제거해도 클라우드 측 파일은 절대 제거되지 않습니다 — 클라우드 제공업체의 서버에 남아 있으며 모든 웹 클라이언트에서 다시 액세스할 수 있습니다.<br><br>

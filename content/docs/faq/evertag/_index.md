Evertag is a music metadata editor for **iPhone, iPad, and Mac** that lets you fix audio tags, replace album covers, edit lyrics, and clean up your library — locally, in iCloud Drive, on Google Drive / Dropbox / OneDrive / MEGA / pCloud, or on a personal NAS over SMB / WebDAV. This FAQ answers the questions users send most often. Looking for a deeper walkthrough? {{% details title="What is Evertag?" closed="true" %}}
Evertag is a music metadata editor and album-artwork manager for **iOS and macOS** that lets you fix tags, add cover art, and clean up audio files stored on your device or in the cloud.<br><br>

The app supports a wide range of popular audio formats — MP3, FLAC, WAV, M4A, AIFF, OGG, OPUS, WMA, APE and many others — and lets you edit common tags such as title, artist, album, genre, year, track number, plus extended fields like BPM, disc number, lyrics, MusicBrainz IDs, replay-gain values, and parental advisory ratings. You can work with one file at a time or switch to **batch mode** to edit multiple tracks simultaneously — perfect for organizing full albums or playlists.<br><br>

One of the standout features of Evertag is its ability to fetch missing album covers from the internet (via MusicBrainz) or let you add your own from the Photos library. The built-in lyrics editor supports both plain and **synced (LRC)** lyrics and includes one-tap shortcuts to Lrclib, Genius, Lyricsify, and Google search. The app supports **open-in-place** editing, so you can modify audio tags without copying files around.<br><br>

Whether you're managing music on your device or in the cloud (iCloud Drive, Google Drive, Dropbox, OneDrive, MEGA, Synology, pCloud, and many more), Evertag offers seamless file access and editing — ideal for musicians, DJs, podcasters, and collectors who want a clean, well-organized library on iPhone, iPad, or Mac without using a desktop tag editor. See the full [user guide](/docs/guide/evertag/) for screenshots and tutorials.<br><br>

{{% details title="Is Evertag safe?" closed="true" %}}
Yes. Evertag uses only **official cloud-provider SDKs and encrypted connections** to access your accounts, and your password is never shared with the app. When you connect a cloud service, the official authorization page provided by the cloud provider is shown — the entire login flow happens outside Evertag, and the provider returns an auth-token that the app uses for API calls.<br><br>

The auth-token is stored on your device in the secure system **Keychain**, never sent to our servers. Files you download from a connected cloud service are placed in the app's "Documents" directory, and you can remove them anytime from the built-in file manager.<br><br>

Evertag does not share any information from your connected cloud account. You can revoke access at any time:<br>
• Inside Evertag — open the cloud account, tap **…** and choose **Disconnect**. The auth-token is removed from your device.<br>
• On the cloud provider's website — sign in, go to the third-party-apps / connected-apps page and remove Evertag from the list.<br><br>

If you delete Evertag from your device, all downloaded data and access tokens are removed with it.<br><br>

{{% details title="Can Evertag write tags in ID3v2.3 or ID3v2.4 format?" closed="true" %}}
Yes. Open **Settings → Audio tags editor → Tag saving options** and toggle **ID3v2.4** on or off.<br><br>

• **ID3v2.4 on** (default) — modern standard with full UTF-8 support, multi-value frames, and richer metadata. Ideal for current desktop players, music managers, and most modern car stereos.<br>
• **ID3v2.4 off** — saves tags in the more widely supported **ID3v2.3** format. Use this if an older device, car head unit, or DJ software shows blank tags after editing.<br><br>

The same settings page exposes the **Duplicate tags** option, which writes common metadata into both ID3 sections of a file for maximum compatibility with legacy players.<br><br>
{{% /details %}}

{{% details title="Can Evertag fix garbled or corrupted characters in audio tags (Cyrillic, Chinese, Japanese, etc.)?" closed="true" %}}
Yes. Evertag includes a built-in **encoding normalizer** that converts mis-encoded text (for example, Cyrillic, Chinese, Japanese, or Korean tags that show up as gibberish like `Ðàñêîëüíèêîâ`) back into readable UTF-8.<br><br>

To use it:<br>
1. Open a file in the tag editor.<br>
2. Tap the **Normalize encoding** button under the artwork.<br>
3. Save the file.<br><br>

If you don't see the button, enable it under **Settings → Audio tags editor → Buttons on the main screen**. Encoding fixes are especially useful for libraries originally tagged on Windows where tag readers default to ANSI / CP-1251 / Shift-JIS instead of UTF-8.<br><br>

{{% details title="Can I edit audio tags in Google Drive, Dropbox, or iCloud Drive without downloading the whole library?" closed="true" %}}
Yes. After you connect a cloud account, Evertag streams only the file you're about to edit — there's no need to mirror your entire library locally.<br><br>

When you tap a remote file:<br>
1. Evertag downloads it to a temporary folder.<br>
2. You edit tags as usual.<br>
3. On **Save**, the updated file is re-uploaded to the same location in the cloud.<br>
4. The temporary copy is removed (or kept, depending on **Settings → Audio tags editor → Edit online files**).<br><br>

You can control how saves behave under **Settings → Audio tags editor → Update online files**: ask before each save, always upload automatically, or skip uploading entirely (local-only edits).<br><br>
{{% /details %}}

{{% details title="Can I use Evertag with a NAS like Synology, QNAP, Asustor, or Apple Time Capsule?" closed="true" %}}
Yes. Connect any NAS using **SMB** or **WebDAV** — they're the two protocols every major NAS vendor (Synology, QNAP, Asustor, TerraMaster, Western Digital, Buffalo, Apple Time Capsule) supports out of the box.<br><br>

• **SMB** — tap **Connect to cloud storage → SMB**, enter `smb://nas-ip-address/shared-folder-name`, choose protocol version (Auto / SMB1 / SMB2), enter login + password, tap **Done**. Full tutorial: [Stream music from Mac or PC via SMB](/docs/howto/stream-your-music-from-mac-or-pc-to-iphone-using-smb/).<br>
• **WebDAV** — same flow, but enter `http://server-name` or `https://server-name` for SSL. Full tutorial: [Connect NAS storage via WebDAV](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac).<br>
• **Synology Drive** — supported as a first-class integration (separate from generic WebDAV).<br><br>

Devices broadcasting on the local network also appear automatically in the **Available devices** section of the Connections screen.<br><br>
{{% /details %}}

{{% details title="How do I automatically update file metadata on a cloud service after editing?" closed="true" %}}
Save behavior for cloud files is controlled in **Settings → Audio tags editor → Update online files** with three options:<br><br>

• **Show confirmation message** *(default)* — the app prompts you before uploading the edited file back to the cloud.<br>
• **Automatically update file's metadata** — the app silently re-uploads the file immediately after you tap **Save** in the tag editor.<br>
• **Do not update file's metadata** — the app skips the upload entirely; your changes stay local and the cloud copy is untouched.<br><br>

Pair this with **Settings → Audio tags editor → Edit online files** to decide whether the locally-downloaded copy is deleted (saves space) or kept (faster reopening) after each edit.<br><br>
{{% /details %}}

{{% details title="Why are my changes not appearing after editing a cloud file?" closed="true" %}}
First, check that your **Update online files** preference is not set to **Do not update file's metadata** — that mode keeps changes local and never re-uploads to the cloud. Open **Settings → Audio tags editor → Update online files** and choose **Show confirmation message** or **Automatically update file's metadata** instead.<br><br>

Other things to verify:<br>
• You have **write permission** for the cloud folder. Some shared / read-only WebDAV / SMB mounts will accept the upload silently but discard it.<br>
• The cloud provider hasn't quietly logged you out — open the account and re-authenticate from **Connections → … → Settings**.<br>
• The desktop or web client of the cloud service may still be syncing the old version. Wait a minute and refresh.<br>
• If you're using **iCloud Drive**, the file may still be uploading from another device. Open the file in the iOS Files app and confirm the sync indicator is no longer spinning.<br><br>

Still stuck? Use **Settings → Send Feedback** with the file format, cloud service, and a short description.<br><br>

{{% details title="How do I download files from cloud storage?" closed="true" %}}
You need at least one connected cloud account first — open the **Connections** screen and add a provider.<br><br>

To download files or folders:<br>
1. Open the connected cloud storage service.<br>
2. Navigate to the folder you want to download.<br>
3. Tap the **More actions** (`…`) button in the top right and choose **Select**.<br>
4. Check the files or folders you need, then tap **Download**.<br><br>

Downloads land in the **Local Files → Downloads** folder and appear in the **transfer queue** — open it from the spinning-arrows icon in the top left of the Local Files screen. You can change the default destination from **Settings → File manager → Save downloaded files to** and toggle background transfers from the same screen.<br><br>

{{% details title="How do I manage files in a network storage?" closed="true" %}}
Open a cloud folder, tap the **…** button in the top right and pick **Select** to enter selection mode. Checkboxes appear next to each file and folder so you can choose one or many items.<br><br>

Actions available on the selection:<br>
• **Edit audio tags** — open one or several files in the [tag editor](/docs/guide/evertag/evertag-guide-tag-editor/) (batch mode for multi-select).<br>
• **Download** — copy file(s) or folder to the **Local Files** section.<br>
• **Move** — relocate the item to another folder inside the same cloud account.<br>
• **Rename** — rename the file or folder on the remote server.<br>
• **Delete** — permanently remove the file or folder from the cloud (irreversible).<br>
• **Sort** — re-order by name, size, or date edited.<br>
• **Grid / List** — switch between table and thumbnail view.<br><br>

If there isn't room for every action, the toolbar shows a **More actions** overflow button — tap it to see the full list.<br><br>

{{% details title="How do I transfer music to Evertag from my computer?" closed="true" %}}

Features that **do** need internet are clearly marked: MusicBrainz auto-search, manual MusicBrainz tag search, the album-artwork web search, and the lyrics-search shortcuts (Lrclib, Genius, Lyricsify, Google). {{% details title="How do I edit several files at once (batch mode)?" closed="true" %}}
Use **batch mode** to apply the same metadata change to many tracks at once. There are two ways to start it:<br><br>

**From the file manager**<br>
1. Open the folder containing the files.<br>
2. Tap the **More actions** (`…`) button → **Select**.<br>
3. Check the files you want to edit.<br>
4. Tap **Edit audio tags**.<br><br>

**From the tag editor**<br>
1. Open any file from the folder.<br>
2. Scroll to the bottom of the tag editor.<br>
3. Tap **Edit files simultaneously** — all audio files in the same folder are loaded together.<br><br>

Make your changes once and tap **Save** — the new values are written to every selected file. Ideal for fixing album name, album artist, genre, year, or track totals across a whole album.<br><br>

{{% details title="How do I change the album cover on multiple songs at once?" closed="true" %}}
Activate **batch mode** (see above), tap the **camera icon** on the artwork carousel, and pick the new image — the same artwork is written into every selected file in one save.<br><br>

You can source the new cover from:<br>
• **Photos library** — useful for a cover you've already saved or scanned.<br>
• **Local Files / Downloads** — any image already on your device.<br>
• **A connected cloud account** — point Evertag at a JPG/PNG in your cloud.<br><br>

Control the saved image quality from **Settings → Audio tags editor → Album cover scaling** — Small / Medium / Large / Extra Large, or "Deactivated" (Premium) to keep the original size.<br><br>
{{% /details %}}

{{% details title="How do I find missing album artwork automatically?" closed="true" %}}
Tap the **Search album cover** button under the artwork in the tag editor. Evertag uses the track's current artist and album metadata to look up matching cover art online.<br><br>

When the search results appear, long-press the image you like and choose **Save to Photos** in the system menu. Return to Evertag, tap the **camera icon** on the artwork, pick **Photo Library**, and select the saved image — the cover is applied to the file. The Save album artwork button (under the same menu) also lets you back up the current artwork to the Documents folder for reuse later.<br><br>

If the Search album cover button isn't visible, enable it under **Settings → Audio tags editor → Buttons on the main screen**.<br><br>

{{% details title="Does Evertag use MusicBrainz?" closed="true" %}}
Yes. Evertag's automatic and manual tag search are powered by the **MusicBrainz** database — one of the largest open music metadata projects with more than 50 million tracks.<br><br>

Two flows are available from the bottom toolbar of the tag editor:<br>
• **Auto-search audio tags** — best-match lookup using existing artist / album / title metadata.<br>
• **Manual search audio tags** — pick the album, the song, and the exact set of fields to apply.<br><br>

You can also store MusicBrainz identifiers directly: Album ID, Album Artist ID, Artist ID, Disc ID, Release Group ID, Release Track ID, Track ID, Work ID, and more (see the [Tag Field Mappings](/docs/guide/evertag/evertag-tag-field-mappings/) reference). These IDs let other MusicBrainz-aware apps (Picard, MusicBrainz Browser, Roon, Plex) cross-reference your tracks reliably.<br><br>

{{% details title="How do I add lyrics to songs in Evertag?" closed="true" %}}
Lyrics live in the **extended tag editor**. Here's the quick path:<br><br>

1. Tap a file to open the tag editor.<br>
2. Tap **Show extended tags** to enable the full field set.<br>
3. Scroll to the **Lyrics unsynced** section and tap the field.<br>
4. Tap **Add new page** — each lyrics page has its own **Language** and **Description**, so a single track can hold multiple translations.<br>
5. In the **Text** field, paste plain lyrics or **LRC-formatted (timestamped) lyrics** — the placeholder shows an example.<br>
6. Tap **Done** on each screen, then **Save** to write the lyrics into the audio file.<br><br>

**Don't want to type them?** The lyrics editor has one-tap **search shortcuts** to the most popular lyrics databases — pre-filled with the current track's artist and title:<br>
• **Lrclib** — the go-to source for **timed (LRC) lyrics**.<br>
• **Genius** — large catalog with accurate plain-text lyrics.<br>
• **Lyricsify** — community database with plain and synced lyrics.<br>
• **Google** — generic web search as a fallback.<br><br>

Each button only appears when the corresponding service is reachable. Full walkthrough: [How to edit lyrics for audio files on iPhone or Mac](/docs/howto/how-to-edit-lyrics-for-audio-files-on-iphone-mac/).<br><br>

**Synced (LRC) lyrics.** The editor accepts plain text **and** LRC-formatted lyrics (`[mm:ss.ms]` line timestamps) — paste the LRC text into the field and save, and the audio file's lyrics tag will store it verbatim, ready for any LRC-aware player. The built-in **Lrclib** and **Lyricsify** shortcuts return LRC strings by default, so you can build a fully synced lyrics library in seconds without leaving the app.<br><br>

{{% details title="How do I uninstall Evertag and remove all my data?" closed="true" %}}
Deleting the app from iOS, iPadOS, or macOS removes **all** files Evertag created, including downloaded songs, cached thumbnails, album covers, settings, the passcode, and auth-tokens for connected cloud accounts.<br><br>

If you'd rather keep the app but free up space without uninstalling:<br>
• **Settings → File manager → Thumbnails for files → Delete all** — drops cached thumbnails.<br>
• **Settings → File manager → Delete temporary files** — clears the cache folder.<br>
• Open **Local Files → Downloads** and delete individual files or folders you no longer need.<br>
• Disconnect cloud accounts from the **Connections** screen to remove their auth-tokens.<br><br>

Cloud-side files are never removed by uninstalling Evertag — they remain on the cloud provider's servers and can be revisited from any web client.<br><br>

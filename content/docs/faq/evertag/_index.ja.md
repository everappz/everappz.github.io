Evertagは**iPhone、iPad、Mac**向けの音楽メタデータエディタで、オーディオタグの修正、アルバムカバーの交換、歌詞の編集、ライブラリの整理が可能です — ローカル、iCloud Drive、Google Drive / Dropbox / OneDrive / MEGA / pCloud、またはSMB / WebDAV経由の個人NASで利用できます。このFAQはユーザーから最もよく寄せられる質問に回答しています。より詳しいガイドをお探しですか？[ユーザーガイド](/docs/guide/evertag/)または[タグフィールドマッピング](/docs/guide/evertag/evertag-tag-field-mappings/)リファレンスからお始めください。

{{% details title="Evertagとは何ですか？" closed="true" %}}
Evertagは**iOSとmacOS**向けの音楽メタデータエディタおよびアルバムアートワークマネージャで、デバイスまたはクラウドに保存されたオーディオファイルのタグを修正し、カバーアートを追加し、整理することができます。<br><br>

このアプリは幅広い人気オーディオフォーマットをサポートしています — MP3、FLAC、WAV、M4A、AIFF、OGG、OPUS、WMA、APEなど多数 — そして**TITLE**、**ARTIST**、**ALBUM**、**GENRE**、**YEAR**、**TRACK NUMBER**などの一般的なタグ、さらに**BPM**、**DISC NUMBER**、**LYRICS**、**MUSICBRAINZ IDs**、**REPLAY-GAIN**値、**PARENTAL ADVISORY**レーティングなどの拡張フィールドを編集できます。1つのファイルずつ作業するか、**一括モード**に切り替えて複数のトラックを同時に編集できます — アルバムやプレイリスト全体の整理に最適です。<br><br>

Evertagの際立った機能の1つは、インターネットから不足しているアルバムカバーを取得する機能（MusicBrainz経由）、またはフォトライブラリから自分のものを追加する機能です。内蔵の歌詞エディタはプレーンと**同期（LRC）**歌詞の両方をサポートし、Lrclib、Genius、Lyricsify、Googleへのワンタップショートカットを含みます。このアプリは**オープンインプレイス**編集をサポートしているため、ファイルをコピーせずにオーディオタグを変更できます。<br><br>

デバイス上またはクラウド（iCloud Drive、Google Drive、Dropbox、OneDrive、MEGA、Synology、pCloudなど）で音楽を管理しているかどうかに関わらず、Evertagはシームレスなファイルアクセスと編集を提供します — デスクトップタグエディタを使わずにiPhone、iPad、またはMacで整理された清潔なライブラリを望む音楽家、DJ、ポッドキャスター、コレクターに最適です。スクリーンショットとチュートリアルは[ユーザーガイド](/docs/guide/evertag/)の完全版をご覧ください。<br><br>
{{% /details %}}

{{% details title="EvertagはiPhone、iPad、Macで動作しますか？" closed="true" %}}
はい — Evertagは**ユニバーサルアプリ**です。同じ購入でiPhone、iPad、Mac（IntelとApple Silicon両方、Mac Catalyst経由）をカバーし、設定、お気に入り、Premiumステータスはデバイス間でiCloud経由で同期されます。<br><br>

レイアウトはデバイスに合わせて調整されます：iPhoneではボトムタブバー、iPadとMacではサイドバー付きの分割ビュー、Macでは完全なキーボードショートカットサポート。タグエディタのすべての機能 — 一括モード、歌詞検索、拡張フィールド、クラウドインテグレーションを含む — すべてのプラットフォームで利用可能です。<br><br>
{{% /details %}}

{{% details title="Evertagは無料ですか？" closed="true" %}}
はい、Evertagは制限を解除するためのオプションのアプリ内購入付きで**無料でダウンロード**できます。このアプリは**一回限りの生涯**購入と2つのサブスクリプションオプション（月額と年額）を提供しているので、最適なプランを選ぶことができます。価格は国や地域によって異なる場合があり、**ファミリー共有は**すべてのプランで**有効** — 最大5人の家族メンバーとPremiumを共有できます。<br><br>

生涯購入とサブスクリプションはiCloud経由でiOSとMac間で共有されます。すでにiOSで支払った場合は、最新のアプリバージョンをインストールし、iCloudが有効であることを確認し、購入がアップロードされるまで約1分待ちます — そしてMac（同じApple ID + iCloud）でアプリを起動すると、Premiumが自動的にアクティブになります。また、いつでも**設定 → 購入 → 購入を復元**をタップできます。<br><br>
{{% /details %}}

{{% details title="Evertag FreeとEvertag Premiumの違いは何ですか？" closed="true" %}}

**Evertag Free**はいくつかの制限付きでコアタグ編集機能へのアクセスを提供します。広告が含まれており、タグエディタ、アルバムカバーエディタ、一括モードを制限された方法で使用できます：接続クラウドアカウント1つ、お気に入り最大10件、1日最大20回の自動タグ検索、1日最大20回のアルバムカバー検索。120以上のオーディオタグを編集し、ローカルまたは内蔵ファイルマネージャを通じてファイルを管理することもできます。<br><br>

**Evertag Premium（月額、年額、または生涯）**は広告を削除してすべてをアンロックします：無制限の120以上のタグフィールド、無制限の一括編集、エンコーディング修正、無制限のお気に入り、**無制限のクラウドアカウント**、無制限の自動タグ検索、無制限のアルバムカバー検索、完全なパーソナライゼーション（代替アプリアイコン、テーマ、元のサイズのアルバムアートワーク）。Premiumは音楽コレクター、DJ、ポッドキャスター、またはデバイスやクラウドプラットフォーム全体で大きなライブラリを管理する人に最適です。<br><br>

**価格オプション**<br>
• Premium月額 — $1.99/月<br>
• Premium年額 — $12.99/年<br>
• Premium生涯 — $24.99（一回限りの購入）<br><br>

すべてのPremiumオプションは同じ機能セットをアンロックします。[専用の比較ページ](/docs/faq/evertag/what-is-the-difference-between-evertag-and-evertag-premium/)で詳細な違いをご確認いただけます。<br><br>
{{% /details %}}

{{% details title="新しいiPhone、iPad、またはMacでPremium購入を復元するにはどうすればよいですか？" closed="true" %}}
元の購入に使用した**同じApple ID**でサインインし、Evertagで**設定 → 購入 → 購入を復元**をタップします。<br><br>

復元でPremiumステータスが戻らない場合：<br>
• 新しいデバイスでiCloudが有効であることを確認してください — EvertagはiCloud経由で購入データを同期します。<br>
• すでにPremiumが有効なデバイスでアプリを開き、購入メタデータがアップロードされるまで約1分待ちます。<br>
• 新しいデバイスで、**設定 → 購入**の**アプリ起動時に購入を復元**を有効にして、アプリを再起動します。<br>
• デバイスがアクティブなインターネット接続を持っていること、そして**iOS設定 → Apple ID → メディアと購入**のApp Storeアカウントがアップグレードを購入したものと一致していることを確認してください。<br><br>

サブスクリプションと生涯購入はiOSとMac（Mac Catalyst）間で共有され、**ファミリー共有**により最大5人の家族メンバーが同じPremiumプランを再利用できます。<br><br>
{{% /details %}}

{{% details title="Evertagは安全ですか？" closed="true" %}}
はい。Evertagはアカウントへのアクセスに**公式クラウドプロバイダーSDKと暗号化接続のみ**を使用し、パスワードをアプリと共有することはありません。クラウドサービスを接続すると、クラウドプロバイダーが提供する公式認証ページが表示されます — ログインフロー全体はEvertagの外部で行われ、プロバイダーはアプリがAPIコールに使用する認証トークンを返します。<br><br>

認証トークンはデバイス上の安全なシステム**キーチェーン**に保存され、当社のサーバーに送信されることはありません。接続されたクラウドサービスからダウンロードしたファイルはアプリの「ドキュメント」ディレクトリに配置され、内蔵ファイルマネージャからいつでも削除できます。<br><br>

Evertagは接続されたクラウドアカウントからの情報を共有しません。いつでもアクセスを取り消すことができます：<br>
• Evertag内 — クラウドアカウントを開き、**…**をタップして**切断する**を選択します。認証トークンはデバイスから削除されます。<br>
• クラウドプロバイダーのウェブサイト — サインインし、サードパーティアプリ / 接続済みアプリページに移動して、リストからEvertagを削除します。<br><br>

デバイスからEvertagを削除すると、ダウンロードされたすべてのデータとアクセストークンが一緒に削除されます。<br><br>
{{% /details %}}

{{% details title="Evertagは個人データを収集または共有しますか？" closed="true" %}}
このアプリは音楽ライブラリに関する情報を当社のサーバーに保存しません。編集したものはすべてデバイス上または自分のクラウドアカウントに残ります。<br><br>

Evertagは分析とクラッシュレポートのために最小限のサービスセット（Firebase AnalyticsとCrashlytics）を使用し、AdMob経由で無料版に広告を表示します。これらのサービスは**設定 → 分析とデータ収集**で確認してオフにすることができます。AppleのApp Tracking Transparencyプロンプトにより、アプリが最初に起動するときに追跡を拒否することもできます。完全な内訳については[プライバシーポリシー](/legal/privacy-policy/)をご覧ください。<br><br>
{{% /details %}}

{{% details title="EvertagをパスコードまたはFace IDで保護できますか？" closed="true" %}}
はい。**設定 → パスコード**を開き、4桁または6桁のコードを設定します。有効にすると、アプリが前景に来るたびにEvertagはパスコード（またはオプトインした場合はFace ID / Touch ID）を要求します。<br><br>

パスコードはiOS Keychainにローカルに保存されます — オンラインでの回復はありません。保護が不要になった場合は、同じ設定画面から無効にしてください。<br><br>
{{% /details %}}

{{% details title="Evertagはどのようなオーディオフォーマットをサポートしていますか？" closed="true" %}}
Evertagは30以上のオーディオフォーマットでタグを読み書きします：<br><br>

**MP3、OGG、OGA、FLAC、MPC、WV、SPX、OPUS、TTA、M4A、M4R、M4B、M4P、MP4、3G2、M4V、WMA、ASF、AIF、AIFF、AFC、AIFC、WAV、APE、MOD、MODULE、NTS、WOW、S3M、IT、XM。**<br><br>

これは最も一般的なロスレスフォーマット（FLAC、WAV、AIFF、M4A内のALAC、APE）からロッシー（MP3、M4A内のAAC、OGG Vorbis、OPUS、WMA）、さらにクラシックなトラッカーフォーマット（MOD、S3M、IT、XM）まですべてをカバーします。必要なフォーマットが欠けている場合は、**設定 → フィードバックを送る**からお知らせください。<br><br>

{{% details title="EvertagはID3v2.3またはID3v2.4フォーマットでタグを書き込めますか？" closed="true" %}}
はい。**設定 → オーディオタグエディタ → タグ保存オプション**を開き、**ID3v2.4**をオンまたはオフにします。<br><br>

• **ID3v2.4オン**（デフォルト） — 完全なUTF-8サポート、マルチバリューフレーム、より豊富なメタデータを持つモダン標準。現在のデスクトッププレーヤー、音楽マネージャー、ほとんどのモダンカーステレオに最適です。<br>
• **ID3v2.4オフ** — より広くサポートされている**ID3v2.3**フォーマットでタグを保存します。古いデバイス、カーヘッドユニット、またはDJソフトウェアが編集後に空白のタグを表示する場合に使用します。<br><br>

同じ設定ページには**重複タグ**オプションも公開されており、これはレガシープレーヤーとの最大の互換性のためにファイルの両方のID3セクションに共通のメタデータを書き込みます。<br><br>
{{% /details %}}

{{% details title="Evertagはオーディオタグ内の文字化けや破損した文字（キリル文字、中国語、日本語など）を修正できますか？" closed="true" %}}
はい。Evertagには組み込みの**エンコーディングノーマライザー**があり、誤ってエンコードされたテキスト（例えば`Ðàñêîëüíèêîâ`のような文字化けとして表示されるキリル文字、中国語、日本語、韓国語のタグ）を読み取り可能なUTF-8に変換します。<br><br>

使用方法：<br>
1. タグエディタでファイルを開きます。<br>
2. アートワークの下にある**エンコーディングを正規化**ボタンをタップします。<br>
3. ファイルを保存します。<br><br>

ボタンが表示されない場合は、**設定 → オーディオタグエディタ → メイン画面のボタン**で有効にしてください。エンコーディング修正は、タグリーダーがデフォルトでUTF-8の代わりにANSI / CP-1251 / Shift-JISを使用するWindowsで元々タグ付けされたライブラリに特に役立ちます。<br><br>
{{% /details %}}

{{% details title="どのようなクラウドサービスがサポートされていますか？" closed="true" %}}
Evertagは人気のあるクラウドサービスの全ラインナップに接続し、リモートファイルのタグを直接編集できます：<br><br>

**iCloud Drive、Google Drive、Dropbox、OneDrive、Box、MEGA、Yandex.Disk、pCloud、Synology Drive、MediaFire、WD My Cloud Home、InfiniCLOUD（TeraCLOUD）、HiDrive、OpenDrive、MyDrive、Put.io、Cloud Mail.ru、Baidu Pan（百度网盘）**。<br><br>

**SMB**または**WebDAV**を話す個人NAS、Apple Time Capsule、またはコンピュータも接続できます — ステップバイステップの設定については[接続ガイド](/docs/guide/evertag/evertag-guide-connections/)をご覧ください。無料版では1つのクラウドアカウントを追加できます；**Premium**では必要なだけ多くのアカウントを接続できます。<br><br>
{{% /details %}}

{{% details title="新しいクラウドアカウントを追加するにはどうすればよいですか？" closed="true" %}}
**接続**タブを開く → **クラウドストレージに接続**をタップ → リストからプロバイダーを選択（iCloud Drive、Google Drive、Dropbox、OneDrive、Box、MEGA、Yandex.Disk、pCloud、Synology Drive、MediaFire、WD My Cloud Home、InfiniCLOUD、HiDrive、OpenDrive、MyDrive、Put.io、Cloud Mail.ru、またはBaidu Pan）→ プロバイダーの公式ページでサインイン → **完了**をタップします。<br><br>

接続が失敗した場合は、インターネット接続と認証情報を再確認し、そのサービスの二要素認証が正しく設定されていることを確認してください。**Premium**バージョンでは無制限の数のアカウントを追加し、深くネストされたフォルダーを接続画面上部の**クイックアクセス**リストにピン留めしてワンタップブラウジングができます。<br><br>
{{% /details %}}

{{% details title="ライブラリ全体をダウンロードせずにGoogle Drive、Dropbox、またはiCloud Driveでオーディオタグを編集できますか？" closed="true" %}}
はい。クラウドアカウントを接続した後、Evertagは編集しようとしているファイルのみをストリーミングします — ライブラリ全体をローカルにミラーリングする必要はありません。<br><br>

リモートファイルをタップすると：<br>
1. Evertagが一時フォルダーにダウンロードします。<br>
2. 通常通りタグを編集します。<br>
3. **保存する**時に、更新されたファイルがクラウドの同じ場所に再アップロードされます。<br>
4. 一時コピーは削除されます（または**設定 → オーディオタグエディタ → オンラインファイルを編集**に応じて保持されます）。<br><br>

保存の動作は**設定 → オーディオタグエディタ → オンラインファイルを更新**で制御できます：各保存前に確認、常に自動的にアップロード、またはアップロードを完全にスキップ（ローカルのみの編集）。<br><br>
{{% /details %}}

{{% details title="Synology、QNAP、Asustor、またはApple Time CapsuleなどのNASでEvertagを使用できますか？" closed="true" %}}
はい。**SMB**または**WebDAV**を使用して任意のNASを接続します — これらはすべての主要なNASベンダー（Synology、QNAP、Asustor、TerraMaster、Western Digital、Buffalo、Apple Time Capsule）がデフォルトでサポートしている2つのプロトコルです。<br><br>

• **SMB** — **クラウドストレージに接続 → SMB**をタップし、`smb://nas-ipアドレス/共有フォルダー名`を入力し、プロトコルバージョン（Auto / SMB1 / SMB2）を選択し、ログイン+パスワードを入力し、**完了**をタップします。完全なチュートリアル：[SMB経由でMacまたはPCから音楽をストリーミング](/docs/howto/stream-your-music-from-mac-or-pc-to-iphone-using-smb/)。<br>
• **WebDAV** — 同じフロー、ただしSSL用に`http://サーバー名`または`https://サーバー名`を入力します。完全なチュートリアル：[WebDAV経由でNASストレージを接続](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac)。<br>
• **Synology Drive** — ファーストクラスの統合としてサポート（汎用WebDAVとは別）。完全なチュートリアル：[Synology NASを接続してiPhoneまたはMacで音楽を聴く](/docs/howto/how-to-connect-synology-nas-and-listen-to-music-on-your-iphone-or-mac/)。<br><br>

ローカルネットワークでブロードキャストしているデバイスも、接続画面の**利用可能なデバイス**セクションに自動的に表示されます。<br><br>
{{% /details %}}

{{% details title="編集後にクラウドサービスのファイルメタデータを自動的に更新するにはどうすればよいですか？" closed="true" %}}
クラウドファイルの保存動作は**設定 → オーディオタグエディタ → オンラインファイルを更新**で3つのオプションで制御されます：<br><br>

• **確認メッセージを表示**（デフォルト） — 編集されたファイルをクラウドに戻す前にアプリがプロンプトを表示します。<br>
• **ファイルのメタデータを自動的に更新** — タグエディタで**保存する**をタップした直後に、アプリは静かにファイルを再アップロードします。<br>
• **ファイルのメタデータを更新しない** — アプリはアップロードを完全にスキップします；変更はローカルのままで、クラウドコピーは手つかずです。<br><br>

これを**設定 → オーディオタグエディタ → オンラインファイルを編集**と組み合わせて、各編集後にローカルにダウンロードされたコピーを削除する（スペースを節約）か保持する（再オープンが速い）かを決定します。<br><br>
{{% /details %}}

{{% details title="クラウドファイルを編集した後に変更が表示されないのはなぜですか？" closed="true" %}}
まず、**オンラインファイルを更新**の設定が**ファイルのメタデータを更新しない**に設定されていないことを確認してください — そのモードは変更をローカルに保持し、クラウドに再アップロードすることはありません。**設定 → オーディオタグエディタ → オンラインファイルを更新**を開き、代わりに**確認メッセージを表示**または**ファイルのメタデータを自動的に更新**を選択してください。<br><br>

その他に確認すること：<br>
• クラウドフォルダーへの**書き込み権限**があること。一部の共有 / 読み取り専用WebDAV / SMBマウントはアップロードを静かに受け入れますが破棄します。<br>
• クラウドプロバイダーが静かにログアウトしていないこと — アカウントを開いて**接続 → … → 設定**から再認証します。<br>
• クラウドサービスのデスクトップまたはWebクライアントがまだ古いバージョンを同期している可能性があります。1分待ってからリフレッシュします。<br>
• **iCloud Drive**を使用している場合、ファイルが別のデバイスからまだアップロード中の可能性があります。iOSファイルアプリでファイルを開き、同期インジケーターがもうスピンしていないことを確認します。<br><br>

まだ解決しない場合は、ファイルフォーマット、クラウドサービス、短い説明を添えて**設定 → フィードバックを送る**を使用してください。<br><br>
{{% /details %}}

{{% details title="クラウドストレージからファイルをダウンロードするにはどうすればよいですか？" closed="true" %}}
最初に少なくとも1つの接続済みクラウドアカウントが必要です — **接続**画面を開いてプロバイダーを追加します。<br><br>

ファイルまたはフォルダーをダウンロードするには：<br>
1. 接続されたクラウドストレージサービスを開きます。<br>
2. ダウンロードしたいフォルダーに移動します。<br>
3. 右上の**その他のアクション**（`…`）ボタンをタップして**選択する**を選択します。<br>
4. 必要なファイルまたはフォルダーにチェックを入れ、**ダウンロード**をタップします。<br><br>

ダウンロードは**ローカルファイル → ダウンロード**フォルダーに届き、**転送キュー**に表示されます — ローカルファイル画面の左上にある回転矢印アイコンから開きます。デフォルトの保存先は**設定 → ファイルマネージャー → ダウンロードしたファイルの保存先**から変更でき、同じ画面からバックグラウンド転送を切り替えられます。<br><br>
{{% /details %}}

{{% details title="ネットワークストレージのファイルを管理するにはどうすればよいですか？" closed="true" %}}
クラウドフォルダーを開き、右上の**…**ボタンをタップして選択モードに入るために**選択する**を選択します。各ファイルとフォルダーの横にチェックボックスが表示され、1つまたは複数のアイテムを選択できます。<br><br>

選択で利用可能なアクション：<br>
• **オーディオタグを編集する** — 1つまたは複数のファイルを[タグエディタ](/docs/guide/evertag/evertag-guide-tag-editor/)で開きます（複数選択の場合は一括モード）。<br>
• **ダウンロード** — ファイルまたはフォルダーを**ローカルファイル**セクションにコピーします。<br>
• **移動** — 同じクラウドアカウント内の別のフォルダーにアイテムを移動します。<br>
• **名前変更** — リモートサーバー上のファイルまたはフォルダーの名前を変更します。<br>
• **削除する** — クラウドからファイルまたはフォルダーを完全に削除します（元に戻せません）。<br>
• **並べ替え** — 名前、サイズ、または編集日で並べ替えます。<br>
• **グリッド / リスト** — テーブルビューとサムネイルビューを切り替えます。<br><br>

すべてのアクションのスペースがない場合、ツールバーは**その他のアクション**オーバーフローボタンを表示します — タップして全リストを確認します。<br><br>
{{% /details %}}

{{% details title="iPhoneまたはiPadにローカルに保存された音楽ファイルのメタデータを編集するにはどうすればよいですか？" closed="true" %}}
**ローカルファイル**を開き、**このiPhone上のファイル**（またはiPad / Mac）セクションにスクロールします。<br><br>

• **ファイルを開く…**を選択してデバイス上の任意の場所で1つ以上のオーディオファイルを選択するか、**フォルダーを開く…**でフォルダー全体を選択します。<br>
• **開く**で確認 — ファイルはすぐにエディタに追加され、**その場で編集**され、Evertagにコピーされることはありません。<br><br>

**デバイスフォルダーへのクイックアクセス。** セクションにフォルダーを追加することで毎回ピッカーをスキップします：<br>
1. **ローカルファイル**を開きます。<br>
2. **このデバイス上のファイル**にスクロールして**フォルダーを接続する**をタップします。<br>
3. 音楽フォルダーを選択して**開く**をタップします。<br><br>

フォルダーは読み取り/書き込みアクセス付きで**このiPhone上のファイル**リストに表示されます — システムファイルアプリまたは別のアプリの公開フォルダーに保存された音楽ライブラリに最適です。後でリンクを解除するには、フォルダーの横にある**…**をタップして**切断する**を選択します。詳細なチュートリアル：[iPhoneまたはMacに保存されたローカル音楽を再生する](/docs/howto/how-to-play-local-music-stored-on-your-iphone-or-mac)。<br><br>
{{% /details %}}

{{% details title="コンピュータからEvertagに音楽を転送するにはどうすればよいですか？" closed="true" %}}
**SMB**、**WebDAV**、**Wi-Fi Drive**、または**Finder File Sharing**（古いmacOSのiTunes File Sharing）を使用してコンピュータまたはNASを接続できます。<br><br>

**SMB** — **クラウドストレージに接続 → SMB**をタップし、`smb://コンピュータのIPアドレス/共有フォルダー名`を入力し、認証情報を設定し、**完了**をタップします。完全なチュートリアル：[SMBプロトコルを使用してコンピュータからiPhoneにファイルを転送](/docs/howto/transfer-your-files-from-the-computer-to-iphone-using-smb-protocol/)。<br><br>

**WebDAV** — `http://サーバー名`（またはSSL用に`https://`）と同じフロー。完全なチュートリアル：[WebDAV経由でNASストレージを接続](/docs/howto/how-to-connect-nas-storage-using-webdav-and-listen-to-music-on-your-iphone-or-mac)。<br><br>

**Wi-Fi Drive** — 完全なワイヤレス。**接続 → コンピュータ → Wi-Fi経由で接続**を開いてサーバーを起動し、コンピュータのブラウザに表示されたURLを開き、ページにファイルを**ドラッグアンドドロップ**します。両方のデバイスが同じWi-Fiネットワークを共有する必要があります。完全なチュートリアル：[Wi-Fi Driveを使用してワイヤレスでファイルを転送](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive)。<br><br>

**Finder File Sharing**（または古いmacOS / WindowsのiTunes File Sharing） — ケーブルでiPhone/iPadをコンピュータに接続し、**Finder → デバイス → ファイル → Evertag**を開き、共有フォルダーにファイルをドラッグします。完全なチュートリアル：[iPhoneでローカルiTunesファイルを再生する](/docs/howto/how-to-play-local-itunes-files-on-my-iphone)。<br><br>
{{% /details %}}

{{% details title="インターネット接続なしでオフラインでオーディオタグを編集できますか？" closed="true" %}}
はい。ローカルファイルの編集は完全にオフラインで動作します — タグエディタ、一括モード、アルバムカバー交換、エンコーディング修正、歌詞エディタはインターネット接続を必要としません。<br><br>

インターネットが必要な機能は明確にマークされています：MusicBrainz自動検索、手動MusicBrainzタグ検索、アルバムアートワークウェブ検索、歌詞検索ショートカット（Lrclib、Genius、Lyricsify、Google）。セッション中にオフラインになると、それらのサービスは単に無効になります — ローカル編集には影響しません。<br><br>
{{% /details %}}

{{% details title="一度に複数のファイルを編集するにはどうすればよいですか（一括モード）？" closed="true" %}}
**一括モード**を使用して同じメタデータ変更を多くのトラックに一度に適用します。開始する2つの方法があります：<br><br>

**ファイルマネージャーから**<br>
1. ファイルが含まれているフォルダーを開きます。<br>
2. **その他のアクション**（`…`）ボタンをタップ → **選択する**。<br>
3. 編集したいファイルにチェックを入れます。<br>
4. **オーディオタグを編集する**をタップします。<br><br>

**タグエディタから**<br>
1. フォルダーから任意のファイルを開きます。<br>
2. タグエディタの一番下までスクロールします。<br>
3. **ファイルを同時に編集**をタップ — 同じフォルダー内のすべてのオーディオファイルが一緒に読み込まれます。<br><br>

一度変更を加えて**保存する**をタップ — 新しい値がすべての選択されたファイルに書き込まれます。アルバム全体のアルバム名、アルバムアーティスト、ジャンル、年、またはトラック合計を修正するのに最適です。<br><br>
{{% /details %}}

{{% details title="複数の曲のアルバムカバーを一度に変更するにはどうすればよいですか？" closed="true" %}}
**一括モード**をアクティブにして（上記参照）、アートワークカルーセルの**カメラアイコン**をタップし、新しい画像を選択 — 同じアートワークが1回の保存で選択されたすべてのファイルに書き込まれます。<br><br>

新しいカバーのソース：<br>
• **フォトライブラリ** — すでに保存またはスキャンしたカバーに役立ちます。<br>
• **ローカルファイル / ダウンロード** — デバイス上にすでにある任意の画像。<br>
• **接続されたクラウドアカウント** — クラウド内のJPG/PNGにEvertagを向けます。<br><br>

保存された画像品質は**設定 → オーディオタグエディタ → アルバムカバースケーリング**から制御できます — Small / Medium / Large / Extra Large、または元のサイズを維持するための「Deactivated」（Premium）。<br><br>
{{% /details %}}

{{% details title="不足しているアルバムアートワークを自動的に見つけるにはどうすればよいですか？" closed="true" %}}
タグエディタのアートワーク下の**アルバムカバーを検索**ボタンをタップします。Evertagはトラックの現在のアーティストとアルバムメタデータを使用して、一致するカバーアートをオンラインで検索します。<br><br>

検索結果が表示されたら、気に入った画像を長押しして、システムメニューの**写真に保存**を選択します。Evertagに戻り、アートワークの**カメラアイコン**をタップして**フォトライブラリ**を選択し、保存した画像を選択 — カバーがファイルに適用されます。アルバムアートワークを保存ボタン（同じメニュー下）でも現在のアートワークを後で再利用するためにドキュメントフォルダーにバックアップできます。<br><br>

アルバムカバーを検索ボタンが表示されない場合は、**設定 → オーディオタグエディタ → メイン画面のボタン**で有効にしてください。<br><br>
{{% /details %}}

{{% details title="EvertagはMusicBrainzを使用しますか？" closed="true" %}}
はい。Evertagの自動および手動タグ検索は、5,000万以上のトラックを持つ最大のオープン音楽メタデータプロジェクトの1つである**MusicBrainz**データベースを利用しています。<br><br>

タグエディタのボトムツールバーから2つのフローが利用可能です：<br>
• **オーディオタグを自動検索** — 既存のアーティスト / アルバム / タイトルメタデータを使用したベストマッチ検索。<br>
• **オーディオタグを手動検索** — アルバム、曲、適用するフィールドの正確なセットを選択します。<br><br>

MusicBrainz識別子を直接保存することもできます：Album ID、Album Artist ID、Artist ID、Disc ID、Release Group ID、Release Track ID、Track ID、Work IDなど（[タグフィールドマッピング](/docs/guide/evertag/evertag-tag-field-mappings/)リファレンスを参照）。これらのIDにより、他のMusicBrainz対応アプリ（Picard、MusicBrainz Browser、Roon、Plex）がトラックを確実にクロスリファレンスできます。<br><br>
{{% /details %}}

{{% details title="Evertagで曲に歌詞を追加するにはどうすればよいですか？" closed="true" %}}
歌詞は**拡張タグエディタ**にあります。クイックパスは次の通りです：<br><br>

1. ファイルをタップしてタグエディタを開きます。<br>
2. **拡張タグを表示**をタップして完全なフィールドセットを有効にします。<br>
3. **同期されていない歌詞**セクションにスクロールしてフィールドをタップします。<br>
4. **新しいページを追加**をタップ — 各歌詞ページには独自の**言語**と**説明**があるため、1つのトラックで複数の翻訳を保持できます。<br>
5. **テキスト**フィールドにプレーンな歌詞または**LRCフォーマット（タイムスタンプ付き）の歌詞**を貼り付けます — プレースホルダーに例が表示されます。<br>
6. 各画面で**完了**をタップし、次に**保存する**をタップして歌詞をオーディオファイルに書き込みます。<br><br>

**タイピングしたくない場合**、歌詞エディタには最も人気のある歌詞データベースへのワンタップ**検索ショートカット**があります — 現在のトラックのアーティストとタイトルが事前入力されています：<br>
• **Lrclib** — **タイムド（LRC）歌詞**の主要ソース。<br>
• **Genius** — 正確なプレーンテキスト歌詞の大規模カタログ。<br>
• **Lyricsify** — プレーンと同期歌詞のコミュニティデータベース。<br>
• **Google** — フォールバックとしての汎用ウェブ検索。<br><br>

各ボタンは対応するサービスに到達可能な場合にのみ表示されます。完全なウォークスルー：[iPhoneまたはMacでオーディオファイルの歌詞を編集する方法](/docs/howto/how-to-edit-lyrics-for-audio-files-on-iphone-mac/)。<br><br>

**同期（LRC）歌詞。** エディタはプレーンテキスト**と**LRCフォーマット歌詞（行タイムスタンプ`[mm:ss.ms]`）の両方を受け付けます — LRCテキストをフィールドに貼り付けて保存すると、オーディオファイルの歌詞タグがそのまま保存され、LRC対応プレーヤーで使用できます。内蔵の**Lrclib**と**Lyricsify**ショートカットはデフォルトでLRC文字列を返すため、アプリを離れることなく数秒で完全に同期された歌詞ライブラリを構築できます。<br><br>
{{% /details %}}

{{% details title="バグを報告したりサポートに連絡するにはどうすればよいですか？" closed="true" %}}
**設定 → フィードバックを送る**を開いて、ログを添付してサポートチームにメールを送ります。ファイルフォーマット（例：FLAC）、クラウドサービス（関連する場合）、iOS / macOSバージョンを含めると診断が大幅に速くなります。<br><br>

セルフサービスの回答については、[ユーザーガイド](/docs/guide/evertag/)、[ハウツー記事](/docs/howto/)、またはこのFAQをご確認ください。ドキュメントにタイポや欠けているトピックを見つけた場合は、同じ方法でお知らせください — 定期的にガイドを更新しています。<br><br>
{{% /details %}}

{{% details title="Evertagをアンインストールしてすべてのデータを削除するにはどうすればよいですか？" closed="true" %}}
iOS、iPadOS、またはmacOSからアプリを削除すると、Evertagが作成した**すべての**ファイルが削除されます。ダウンロードした曲、キャッシュされたサムネイル、アルバムカバー、設定、パスコード、接続されたクラウドアカウントの認証トークンを含みます。<br><br>

アプリを保持しつつアンインストールせずにスペースを解放したい場合：<br>
• **設定 → ファイルマネージャー → ファイルのサムネイル → すべて削除** — キャッシュされたサムネイルを削除します。<br>
• **設定 → ファイルマネージャー → 一時ファイルを削除** — キャッシュフォルダーをクリアします。<br>
• **ローカルファイル → ダウンロード**を開き、不要になった個々のファイルやフォルダーを削除します。<br>
• **接続**画面からクラウドアカウントを切断して認証トークンを削除します。<br><br>

クラウド側のファイルはEvertagをアンインストールしても削除されません — クラウドプロバイダーのサーバーに残り、任意のWebクライアントから再度アクセスできます。<br><br>

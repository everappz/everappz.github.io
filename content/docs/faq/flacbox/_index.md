<div class="hx:mt-6"></div>

Flacbox is a hi-res music player for **iPhone, iPad, and Mac** that streams and downloads tracks from iCloud Drive, Google Drive, Dropbox, OneDrive, MEGA, Box, pCloud, Yandex Disk, Synology, QNAP, WD My Cloud Home, and any other WebDAV, SMB, or DLNA server. Built on the system codecs plus the bundled **FFmpeg** library, it adds support for formats Apple’s default player can’t handle — WMA, OGG, OPUS, APE, DSD — and gives audiophiles control over output sample rate, multi-channel routing, pitch correction, and IO buffer duration. This FAQ answers the questions users send most often. For deeper walkthroughs, see the [Flacbox User Guide](/docs/guide/flacbox/) or browse our [How-to articles](/docs/howto/).

{{% details title="What is the difference between Flacbox and Evermusic?" closed="true" %}}
**Flacbox is the audiophile-focused player with FFmpeg-powered format support, configurable sample rate, multi-channel output, and pitch correction; Evermusic is the mainstream player with crossfade, gapless, spatial audio, and a broader cloud feature set.**<br><br>

**Flacbox** is built to support all default iOS audio formats along with many additional formats not natively supported on iPhone, such as **WMA, OGG, OPUS, APE, DSD, DSF, DFF, TTA, MPC, WV,** and more. It uses a custom audio engine (system codecs + FFmpeg) to handle nearly every format, and offers audiophile-grade features like **adjustable audio output sample rate (44.1 – 96 kHz), multi-channel output (Mono → 5.1 → SDDS), IO buffer tuning, and pitch correction (-1000 to +1000)**.<br><br>

**Evermusic** supports the default Apple audio formats and includes consumer-friendly features like **crossfade playback, gapless playback, spatial audio, audio equalizer, playback speed control,** and broader cloud-streaming tooling. If you mainly use MP3, AAC, ALAC, or FLAC and want crossfade, gapless, or spatial audio, **Evermusic** may be the better option. If you need broad compatibility with a wider range of audio formats — especially DSD or WMA — and want fine control over the audio output chain, **Flacbox** is the right choice.<br><br>

[Learn more about the differences](/docs/faq/evermusic/what-is-the-difference-between-evermusic-and-flacbox/)

{{% details title="Is Flacbox safe?" closed="true" %}}
**Yes — Flacbox uses only official SDKs and secure OAuth connections to interact with connected cloud services, never sees your password, and stores only encrypted access tokens in the iOS Keychain.**<br><br>

When you enter your login and password, the application shows you the official authorization page provided by the cloud service provider, and the entire authorization process happens outside Flacbox. The cloud provider then sends an **auth-token** to the application after successful authorization, and that token is used to make API calls.<br><br>

An auth-token is a digital key that allows third-party applications to interact with cloud storage. The token is stored on your device in **Apple’s secure system storage (Keychain)**, which is encrypted at rest and protected by your device passcode or biometric lock. Downloaded files are stored in the app’s **Documents** directory; you can remove them at any time using the built-in file manager.<br><br>

The application **does not share any information** from connected cloud accounts with Everappz, advertisers, or any third party. You can revoke access to your cloud account at any time by opening the account-settings page in your web browser. To revoke an auth-token, log in to your cloud account on the web, navigate to the security or connected-apps page, and remove the entry for Flacbox.<br><br>

You can also disconnect the cloud account inside the application itself — when you do, the auth-token is immediately deleted from your device. Uninstalling the application deletes all downloaded data and tokens automatically.<br><br>

{{% details title="What cloud services, media servers, and protocols does Flacbox support?" closed="true" %}}
**Flacbox supports virtually every popular cloud storage service, self-hosted media server, file-share protocol, and S3-compatible object store — all from one Connect to cloud storage screen.**<br><br>

**Personal cloud storage:** iCloud Drive · Google Drive · Dropbox · OneDrive (Personal and Business) · Box · MEGA · pCloud · Yandex Disk · WD My Cloud Home · MediaFire · TeraCLOUD (InfiniCLOUD) · HiDrive · IceDrive · Koofr · OpenDrive · MyDrive · Put.io · Cloud Mail.ru · Internxt · Proton Drive · AliDrive (阿里云盘) · Baidu Pan (百度网盘).<br><br>

**Self-hosted media servers and music-server APIs:** **Plex** · **Jellyfin** · **Emby** · **Subsonic** (and every Subsonic-compatible server — Airsonic, Funkwhale, Gonic, Logitech Media Server, Ampache) · **Navidrome** · Nextcloud (and ownCloud via the shared API) · **Synology Drive** · **QNAP**.<br><br>

**NAS and file-share protocols:** **SMB** (SMB1, SMB2, Auto) · **WebDAV** (HTTP / HTTPS) · **FTP / FTPS** · **SFTP** (password or public-key auth) · **NFS** · **DLNA / UPnP** (playback and download). Works with Apple Time Capsule, Synology, QNAP, WD My Cloud Home, Buffalo, any Linux Samba / NFS / SSH host, or a shared folder on your Mac or Windows PC.<br><br>

**S3-compatible object storage:** AWS S3, Backblaze B2, Wasabi, Cloudflare R2, MinIO, DigitalOcean Spaces, and any other S3-API endpoint.<br><br>

**Local-network discovery:** the **Available Devices** section auto-lists every Bonjour / mDNS service on your Wi-Fi network — Plex, Jellyfin, Emby servers, Synology, QNAP, AirPort routers with attached drives, Time Capsule — so you can tap to connect without typing an IP address.<br><br>

Each connection uses the **official SDK or open protocol** of the service, with OAuth-based authorization where supported. You can connect multiple accounts of the same service (for example, two Google Drive accounts, a personal Dropbox alongside a work one, or both a Plex and a Jellyfin server). Premium users can add an unlimited number of services; the free version is limited to three.<br><br>

{{% details title="How do I transfer music to Flacbox from my computer?" closed="true" %}}
**You can transfer music to Flacbox over SMB, WebDAV, FTP / SFTP, DLNA, Wi-Fi Drive, or via iTunes / Finder File Sharing — any method that exposes the files to your device works.**<br><br>

To connect a computer using **SMB** protocol, tap **Connect a cloud service → SMB**. Enter the computer’s IP address and the shared folder name in the URL field using the format `smb://computer-ip-address/shared-folder-name`, enter your login and password, then tap **Done**. If the connection is successful, the share will appear in the **Cloud Services** section of the Connections screen.<br><br>

[Full tutorial: transfer your files from the computer to iPhone using SMB protocol](/docs/howto/transfer-your-files-from-the-computer-to-iphone-using-smb-protocol/)<br><br>

For **WebDAV**, all steps are the same except for the URL field — use `http://server-name` or `https://server-name` if the server supports SSL. WebDAV works great with Synology, QNAP, Nextcloud, ownCloud, and any other server that exposes a WebDAV endpoint.<br><br>

**Wi-Fi Drive** is a popular technology that lets you transfer files from your computer to an iOS device wirelessly using any desktop web browser. To use this feature, your device and computer should be on the same Wi-Fi network. Open **Connections → Computer → Connect using Wi-Fi** and enable the server. Then open a desktop browser and enter the URL shown by the app. You can drag and drop files from your computer onto the open web page and they will appear on your device almost immediately.<br><br>

[More detailed tutorial: how to transfer files wirelessly from a computer to iPhone using WiFi-Drive](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive)<br><br>

**iTunes / Finder File Sharing** is another option, transferring files from a computer to a device over a Lightning or USB-C cable. Connect the device to the computer, open iTunes (or Finder on macOS Catalina and later), select the device, find **Flacbox** in the apps list, and copy files into its shared folder.<br><br>

[Detailed instructions: how to play local files on iPhone](/docs/howto/how-to-play-local-itunes-files-on-my-iphone)<br><br>

{{% details title="Does Flacbox support Plex, Jellyfin, Emby, Subsonic, and Navidrome?" closed="true" %}}
**Yes — Flacbox connects natively to Plex Media Server, Jellyfin, Emby, Subsonic, and Navidrome**, so you can stream your self-hosted music library without ever exposing the underlying file share.<br><br>

- **Plex Media Server** — tap **Connect to cloud storage → Plex**, sign in with your Plex account, and pick a server. Plex servers on the same local network are also discovered automatically in **Available Devices**.<br>
- **Jellyfin** (open-source) — tap **Connect to cloud storage → Jellyfin**, enter your server URL (something like `http://server-ip:8096`) and credentials.<br>
- **Emby** — tap **Connect to cloud storage → Emby**, then enter the server URL and login.<br>
- **Subsonic and Subsonic-compatible servers** — tap **Connect to cloud storage → Subsonic**, enter the server URL and credentials. This works with the original **Subsonic** server as well as **Navidrome**, **Airsonic**, **Funkwhale**, **Gonic**, **Logitech Media Server (LMS)**, and **Ampache** — anything that speaks the Subsonic API.<br><br>

Once connected, the server appears alongside your cloud accounts in the Connections screen. {{% details title="Does Flacbox support FTP, FTPS, SFTP, and NFS?" closed="true" %}}
**Yes — Flacbox supports the classic Unix file-transfer protocols: FTP, FTPS, SFTP (SSH File Transfer Protocol), and NFS.**<br><br>

- **FTP / FTPS** — tap **Connect to cloud storage → FTP**, enter the host URL in the form `ftp://server-name` (or `ftps://server-name` for an encrypted connection), provide your login and password, then tap **Done**.<br>
- **SFTP** — tap **Connect to cloud storage → SFTP**, enter the server address, and authenticate with either a password or an SSH key pair. SFTP works with NAS devices, Linux hosts, and any server that runs `sshd`.<br>
- **NFS (Network File System)** — tap **Connect to cloud storage → NFS** and enter the server address and exported path. Handy for Linux hosts, BSD servers, and NAS devices that prefer NFS over SMB.<br><br>

This is on top of **SMB (SMB1, SMB2, Auto), WebDAV (HTTP / HTTPS),** and **DLNA / UPnP**, which Flacbox also supports natively.

{{% details title="How do I use the Wi-Fi Drive feature in Flacbox?" closed="true" %}}

**Transfer files using Windows File Explorer**<br>
1. Right-click **This PC → Map Network Drive…**.<br>
2. Enter the server URL from the Flacbox app in the **Folder** field.<br>
3. Choose a drive letter, click **Finish**, and browse your iPhone’s files directly.<br><br>

{{% details title="Does Flacbox support Hi-Res audio (24-bit / 96 kHz)?" closed="true" %}}

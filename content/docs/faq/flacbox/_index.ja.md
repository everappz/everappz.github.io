{{% details title="FlacboxとEvermusicの違いは何ですか？" closed="true" %}}
**FlacboxはFFmpeg対応のフォーマットサポート、設定可能なサンプルレート、マルチチャンネル出力、ピッチ補正を備えたオーディオファイル向けプレーヤーです。Evermusicはクロスフェード、ギャップレス、空間オーディオ、より広範なクラウド機能セットを備えたメインストリームプレーヤーです。**<br><br>

**Flacbox**は、すべてのデフォルトiOSオーディオフォーマットに加え、**WMA、OGG、OPUS、APE、DSD、DSF、DFF、TTA、MPC、WV**など、iPhoneでネイティブサポートされていない多くの追加フォーマットをサポートするように設計されています。カスタムオーディオエンジン（システムコーデック + FFmpeg）を使用してほぼすべてのフォーマットを処理し、**調整可能なオーディオ出力サンプルレート（44.1〜96kHz）、マルチチャンネル出力（モノ→5.1→SDDS）、IOバッファ調整、ピッチ補正（-1000〜+1000）**などのオーディオファイル品質の機能を提供します。<br><br>

**Evermusic**はデフォルトのAppleオーディオフォーマットをサポートし、**クロスフェード再生、ギャップレス再生、空間オーディオ、オーディオイコライザー、再生速度コントロール**などの消費者向け機能と、より広範なクラウドストリーミングツールを含んでいます。主にMP3、AAC、ALAC、FLACを使用し、クロスフェード、ギャップレス、または空間オーディオが必要な場合は、**Evermusic**の方が良い選択肢かもしれません。より幅広いオーディオフォーマット（特にDSDやWMA）との広い互換性が必要で、オーディオ出力チェーンの細かいコントロールが必要な場合は、**Flacbox**が適切な選択です。<br><br>

{{% details title="Flacboxは安全ですか？" closed="true" %}}
**はい — Flacboxは公式SDKと安全なOAuth接続のみを使用して接続されたクラウドサービスと対話し、パスワードを見ることは一切なく、暗号化されたアクセストークンのみをiOS Keychain に保存します。**<br><br>

ログインとパスワードを入力すると、アプリはクラウドサービスプロバイダーが提供する公式の認証ページを表示し、認証プロセス全体がFlacboxの外部で行われます。クラウドプロバイダーは認証成功後にアプリに**auth-token**を送信し、そのトークンがAPIコールに使用されます。<br><br>

auth-tokenはサードパーティアプリケーションがクラウドストレージと対話できるようにするデジタルキーです。トークンはデバイスの**Appleのセキュアなシステムストレージ（Keychain）**に保存されており、保存時に暗号化され、デバイスのパスコードまたは生体認証ロックで保護されています。ダウンロードされたファイルはアプリの**ドキュメント**ディレクトリに保存されます。内蔵ファイルマネージャーを使用していつでも削除できます。<br><br>

アプリは接続されたクラウドアカウントから**いかなる情報もEverappz、広告主、または第三者と共有しません**。ウェブブラウザでアカウント設定ページを開くことで、いつでもクラウドアカウントへのアクセスを取り消すことができます。auth-tokenを取り消すには、ウェブ上でクラウドアカウントにログインし、セキュリティまたは接続済みアプリページに移動して、Flacboxのエントリーを削除してください。<br><br>

アプリ内でもクラウドアカウントを切断できます — そうすると、auth-tokenはすぐにデバイスから削除されます。アプリをアンインストールすると、ダウンロードされたすべてのデータとトークンが自動的に削除されます。<br><br>

{{% details title="Flacboxでプレイリストを作成するにはどうすればよいですか？" closed="true" %}}
**プレイリストを作成するには、「プレイリスト」タブを開き、「+」または「...」メニューをタップし、「新しいプレイリスト」を選択して名前を付け、ライブラリ、このアプリのファイル、このデバイスのファイル、または接続からトラックを選択して追加します。**<br><br>

- **プレイリスト**セクションを開きます。<br>
- **+**ボタンまたは右上隅の**「...」**ボタンをタップして、**新しいプレイリスト**を選択します。<br>
- プレイリストの名前を入力して**保存する**をタップします。**曲を追加**ダイアログが表示されます。<br>
- プレイリストに追加したいトラックを選択し、**完了**をタップします。<br><br>

デフォルトでは、各曲はプレイリストに1度しか追加できません。重複した曲を許可するには、**設定 → 音楽ライブラリ → プレイリスト → プレイリスト内の重複 → 有効にする**を有効にします。後で曲の順序を変更したり、プレイリストの名前を変更したり、カバー画像を編集したり、ZIPとしてアーカイブしたり、M3U、CSV、TXTにエクスポートしたりできます。<br><br>

{{% details title="Flacboxはどのようなクラウドサービス、メディアサーバー、プロトコルをサポートしていますか？" closed="true" %}}
**Flacboxはほぼすべての人気クラウドストレージサービス、セルフホストメディアサーバー、ファイル共有プロトコル、S3互換オブジェクトストアをサポートしています — すべて1つの「クラウドストレージに接続」画面から。**<br><br>

**個人用クラウドストレージ：** iCloud Drive · Google Drive · Dropbox · OneDrive（個人・ビジネス） · Box · MEGA · pCloud · Yandex Disk · WD My Cloud Home · MediaFire · TeraCLOUD (InfiniCLOUD) · HiDrive · IceDrive · Koofr · OpenDrive · MyDrive · Put.io · Cloud Mail.ru · Internxt · Proton Drive · AliDrive（阿里云盘）· Baidu Pan（百度网盘）<br><br>

**セルフホストメディアサーバーと音楽サーバーAPI：** **Plex** · **Jellyfin** · **Emby** · **Subsonic**（およびSubsonic互換のすべてのサーバー — Airsonic、Funkwhale、Gonic、Logitech Media Server、Ampache）· **Navidrome** · Nextcloud（共有API経由のownCloudを含む）· **Synology Drive** · **QNAP**<br><br>

**NASとファイル共有プロトコル：** **SMB**（SMB1、SMB2、Auto）· **WebDAV**（HTTP / HTTPS）· **FTP / FTPS** · **SFTP**（パスワードまたは公開鍵認証）· **NFS** · **DLNA / UPnP**（再生とダウンロード）。Apple Time Capsule、Synology、QNAP、WD My Cloud Home、Buffalo、Linux Samba / NFS / SSHホスト、またはMacやWindows PCの共有フォルダで動作します。<br><br>

**S3互換オブジェクトストレージ：** AWS S3、Backblaze B2、Wasabi、Cloudflare R2、MinIO、DigitalOcean Spaces、およびS3-APIエンドポイントを公開するその他のサービス<br><br>

**ローカルネットワーク検出：** **利用可能なデバイス**セクションはWi-Fiネットワーク上のすべてのBonjour / mDNSサービスを自動的にリストします — Plex、Jellyfin、Embyサーバー、Synology、QNAP、ドライブが接続されたAirPortルーター、Time Capsule — IPアドレスを入力せずにタップして接続できます。<br><br>

各接続はサービスの**公式SDKまたはオープンプロトコル**を使用し、サポートされている場合はOAuthベースの認証を使用します。同じサービスの複数のアカウントを接続できます（例えば、2つのGoogle Driveアカウント、個人用のDropboxと仕事用のDropbox、またはPlexとJellyfinの両サーバー）。Premiumユーザーは無制限のサービスを追加できます。無料版は3つに制限されています。<br><br>

{{% details title="Flacboxでオフラインモードを有効にするにはどうすればよいですか？" closed="true" %}}
**クラウドサービスに接続し、音楽フォルダを見つけ、「その他のアクション → オフラインモードを有効にする」をタップします — フォルダとそこに追加された新しいファイルが自動的に「ローカルファイル → オフラインフォルダー」にダウンロードされます。**<br><br>

- **クラウドサービスに接続：** **接続**タブを開きます。**クラウドサービスに接続**を選択し、指示に従います。<br>
- **音楽フォルダーに移動：** 接続されたクラウドサービスを開き、音楽フォルダーを見つけます。<br>
- **オフラインモードを有効にする：** フォルダー名の横の**「...」**ボタンをタップして**オフラインモードを有効にする**を選択します。<br>
- **コンテンツをダウンロード：** 選択されたフォルダーとそのすべてのコンテンツが**ローカルファイル → オフラインフォルダー**にダウンロードされます。<br>
- **自動更新：** アプリは継続的に変更をスキャンします。オンラインフォルダーに追加された新しいファイルは自動的にダウンロードされます。<br>
- **スキャンタイムアウトを設定：** スキャン頻度を設定するには、**設定 → ファイルマネージャー → オフラインフォルダー → 時間間隔**に移動します。<br>
- **手動同期：** 手動で同期するには、**設定 → ファイルマネージャー → オフラインフォルダー → 同期されたオフラインフォルダー**に移動します。**「...」**をタップして**同期する**を選択します。<br><br>

{{% details title="iPhoneにローカルダウンロードした音楽を再生するにはどうすればよいですか？" closed="true" %}}
**「ローカルファイル」を開き、「このiPhone上のファイル」にスクロールし、「ファイルを開く...」または「フォルダを開く...」をタップして、再生したいオーディオファイルまたはフォルダを選択します。**<br><br>

アプリはフォルダのコンテンツをスキャンし、見つかったオーディオファイルをプレーヤーキューに追加し、アプリケーションバンドルに何もコピーせずに元の場所から直接再生します。<br><br>

**クイックアクセスのためにフォルダをお気に入りに追加する**<br>
**ローカルファイル**画面を開き、**クイックアクセス**までスクロールして**お気に入り**をタップして**お気に入りのファイル**画面にアクセスします。右上隅のその他のアクションボタン**「...」**をタップして**フォルダを追加**を選択します。希望のフォルダに移動して**開く**をタップして確認します。フォルダが**お気に入りのファイル**に追加され、手順を繰り返さずに音楽にすばやくアクセスできます。<br><br>

**ローカルファイルを音楽ライブラリにインポートする**<br>
ローカルファイルを音楽ライブラリ内で整理する場合は、**音楽ライブラリ**画面を開き、右上隅の三点ボタンをタップして**+ 音楽を追加**を選択します。**ローカルファイル**メニュー項目を選択し、**このiPhone上のファイル**セクションまでスクロールして、**ファイルを開く...**をタップします。追加したいファイルを選択して確認のために**開く**をタップします。アプリは選択されたファイルをスキャンして音楽ライブラリに追加し、アーティスト、アルバム、ジャンルなどのメタデータで整理します。<br><br>

**プレイリストにローカルファイルを追加する**<br>
プレイリストにローカルファイルを追加するには、**プレイリスト**画面を開いて右上隅のその他ボタンをタップします。**+ 新しいプレイリスト**を選択し、新しいプレイリストの名前を入力して、次の画面で**ローカルファイル**オプションを選択します。**このiPhone上のファイル**セクションまでスクロールして**ファイルを開く...**をタップします。追加したいオーディオファイルを選択して確認のために**開く**をタップします。ファイルがプレイリストに追加され、その他ボタンを使用して曲の順序を変更したりその他のアクションを実行したりできます。<br><br>

{{% details title="プレイリストを途中から再開するにはどうすればよいですか？" closed="true" %}}
**設定 → オーディオプレーヤー → 一般で「オーディオプレーヤーの状態を保存する」を有効にし、任意のプレイリスト、アルバム、アーティスト、またはフォルダーの上部にある「再生を続ける」をタップすると、最後の正確な位置から再開します。**<br><br>

別のプレイリストに切り替えて戻ると、アルバムアートワークの下のトップツールバーに4つのアクションが表示されます：**検索、再生を続ける、すべて再生**、**シャッフル再生**。**再生を続ける**をタップして、最後に保存した状態とメディア位置からプレイリストを再開します。各トラック内の位置を記憶するために**再生位置を保存する**を有効にすることもできます — オーディオブックや長いミックスに便利です。

**はい — Flacboxは同梱のFFmpegエンジンを通じてDSDファミリーフォーマット（DSF、DFF）を再生します。** ほとんどのコンシューマーDACはLightning / USB-Cを介してネイティブDSDを受け付けないため、DSDは出力前にリアルタイムでPCMにデコードされます。<br><br>

DSDで最良の結果を得るには、**設定 → オーディオプレーヤー → 一般**で**オーディオコーデック**を**FFmpeg**に切り替えて、DACに合わせた**オーディオ出力サンプルレート** **88.2kHz**または**96kHz**を選択します。完全に透明なデコードのためにFlacboxをUSBまたはLightning DACとペアリングします。
{{% /details %}}

{{% details title="FlacboxはハイレゾオーディオをサポートしていますかHi-Res（24ビット / 96kHz）？" closed="true" %}}
**はい — FlacboxはFFmpegエンジンを通じてハイレゾオーディオ再生をサポートし、44.1、48、64、88.2、96kHzの設定可能な出力サンプルレートに対応しています。**<br><br>

**設定 → オーディオプレーヤー → 一般**で**オーディオコーデック**を**FFmpeg**に切り替えて、希望の**オーディオ出力サンプルレート**とチャンネル数を選択します。iPhone、iPad、またはMacからビット完全な出力を得るために、互換性のある**外部DACまたはUSB / Lightningオーディオインターフェース**と組み合わせます。
{{% /details %}}

{{% details title="FlacboxでUSB DACを使用するにはどうすればよいですか？" closed="true" %}}
**USB DACをiPhone、iPad、またはMacに接続し（Lightning-to-USB、USB-C、または直接）、設定 → オーディオプレーヤー → 一般でFFmpegコーデックに切り替えてDACが期待するサンプルレートとチャンネル数を選択します。**<br><br>

多くのオーディオファイルDACは**ステレオ**出力で**88.2kHz**または**96kHz**を好みます。低遅延のために**オーディオ出力優先IOバッファ時間**を約**5ms（0.005秒）**に下げ、ノイズが聞こえる場合は少し上げます。FlacboxはAudioQuest、iFi、Chord、FiiO、ToppingなどのほとんどのクラスコンプライアントUSBおよびLightning DACと動作します。
{{% /details %}}

{{% details title="FlacboxはWMA、OGG、OPUS、APEファイルを再生できますか？" closed="true" %}}
**はい — Flacboxは同梱のFFmpegエンジンを通じてWMA、OGG、OPUS、APE、TTA、MPC、WVおよび他の多くのあまり一般的でないフォーマットを再生します**。iOSがネイティブにサポートしていない場合でも。<br><br>

{{% details title="Flacboxはマルチチャンネルまたは5.1サラウンド出力をサポートしていますか？" closed="true" %}}

{{% details title="Flacboxでクラウド音楽が同期されない理由は何ですか？" closed="true" %}}
**ほとんどの同期の問題は期限切れのauth-token、アプリのバックグラウンド化、またはアクティブなインターネット接続がないことが原因です — 接続を開き、サービスを再認証して、設定 → ライブラリ → オンライン同期から手動で同期を実行します。**<br><br>

オンライン同期はアプリがフォアグラウンドにあるときにのみ実行されるため、大きなライブラリの同期には時間がかかることがあります。速くするには、Flacboxを開いたままにしてデバイスを電源に接続し、**設定 → 画面 → 常にアクティブ**を有効にします。非常に大きなライブラリの場合は、アプリのデスクトップバージョンで同期を実行して、**バックアップと復元**を使用して結果をiOSに転送します。
{{% /details %}}

{{% details title="Flacboxのサポートに連絡するにはどうすればよいですか？" closed="true" %}}
**設定 → フィードバックを送信を開いて、診断情報が自動的に添付された状態でアプリから直接サポートチームにメールを送ります。**<br><br>

[ヘルプセンター](/docs/)を訪れたり、[ハウツーガイド](/docs/howto/)を閲覧したり、より広い[FAQ](/docs/faq/)でセルフサービスの回答を確認することもできます。通常は1営業日以内に返信します。

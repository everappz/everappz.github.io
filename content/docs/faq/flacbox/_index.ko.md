Flacbox는 iCloud Drive, Google Drive, Dropbox, OneDrive, MEGA, Box, pCloud, Yandex Disk, Synology, QNAP, WD My Cloud Home 및 WebDAV, SMB, DLNA 서버에서 트랙을 스트리밍하고 다운로드하는 **iPhone, iPad, Mac**용 **고해상도 음악 플레이어**입니다. 시스템 코덱과 번들 **FFmpeg** 라이브러리를 기반으로 구축되어 Apple의 기본 플레이어가 처리할 수 없는 형식(WMA, OGG, OPUS, APE, DSD)을 추가 지원하며, 오디오파일에게 출력 샘플 레이트, 멀티채널 라우팅, 피치 보정, IO 버퍼 지속 시간에 대한 제어권을 제공합니다. 이 FAQ는 사용자들이 가장 자주 보내는 질문에 답합니다. 더 심층적인 안내는 [Flacbox 사용자 가이드](/docs/guide/flacbox/)를 참조하거나 [사용 방법 문서](/docs/howto/)를 탐색하세요.

전체 음악 컬렉션을 **Dropbox, Google Drive, OneDrive, iCloud Drive, MEGA, Box, pCloud** 또는 개인 NAS와 같은 클라우드 서비스에 업로드하고 완전한 제어권을 가지고 클라우드에서 직접 음악을 재생할 수 있습니다. 오프라인으로 듣기 위해서는 내장 다운로드 관리자를 사용하여 기기에 음악을 저장하세요. 파일은 저장소에 그대로 저장됩니다 — Flacbox는 단순히 재생합니다.<br><br>

{{% details title="Flacbox는 무료인가요?" closed="true" %}}
**네 — Flacbox는 일부 제한이 있지만 무료로 사용할 수 있으며**, Premium 버전으로 업그레이드하면 제한을 해제할 수 있습니다.<br><br>

{{% details title="Flacbox와 Evermusic의 차이점은 무엇인가요?" closed="true" %}}
**Flacbox는 FFmpeg 기반 형식 지원, 설정 가능한 샘플 레이트, 멀티채널 출력, 피치 보정을 갖춘 오디오파일 중심 플레이어이며; Evermusic은 크로스페이드, 갭리스, 공간 오디오, 더 광범위한 클라우드 기능을 갖춘 일반 사용자용 플레이어입니다.**<br><br>

**Flacbox**는 모든 기본 iOS 오디오 형식과 함께 iPhone에서 기본적으로 지원되지 않는 많은 추가 형식(예: **WMA, OGG, OPUS, APE, DSD, DSF, DFF, TTA, MPC, WV** 등)을 지원하도록 구축되었습니다. 거의 모든 형식을 처리하기 위해 사용자 정의 오디오 엔진(시스템 코덱 + FFmpeg)을 사용하며, **조정 가능한 오디오 출력 샘플 레이트(44.1 – 96 kHz), 멀티채널 출력(Mono → 5.1 → SDDS), IO 버퍼 튜닝, 피치 보정(-1000 ~ +1000)**과 같은 오디오파일 수준의 기능을 제공합니다.<br><br>

**Evermusic**은 기본 Apple 오디오 형식을 지원하며 **크로스페이드 재생, 갭리스 재생, 공간 오디오, 오디오 이퀄라이저, 재생 속도 제어**와 같은 소비자 친화적 기능과 더 광범위한 클라우드 스트리밍 도구를 포함합니다. 주로 MP3, AAC, ALAC, FLAC를 사용하고 크로스페이드, 갭리스 또는 공간 오디오를 원한다면 **Evermusic**이 더 나은 선택일 수 있습니다. DSD 또는 WMA와 같은 더 광범위한 오디오 형식 호환성이 필요하고 오디오 출력 체인을 세밀하게 제어하고 싶다면 **Flacbox**가 올바른 선택입니다.<br><br>

{{% details title="Flacbox는 안전한가요?" closed="true" %}}
**네 — Flacbox는 연결된 클라우드 서비스와 상호 작용하기 위해 공식 SDK와 보안 OAuth 연결만 사용하며, 비밀번호를 볼 수 없으며, 암호화된 액세스 토큰만 iOS Keychain에 저장합니다.**<br><br>

로그인과 비밀번호를 입력하면 애플리케이션은 클라우드 서비스 제공업체가 제공한 공식 승인 페이지를 표시하며, 전체 승인 프로세스는 Flacbox 외부에서 진행됩니다. 그런 다음 클라우드 제공업체는 성공적인 승인 후 **인증 토큰**을 애플리케이션으로 전송하며, 해당 토큰은 API 호출에 사용됩니다.<br><br>

인증 토큰은 제3자 애플리케이션이 클라우드 스토리지와 상호 작용할 수 있게 해주는 디지털 키입니다. 토큰은 **Apple의 보안 시스템 스토리지(Keychain)**에 저장되며, 이는 저장 시 암호화되어 기기 패스코드 또는 생체 인식 잠금으로 보호됩니다. 다운로드된 파일은 앱의 **Documents** 디렉터리에 저장되며, 내장 파일 관리자를 사용하여 언제든지 제거할 수 있습니다.<br><br>

애플리케이션은 연결된 클라우드 계정의 **어떠한 정보도 Everappz, 광고주 또는 제3자와 공유하지 않습니다**. 웹 브라우저에서 계정 설정 페이지를 열어 언제든지 클라우드 계정에 대한 액세스를 취소할 수 있습니다. 인증 토큰을 취소하려면 웹에서 클라우드 계정에 로그인하고, 보안 또는 연결된 앱 페이지로 이동하여 Flacbox 항목을 제거하세요.<br><br>

애플리케이션 자체 내에서도 클라우드 계정을 연결 해제할 수 있으며 — 그렇게 하면 인증 토큰이 즉시 기기에서 삭제됩니다. 애플리케이션을 제거하면 다운로드된 모든 데이터와 토큰이 자동으로 삭제됩니다.<br><br>

{{% details title="Flacbox에서 재생 목록을 어떻게 만드나요?" closed="true" %}}
**재생 목록을 만들려면 재생 목록 탭을 열고, '+'또는 '...' 메뉴를 탭하고, '새 재생 목록'을 선택하고, 이름을 지정한 다음, 라이브러리, 이 애플리케이션의 파일, 이 기기의 파일, 또는 연결에서 추가할 트랙을 선택하세요.**<br><br>

- **재생 목록** 섹션을 여세요.<br>
- **+** 버튼이나 오른쪽 상단 모서리의 **'...'** 버튼을 탭하고 **새 재생 목록**을 선택하세요.<br>
- 재생 목록의 이름을 입력하고 **저장하다**를 탭하세요. **노래 추가** 대화 상자가 나타납니다.<br>
- 재생 목록에 추가할 트랙을 선택한 다음 **완료됨**을 탭하세요.<br><br>

기본적으로 각 곡은 재생 목록에 한 번만 추가될 수 있습니다. 중복 곡을 허용하려면 **설정 → 음악 라이브러리 → 재생 목록 → 재생 목록의 중복 → 활성화**를 활성화하세요. 나중에 노래 순서를 변경하고, 재생 목록의 이름을 바꾸고, 커버 이미지를 편집하고, ZIP으로 보관하고, M3U, CSV 또는 TXT로 내보낼 수 있습니다.<br><br>

{{% details title="Flacbox는 어떤 클라우드 서비스, 미디어 서버, 프로토콜을 지원하나요?" closed="true" %}}
**Flacbox는 하나의 클라우드 스토리지 연결 화면에서 거의 모든 인기 클라우드 스토리지 서비스, 자체 호스팅 미디어 서버, 파일 공유 프로토콜, S3 호환 오브젝트 스토어를 지원합니다.**<br><br>

**개인 클라우드 스토리지:** iCloud Drive · Google Drive · Dropbox · OneDrive (개인 및 비즈니스) · Box · MEGA · pCloud · Yandex Disk · WD My Cloud Home · MediaFire · TeraCLOUD (InfiniCLOUD) · HiDrive · IceDrive · Koofr · OpenDrive · MyDrive · Put.io · Cloud Mail.ru · Internxt · Proton Drive · AliDrive (阿里云盘) · Baidu Pan (百度网盘).<br><br>

**자체 호스팅 미디어 서버 및 음악 서버 API:** **Plex** · **Jellyfin** · **Emby** · **Subsonic** (모든 Subsonic 호환 서버 포함 — Airsonic, Funkwhale, Gonic, Logitech Media Server, Ampache) · **Navidrome** · Nextcloud (공유 API를 통한 ownCloud 포함) · **Synology Drive** · **QNAP**.<br><br>

**NAS 및 파일 공유 프로토콜:** **SMB** (SMB1, SMB2, 자동) · **WebDAV** (HTTP / HTTPS) · **FTP / FTPS** · **SFTP** (비밀번호 또는 공개 키 인증) · **NFS** · **DLNA / UPnP** (재생 및 다운로드). Apple Time Capsule, Synology, QNAP, WD My Cloud Home, Buffalo, Linux Samba / NFS / SSH 호스트, Mac 또는 Windows PC의 공유 폴더와 함께 작동합니다.<br><br>

**S3 호환 오브젝트 스토리지:** AWS S3, Backblaze B2, Wasabi, Cloudflare R2, MinIO, DigitalOcean Spaces, 및 기타 S3-API 엔드포인트.<br><br>

**로컬 네트워크 검색:** **사용 가능한 장치** 섹션은 Wi-Fi 네트워크의 모든 Bonjour / mDNS 서비스(Plex, Jellyfin, Emby 서버, Synology, QNAP, 연결된 드라이브가 있는 AirPort 라우터, Time Capsule)를 자동으로 나열하므로 IP 주소를 입력하지 않고도 탭하여 연결할 수 있습니다.<br><br>

각 연결은 서비스의 **공식 SDK 또는 개방형 프로토콜**을 사용하며, 지원되는 경우 OAuth 기반 인증을 사용합니다. 동일한 서비스의 여러 계정을 연결할 수 있습니다(예: 두 개의 Google Drive 계정, 개인 Dropbox와 업무용 Dropbox, 또는 Plex와 Jellyfin 서버 모두). Premium 사용자는 무제한 서비스를 추가할 수 있으며, 무료 버전은 3개로 제한됩니다.<br><br>

{{% details title="Flacbox에서 오디오 이퀄라이저를 어떻게 사용하나요?" closed="true" %}}
**오디오 플레이어 화면을 열고, 이퀄라이저 아이콘을 탭하고, 이퀄라이저를 켜고, 프리셋을 선택하거나 슬라이더를 이동하여 직접 만드세요.**<br><br>

- **오디오 플레이어** 화면을 여세요.<br>
- 화면 하단의 **이퀄라이저** 아이콘을 탭하세요.<br>
- 이퀄라이저 화면의 오른쪽 상단 모서리에 있는 스위치를 토글하여 활성화하세요.<br>
- 사용 가능한 프리셋 중 하나(Rock, Pop, Dance, Disco, Classical, Jazz, Acoustic, Bass Booster 등)를 선택하거나 밴드 슬라이더를 끌어 사용자 정의 프리셋을 만드세요.<br>
- **프리앰플리파이어**를 사용하여 전체 게인을 추가하거나 줄이세요. **저장하다**를 탭하여 원하는 이름으로 사용자 정의 프리셋을 유지하세요.<br><br>

Flacbox는 **10밴드 이퀄라이저**를 사용하며, EQ는 AirPlay, Chromecast, CarPlay 재생 중에도 사용할 수 있습니다. 전체 튜토리얼은 여기에 있습니다:<br>

{{% details title="Flacbox가 iPhone에서 FLAC을 재생하나요?" closed="true" %}}
**네 — Flacbox는 iPhone, iPad, Mac에서 FLAC 무손실 오디오를 기본으로 재생**하며, 임베디드 앨범 아트, 가사, 메타데이터를 완전히 지원합니다.<br><br>

FLAC 파일은 연결된 클라우드 스토리지에서 직접 스트리밍하거나 변환이나 트랜스코딩 없이 로컬에서 재생합니다. 매우 고해상도 파일에서 최상의 결과를 얻으려면 **오디오 코덱**을 **시스템 코덱 + FFmpeg**로 설정하거나 **설정 → 오디오 플레이어 → 일반**에서 **FFmpeg**를 선택하여 샘플 레이트와 피치 보정도 조정할 수 있습니다.
{{% /details %}}

{{% details title="Flacbox가 DSD, DSF, DFF 파일을 재생하나요?" closed="true" %}}
**네 — Flacbox는 번들 FFmpeg 엔진을 통해 DSD 계열 형식(DSF, DFF)을 재생합니다.** 대부분의 소비자 DAC는 Lightning / USB-C를 통해 기본 DSD를 수신하지 않으므로 DSD는 출력 전에 실시간으로 PCM으로 디코딩됩니다.<br><br>

DSD로 최상의 결과를 얻으려면 **설정 → 오디오 플레이어 → 일반**에서 **오디오 코덱**을 **FFmpeg**로 전환하고 DAC에 맞는 **88.2 kHz** 또는 **96 kHz**의 **오디오 출력 샘플 레이트**를 선택하세요. 완전히 투명한 디코딩을 위해 Flacbox와 USB 또는 Lightning DAC를 페어링하세요.
{{% /details %}}

{{% details title="Flacbox는 Hi-Res 오디오(24비트 / 96kHz)를 지원하나요?" closed="true" %}}
**네 — Flacbox는 FFmpeg 엔진을 통해 Hi-Res 오디오 재생을 지원하며, 44.1, 48, 64, 88.2, 96 kHz의 설정 가능한 출력 샘플 레이트를 제공합니다.**<br><br>

**설정 → 오디오 플레이어 → 일반**에서 **오디오 코덱**을 **FFmpeg**로 전환한 다음 원하는 **오디오 출력 샘플 레이트**와 채널 수를 선택하세요. 호환되는 **외부 DAC 또는 USB / Lightning 오디오 인터페이스**와 페어링하여 iPhone, iPad 또는 Mac에서 비트 퍼펙트 출력을 얻으세요.
{{% /details %}}

{{% details title="Flacbox에서 외부 USB DAC를 어떻게 사용하나요?" closed="true" %}}
**iPhone, iPad 또는 Mac에 USB DAC를 연결(Lightning-to-USB, USB-C 또는 직접 연결)한 다음, 설정 → 오디오 플레이어 → 일반에서 FFmpeg 코덱으로 전환하고 DAC가 기대하는 샘플 레이트와 채널 수를 선택하세요.**<br><br>

많은 오디오파일 DAC는 **스테레오** 출력으로 **88.2 kHz** 또는 **96 kHz**를 선호합니다. 저지연을 위해 **오디오 출력 선호 IO 버퍼 지속 시간**을 약 **5 ms (0.005 s)**로 낮춘 다음 끊김 현상이 들리면 약간 높이세요. Flacbox는 AudioQuest, iFi, Chord, FiiO, Topping 등의 대부분의 클래스 호환 USB DAC 및 Lightning DAC와 함께 작동합니다.
{{% /details %}}

{{% details title="Flacbox가 WMA, OGG, OPUS, APE 파일을 재생하나요?" closed="true" %}}
**네 — Flacbox는 iOS가 기본적으로 지원하지 않더라도 번들 FFmpeg 엔진을 통해 WMA, OGG, OPUS, APE, TTA, MPC, WV 및 기타 덜 일반적인 형식들을 재생합니다.**<br><br>

변환이 필요 없습니다: 파일을 iCloud Drive, Dropbox, Google Drive 또는 NAS에 저장하고, Flacbox에서 해당 서비스를 연결하면 파일이 바로 재생됩니다. 임베디드 태그, 앨범 아트, 가사는 있는 경우 읽힙니다.

{{% details title="Flacbox에서 노래, 앨범 또는 아티스트를 어떻게 검색하나요?" closed="true" %}}
**음악 라이브러리, 재생 목록, 앨범, 아티스트, 장르 또는 폴더 내의 목록에서 돋보기 아이콘을 탭하고, 이름을 입력하면 결과가 즉시 필터링됩니다.**<br><br>

검색은 로컬이며 라이브러리 데이터베이스에 대해 완전히 오프라인으로 작동하므로 느린 네트워크에서도 입력하는 동안 결과가 나타납니다. 특정 재생 목록, 앨범 또는 폴더 내에서도 검색하여 수백 개 중에서 단일 트랙을 찾을 수 있습니다. 트랙, 앨범, 아티스트, 장르, 작곡가, 폴더 모두 검색 가능합니다.
{{% /details %}}

{{% details title="Flacbox가 Bluetooth 헤드폰, AirPods, 외부 DAC와 함께 작동하나요?" closed="true" %}}
**네 — Flacbox는 iPhone, iPad 또는 Mac이 사용할 수 있는 모든 오디오 출력을 통해 재생합니다: 내장 스피커, 유선 헤드폰, Bluetooth 기기(AirPods, Beats, Sony, Bose 등), USB / Lightning DAC.**<br><br>

외부 DAC로의 Hi-Res 출력을 위해 **설정 → 오디오 플레이어 → 일반**에서 오디오 코덱을 **FFmpeg**로 전환하고 적절한 **오디오 출력 샘플 레이트**와 채널 수를 선택하세요. AirPods의 스템 컨트롤(재생 / 일시 정지, 다음 / 이전)은 예상대로 작동하며, 잠금 화면 컨트롤은 **설정 → 오디오 플레이어 → 개인화 → 잠금 화면의 재생 컨트롤**에서 사용자 정의할 수 있습니다.
{{% /details %}}

{{% details title="iPhone 또는 iPad에서 Flacbox로 USB 플래시 드라이브 또는 SD 카드를 어떻게 사용하나요?" closed="true" %}}
**Lightning-to-USB 또는 USB-C 어댑터를 통해 드라이브를 iPhone 또는 iPad에 연결한 다음, Flacbox에서 로컬 파일 → 이 iPhone의 파일 → 폴더 열기를 열고, 드라이브로 이동하여 음악 폴더를 선택하세요.**<br><br>

Flacbox는 파일을 내부 스토리지에 복사하지 않고 드라이브에서 직접 재생하며, 이는 매우 대용량 무손실 라이브러리에 유용합니다. 같은 방식으로 음악 라이브러리에 트랙을 가져오거나 재생 목록에 추가할 수도 있습니다. 앱은 **Apple Certified 카드 리더기**와 **iXpand 플래시 드라이브**를 지원합니다 — iXpand의 경우 Lightning 포트에 삽입하면 드라이브가 연결 화면에 자동으로 나타납니다. [iPhone에 USB 플래시 드라이브를 연결하고 음악을 듣는 방법](/docs/howto/how-to-connect-a-usb-flashcard-to-the-iphone-and-listen-to-music-or-manage-files-located-on-it)에서 전체 단계별 지침을 확인하세요.
{{% /details %}}

{{% details title="Flacbox에서 클라우드 음악이 동기화되지 않는 이유는 무엇인가요?" closed="true" %}}
**대부분의 동기화 문제는 만료된 인증 토큰, 앱 백그라운드 처리, 또는 활성 인터넷 연결 없음으로 인해 발생합니다 — 연결을 열고, 서비스를 다시 인증한 다음, 설정 → 라이브러리 → 온라인 동기화에서 수동으로 동기화를 실행하세요.**<br><br>

온라인 동기화는 앱이 포그라운드에 있을 때만 실행되므로 대용량 라이브러리를 동기화하는 데 시간이 걸릴 수 있습니다. 속도를 높이려면 Flacbox를 열어 두고, 기기를 전원에 연결하고, **설정 → 화면 → 항상 활성화**를 활성화하세요. 매우 대용량 라이브러리의 경우 앱의 데스크톱 버전에서 동기화를 실행하고 **백업 및 복원**을 사용하여 결과를 iOS로 전송하세요.
{{% /details %}}

Evervideo **iPhone, iPad और Mac** के लिए एक HD और 4K **क्लाउड वीडियो प्लेयर** है जो iCloud Drive, Google Drive, Dropbox, OneDrive, MEGA, Box, pCloud, Yandex Disk, Synology, QNAP और किसी भी WebDAV, SMB, NFS, FTP / SFTP, DLNA या S3 स्रोत से वीडियो स्ट्रीम और डाउनलोड करता है — साथ ही **Plex, Jellyfin, Emby, Subsonic** और **Navidrome** जैसे सेल्फ-होस्टेड मीडिया सर्वर, और IP कैमरों से लाइव **RTSP** स्ट्रीम। कस्टम **FFmpeg-आधारित प्लेयर इंजन** के साथ **हार्डवेयर-त्वरित H.264 / HEVC डिकोडिंग**, प्राथमिक और द्वितीयक सब्टाइटल ट्रैक (SRT / VTT / ASS / libass), प्रीसेट के साथ वीडियो इक्वलाइज़र, ऑडियो इक्वलाइज़र, **Picture-in-Picture**, Insta360 और GoPro Max फुटेज के लिए **360° / VR प्लेबैक**, **AirPlay 2** और **Chromecast** कास्टिंग, और हमेशा-ऑन-स्क्रीन कॉम्पैक्ट प्लेयर पर बनाया गया। यह FAQ उन सवालों के जवाब देता है जो उपयोगकर्ता सबसे अधिक भेजते हैं। गहरी जानकारी के लिए [Evervideo उपयोगकर्ता गाइड](/docs/guide/evervideo/) देखें।

{{% details title="Evervideo क्या है?" closed="true" %}}
**Evervideo iPhone, iPad और Mac के लिए एक पूर्ण-सुविधा संपन्न HD और 4K क्लाउड वीडियो प्लेयर है** जो किसी भी क्लाउड स्टोरेज अकाउंट, NAS या मीडिया सर्वर को आपकी फ़ाइलों पर पूर्ण नियंत्रण के साथ आपकी व्यक्तिगत वीडियो लाइब्रेरी में बदल देता है।<br><br>

हार्डवेयर-त्वरित H.264 और HEVC डिकोडिंग के साथ कस्टम FFmpeg-आधारित प्लेयर इंजन पर बनाया गया, Evervideo लगभग हर आधुनिक कंटेनर और कोडेक (MP4, MKV, AVI, MOV, FLV, WMV, WebM, TS, M2TS, और FFmpeg फॉर्मेट की लंबी सूची) चलाता है, प्राथमिक और द्वितीयक सब्टाइटल का समर्थन करता है, ऑडियो और वीडियो इक्वलाइज़र शामिल करता है, और Picture-in-Picture मोड में काम करता है ताकि आप अन्य ऐप्स का उपयोग करते हुए देखना जारी रख सकें।
{{% /details %}}

{{% details title="क्या Evervideo मुफ़्त है? Free और Premium में क्या अंतर है?" closed="true" %}}
**हां — Evervideo डाउनलोड और उपयोग करने के लिए मुफ़्त है**, सीमाएं हटाने और पूर्ण Premium फीचर सेट अनलॉक करने के लिए वैकल्पिक इन-ऐप खरीदारी के साथ।<br><br>

Premium **एकमुश्त आजीवन खरीद** या **मासिक या वार्षिक सब्सक्रिप्शन** के रूप में उपलब्ध है, इसलिए आप वह विकल्प चुन सकते हैं जो आपको सबसे अच्छा लगे। कीमतें देश के अनुसार भिन्न हो सकती हैं। **Family Sharing** हर प्लान के लिए सक्षम है, इसलिए आप अपने परिवार के पांच सदस्यों तक Premium साझा कर सकते हैं। आजीवन खरीद और सब्सक्रिप्शन iCloud के माध्यम से iOS और macOS के बीच साझा किए जाते हैं — प्रत्येक डिवाइस पर नवीनतम संस्करण इंस्टॉल करें, एक ही Apple ID से साइन इन करें, और खरीद जानकारी सिंक होने के लिए लगभग एक मिनट प्रतीक्षा करें।<br><br>

**Evervideo Premium** मुफ़्त संस्करण की हर सीमा हटाता है: विज्ञापन-मुक्त प्लेबैक, असीमित प्लेलिस्ट, असीमित कनेक्टेड क्लाउड सेवाएं, असीमित पसंदीदा, असीमित ऑफ़लाइन डाउनलोड, मीडिया संग्रह का आर्काइव (ZIP), और पूर्ण वैयक्तिकरण (कस्टम ऐप आइकन, थीम, रंग योजना)।<br><br>

{{% details title="क्या Evervideo सुरक्षित है?" closed="true" %}}
Evervideo कनेक्टेड क्लाउड सेवाओं के साथ इंटरेक्ट करने के लिए केवल आधिकारिक SDK और सुरक्षित कनेक्शन का उपयोग करता है। आपका लॉगिन और पासवर्ड एप्लिकेशन के लिए उपलब्ध नहीं हैं। एप्लिकेशन से क्लाउड सेवा तक सभी अनुरोध एन्क्रिप्टेड हैं।<br>
जब आप लॉगिन और पासवर्ड दर्ज करते हैं तो एप्लिकेशन आपको क्लाउड सेवा प्रदाता द्वारा प्रदान किया गया आधिकारिक प्राधिकरण पृष्ठ दिखाता है और सभी प्राधिकरण प्रक्रिया एप्लिकेशन के बाहर होती है। क्लाउड सेवा प्रदाता सफल प्राधिकरण के बाद एप्लिकेशन को एक auth-token भेजता है और उस token का उपयोग API कॉल करने के लिए किया जाता है।<br><br>

Auth-token एक डिजिटल कुंजी है जो तृतीय-पक्ष एप्लिकेशन को क्लाउड स्टोरेज के साथ इंटरेक्ट करने की अनुमति देती है। Auth-token आपके डिवाइस पर Keychain नामक सुरक्षित सिस्टम स्टोरेज में संग्रहीत है। आप कनेक्टेड क्लाउड सेवा से अपनी फ़ाइलें डिवाइस पर डाउनलोड कर सकते हैं और वे फ़ाइलें ऐप "दस्तावेज़" डायरेक्टरी में रखी जाएंगी। आप बिल्ट-इन फ़ाइल मैनेजर का उपयोग करके उन फ़ाइलों को कभी भी हटा सकते हैं।<br>
एप्लिकेशन कनेक्टेड क्लाउड अकाउंट से कोई भी जानकारी साझा नहीं करता है। आप अपने वेब ब्राउज़र पर अकाउंट सेटिंग्स पृष्ठ खोलकर किसी भी समय अपने क्लाउड अकाउंट तक पहुंच रद्द कर सकते हैं।<br><br>

auth-token को अस्वीकार करने के लिए अपने वेब ब्राउज़र पर अपने अकाउंट में लॉगिन करें और सेटिंग्स पृष्ठ पर नेविगेट करें। वहां आप अपने क्लाउड अकाउंट से जुड़े सभी तृतीय-पक्ष ऐप्स पा सकते हैं और यदि आप उस एप्लिकेशन का उपयोग नहीं करना चाहते हैं तो उनमें से किसी को भी हटा सकते हैं।<br><br>

एक पूर्ण ट्यूटोरियल यहां उपलब्ध है:<br>
[अपने Google अकाउंट से तृतीय-पक्ष ऐप को कैसे डिस्कनेक्ट करें](/docs/howto/how-to-disconnect-third-party-app-from-your-google-account/)<br><br>

आप एप्लिकेशन में कनेक्टेड क्लाउड अकाउंट को भी डिस्कनेक्ट कर सकते हैं और auth-token भी आपके डिवाइस से हटा दिया जाएगा। यदि आप अपने डिवाइस से एप्लिकेशन हटाते हैं तो सभी डाउनलोड किया गया डेटा और एक्सेस token भी हटा दिए जाएंगे।<br><br>

{{% details title="Evervideo कौन से वीडियो फॉर्मेट सपोर्ट करता है?" closed="true" %}}
**Evervideo iPhone, iPad और Mac पर लगभग हर आधुनिक वीडियो कंटेनर और कोडेक चलाता है** बंडल किए गए FFmpeg इंजन और हार्डवेयर-त्वरित H.264 / HEVC डिकोडिंग के संयोजन के कारण।<br><br>

**कंटेनर:** MP4, M4V, MKV, MOV, AVI, FLV, WMV, ASF, WebM, TS, M2TS, MTS, MPG, MPEG, OGV, 3GP, 3G2, F4V, RM, RMVB, VOB, DAT, और कई अन्य।<br>
**वीडियो कोडेक:** H.264 (AVC), H.265 (HEVC), VP9, VP8, AV1, MPEG-2, MPEG-4, MJPEG, ProRes, Theora, WMV, और FFmpeg द्वारा समर्थित लगभग हर अन्य कोडेक।<br>
**ऑडियो कोडेक (वीडियो के अंदर):** AAC, MP3, FLAC, ALAC, OGG / Vorbis, OPUS, AC-3, EAC-3, DTS, AMR, WMA, APE।<br>
**सब्टाइटल:** SRT, VTT (WebVTT), ASS / SSA (libass के माध्यम से रेंडर किया गया), एम्बेडेड इमेज और टेक्स्ट ट्रैक।<br>
**स्ट्रीमिंग प्रोटोकॉल:** HTTP / HTTPS, HLS (m3u8), RTSP (IP कैमरे और IPTV)।<br>
**डायरेक्ट स्ट्रीमिंग:** SMB / WebDAV / FTP / SFTP / NFS / DLNA।<br>
{{% /details %}}

{{% details title="क्या Evervideo iPhone पर MKV फ़ाइलें चलाता है?" closed="true" %}}
**हां — Evervideo iPhone, iPad और Mac पर MKV फ़ाइलें नेटिव रूप से चलाता है** बिना किसी कनवर्जन के, जिसमें H.264, HEVC, VP9 या AV1 वीडियो, एकाधिक ऑडियो ट्रैक, और एम्बेडेड SRT / ASS सब्टाइटल वाले MKV कंटेनर शामिल हैं।<br><br>

यह मुख्य कारणों में से एक है कि लोग Apple के डिफ़ॉल्ट प्लेयर की बजाय Evervideo इंस्टॉल करते हैं: iOS MKV बिल्कुल नहीं खोलता है, लेकिन Evervideo बंडल किए गए FFmpeg इंजन के माध्यम से उन्हें हैंडल करता है। आप डाउनलोड किए बिना क्लाउड स्टोरेज या NAS से सीधे MKV फ़ाइलें स्ट्रीम कर सकते हैं।
{{% /details %}}

{{% details title="क्या Evervideo 4K और HDR वीडियो सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo हर उस डिवाइस पर 4K (Ultra HD) वीडियो और HDR-एन्कोडेड सामग्री चलाता है** जिसके पास इसे डिकोड करने का हार्डवेयर है। आधुनिक iPhone, iPad और Apple Silicon Mac सभी हार्डवेयर में 4K H.264 और 4K HEVC डिकोड करते हैं, इसलिए प्लेबैक स्मूद और बैटरी-अनुकूल है।<br><br>

क्लाउड-स्ट्रीम किए गए 4K पर सर्वोत्तम परिणामों के लिए, **सेटिंग्स → प्लेयर → फ़ाइल लोडिंग** में **Preloading Time** बढ़ाएं ताकि बफर उच्च-बिटरेट फ़ाइलों के साथ बना रहे, और सेलुलर की बजाय Wi-Fi नेटवर्क से कनेक्ट करें।
{{% /details %}}

{{% details title="क्या Evervideo हार्डवेयर H.264 और HEVC डिकोडिंग सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo डिफ़ॉल्ट रूप से हर iPhone, iPad और Mac पर हार्डवेयर H.264 (AVC) और H.265 (HEVC) डिकोडर का उपयोग करता है** जो उन्हें सपोर्ट करते हैं, जिसका अर्थ है शुद्ध सॉफ़्टवेयर डिकोडिंग की तुलना में स्मूदर प्लेबैक, कम बैटरी उपयोग और ठंडा डिवाइस तापमान।<br><br>

आप **सेटिंग्स → प्लेयर → वीडियो → Hardware Decode H.264 / H.265** में H.264 और HEVC के लिए स्वतंत्र रूप से हार्डवेयर डिकोडिंग टॉगल कर सकते हैं। यदि किसी विशिष्ट फ़ाइल में संगतता समस्याएं हैं (दूषित स्ट्रीम, विदेशी प्रोफाइल), FFmpeg सॉफ़्टवेयर डिकोडिंग पर वापस जाने के लिए उस फ़ाइल के लिए हार्डवेयर डिकोडिंग अक्षम करें।
{{% /details %}}

{{% details title="क्या Evervideo Picture-in-Picture (PiP) सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo iPhone और iPad पर Picture-in-Picture को पूरी तरह से सपोर्ट करता है।** जब आप प्लेयर पर PiP आइकन टैप करते हैं, तो वीडियो हर दूसरे ऐप के ऊपर एक फ्लोटिंग विंडो में चलता रहता है।<br><br>

{{% details title="वीडियो इक्वलाइज़र और प्रीसेट का उपयोग कैसे करें?" closed="true" %}}
**प्लेयर खोलें, अधिक क्रियाएँ → वीडियो इक्वलाइज़र टैप करें, फिर brightness, contrast, saturation और hue के लिए स्लाइडर खींचें — या एक प्रीसेट चुनें।**<br><br>

Evervideo **वीडियो इक्वलाइज़र** एक रियल-टाइम पिक्चर एडजस्टमेंट टूल है जो FFmpeg रेंडरिंग पाइपलाइन के अंदर चलता है:<br>

- **Brightness** — कम रोशनी में शूट किए गए अंधेरे दृश्य को बूस्ट करें, या blown-out exposure को वापस खींचें।<br>
- **Contrast** — धुले हुए content में पंच जोड़ें या over-contrasty source को सॉफ्ट करें।<br>
- **Saturation** — म्यूट रंगों को गर्म करें या सिनेमाटिक लुक के लिए desaturate करें।<br>
- **Hue** — हरे या मैजेंटा cast को ठीक करने के लिए रंग संतुलन शिफ्ट करें।<br><br>

आप अपनी पसंदीदा सेटिंग्स को **कस्टम प्रीसेट** के रूप में सहेज सकते हैं और इसे किसी भी भविष्य के वीडियो पर एक टैप से पुनः लागू कर सकते हैं। प्रीसेट को **एक्सपोर्ट और इम्पोर्ट** भी किया जा सकता है ताकि आप उन्हें iPhone, iPad और Mac के बीच साझा कर सकें, या उनका बैकअप ले सकें। चित्र और ध्वनि दोनों पर पूर्ण नियंत्रण के लिए वीडियो इक्वलाइज़र को **ऑडियो इक्वलाइज़र** (अपनी प्रीसेट लाइब्रेरी के साथ 10-बैंड EQ) के साथ मिलाएं।

{{% details title="वीडियो को कैसे रोटेट करें या स्केलिंग मोड कैसे बदलें?" closed="true" %}}
**प्लेयर पर अधिक क्रियाएँ टैप करें और Rotation (0° / 90° / 180° / 270°) या Scaling Mode (Fit / Fill / Stretch / Original) चुनें।**<br><br>

**Rotation** उन वीडियो के लिए उपयोगी है जो किनारे या उल्टे रिकॉर्ड किए गए हैं — प्लेयर छोड़े बिना चित्र को घुमाएं। **Scaling Mode** नियंत्रित करता है कि चित्र स्क्रीन को कैसे भरता है:<br>

- **Fit** — जरूरत पड़ने पर काले बार के साथ मूल आस्पेक्ट रेशियो संरक्षित करें।<br>
- **Fill** — जरूरत पड़ने पर वीडियो क्रॉप करके पूरी स्क्रीन भरें।<br>
- **Stretch** — स्क्रीन भरने के लिए खिंचाव, चित्र को विकृत करना।<br>
- **Original** — 1:1 पर नेटिव रेज़ोल्यूशन रखें।

{{% details title="Evervideo में सब्टाइटल कैसे जोड़ें या बदलें?" closed="true" %}}

Evervideo स्वचालित रूप से वीडियो में एम्बेडेड हर सब्टाइटल ट्रैक को सूचीबद्ध करता है। एक external subtitle फ़ाइल लोड करने के लिए, **External File** चुनें और अपने डिवाइस, iCloud Drive, या किसी कनेक्टेड क्लाउड सेवा से `.srt`, `.vtt`, `.ass`, या `.ssa` फ़ाइल चुनें। आप **सेटिंग्स → प्लेयर → Subtitles** में डिफ़ॉल्ट सब्टाइटल व्यवहार भी कॉन्फ़िगर कर सकते हैं।

{{% details title="क्या Evervideo बाहरी SRT, VTT और ASS सब्टाइटल फ़ाइलों को सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo आपके डिवाइस पर या किसी भी कनेक्टेड क्लाउड सेवा से SRT, VTT (WebVTT), ASS और SSA फॉर्मेट में external subtitle फ़ाइलें लोड करता है।**<br><br>

{{% details title="Evervideo से वीडियो कैसे डिलीट करें बिना क्लाउड स्टोरेज से डिलीट किए?" closed="true" %}}
**किसी भी वीडियो पर "..." → Delete from Media Library टैप करें — यह आपके लाइब्रेरी डेटाबेस से प्रविष्टि हटाता है लेकिन क्लाउड स्टोरेज, आपके NAS, या iOS Photos लाइब्रेरी में मूल फ़ाइल को अछूता छोड़ता है।**<br><br>

क्लाउड में कुछ भी छुए बिना डाउनलोड की गई प्रति को हटाने के लिए, **फ़ाइलें** टैब खोलें, **Files in This Application** या **Offline Folders** के अंतर्गत वीडियो ढूंढें, और **"..." → Delete** का उपयोग करें। क्लाउड ओरिजिनल ठीक वहीं रहता है जहां था — Evervideo केवल स्थानीय प्रति हटाता है।

यदि आप एक cloud-connected ऐप चाहते हैं जो CarPlay सपोर्ट करे, तो हमारे संगीत ऐप **Evermusic** और **Flacbox** दोनों Library, Connections, Local Files और Settings के लिए समर्पित टैब के साथ CarPlay पर पूरी तरह से काम करते हैं।

Premium उपयोगकर्ता असीमित सेवाओं से जुड़ सकते हैं; मुफ़्त संस्करण तीन तक सीमित है।

{{% details title="क्या Evervideo Plex, Jellyfin, Emby, Subsonic और Navidrome सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo Plex Media Server, Jellyfin, Emby, Subsonic और Navidrome से नेटिव रूप से जुड़ता है**, इसलिए आप अंतर्निहित फ़ाइल शेयर को expose किए बिना अपनी सेल्फ-होस्टेड वीडियो लाइब्रेरी को सीधे स्ट्रीम कर सकते हैं।<br><br>

- **Plex Media Server** — **फ़ाइलें → संपर्क → Plex** टैप करें, अपने Plex अकाउंट से साइन इन करें, और एक सर्वर चुनें। उसी स्थानीय नेटवर्क पर Plex सर्वर **उपलब्ध उपकरण** अनुभाग में auto-discovered भी हैं।<br>
- **Jellyfin** (open-source) — **फ़ाइलें → संपर्क → Jellyfin** टैप करें, अपना सर्वर URL (जैसे `http://server-ip:8096`) और credentials दर्ज करें।<br>
- **Emby** (commercial) — **फ़ाइलें → संपर्क → Emby** टैप करें और सर्वर URL और लॉगिन दर्ज करें।<br>
- **Subsonic और Subsonic-compatible servers** — **फ़ाइलें → संपर्क → Subsonic** टैप करें, सर्वर URL और credentials दर्ज करें। वही API path **Navidrome**, **Airsonic**, **Funkwhale**, **Gonic**, **Logitech Media Server (LMS)**, और **Ampache** के साथ काम करता है।<br><br>

एक बार कनेक्ट होने पर, प्रत्येक सर्वर Files टैब में आपके क्लाउड अकाउंट के साथ दिखाई देता है। आप Movies, TV Shows, Home Videos, Music, Playlists, और संग्रह ब्राउज़ कर सकते हैं; ऑफ़लाइन प्लेबैक के लिए डाउनलोड कर सकते हैं; प्लेयर में आइटम queue कर सकते हैं; और उन्हें अपनी वैश्विक Media Library में खींच सकते हैं — सभी Evervideo छोड़े बिना।
{{% /details %}}

{{% details title="क्या Evervideo SMB, WebDAV, FTP / SFTP, NFS और DLNA के माध्यम से कनेक्ट होता है?" closed="true" %}}
**हां — Evervideo हर प्रमुख NAS और file-share प्रोटोकॉल को सपोर्ट करता है: SMB (SMB1, SMB2, Auto), WebDAV (HTTP / HTTPS), FTP / FTPS, SFTP (पासवर्ड या public-key auth), NFS, और DLNA / UPnP।**<br><br>

यह आपको लगभग किसी भी NAS डिवाइस (Synology, QNAP, WD My Cloud Home, Buffalo, Apple Time Capsule), Linux / macOS / Windows file share, सेल्फ-होस्टेड Nextcloud / ownCloud सर्वर, या किसी भी UPnP / DLNA मीडिया सर्वर से **फ़ाइलें → संपर्क** मेनू से कनेक्ट करने देता है।
{{% /details %}}

{{% details title="क्या Evervideo S3-संगत object storage सपोर्ट करता है?" closed="true" %}}
**हां — Evervideo में एक S3-compatible connector शामिल है** जो **AWS S3, Backblaze B2, Wasabi, Cloudflare R2, MinIO, DigitalOcean Spaces** और किसी भी अन्य सेवा के साथ काम करता है जो S3-API endpoint expose करती है।<br><br>

**फ़ाइलें → संपर्क → S3 storage** टैप करें, फिर endpoint URL, region, access key, secret key और bucket name दर्ज करें। एक बार कनेक्ट होने पर, bucket किसी भी अन्य क्लाउड की तरह व्यवहार करता है — ब्राउज़, स्ट्रीम, डाउनलोड करें, queue करें और अपनी लाइब्रेरी में जोड़ें।
{{% /details %}}

{{% details title="क्या Evervideo में RTSP स्ट्रीम (IP कैमरे, IPTV) चला सकते हैं?" closed="true" %}}
**हां — Evervideo में native RTSP support है**, इसलिए आप इसे किसी भी `rtsp://` URL पर point कर सकते हैं — security cameras, doorbell cameras, baby monitors, IPTV providers, broadcast feeds — और Evervideo live stream को pull और decode करेगा।<br><br>

**फ़ाइलें → Online Links → Add link** टैप करें, पूरा URL (`rtsp://camera-ip:port/stream-path`) paste करें, यदि आवश्यक हो तो लॉगिन और पासवर्ड दें, और **पूर्ण करना** टैप करें। RTSP स्ट्रीम Picture-in-Picture में, compact player में काम करती हैं, और वे AirPlay 2 और Chromecast पर regular video की तरह ही cast होती हैं।

{{% details title="अपने कंप्यूटर से Evervideo में वीडियो कैसे ट्रांसफर करें?" closed="true" %}}
आप SMB, WebDAV, या DLNA प्रोटोकॉल का उपयोग करके अपने कंप्यूटर या व्यक्तिगत NAS को कनेक्ट कर सकते हैं। वैकल्पिक रूप से, मीडिया फ़ाइलें ट्रांसफर करने के लिए iTunes File Sharing का उपयोग करें।<br><br>

SMB प्रोटोकॉल का उपयोग करके कंप्यूटर कनेक्ट करने के लिए "फ़ाइलें" "संपर्क" → SMB टैप करें। URL फ़ील्ड में smb://computer-ip-address/shared-folder-name फॉर्मेट का उपयोग करके कंप्यूटर IP address और shared folder name दर्ज करें, लॉगिन और पासवर्ड दर्ज करें और "पूर्ण करना" टैप करें। यदि आपका कनेक्शन सफल है तो आप "Cloud storage" सेक्शन में connected storage देखेंगे।<br><br>

एक पूर्ण ट्यूटोरियल यहां उपलब्ध है:<br>
[SMB प्रोटोकॉल का उपयोग करके कंप्यूटर से iPhone पर अपनी फ़ाइलें ट्रांसफर करें](/docs/howto/transfer-your-files-from-the-computer-to-iphone-using-smb-protocol/)<br><br>

WebDAV प्रोटोकॉल के लिए, सभी चरण URL फ़ील्ड को छोड़कर समान हैं। URL http://server-name फॉर्मेट में होना चाहिए या https://server-name यदि सर्वर SSL सपोर्ट करता है।<br><br>

एक पूर्ण ट्यूटोरियल यहां उपलब्ध है:<br>

Wi-Fi Drive एक लोकप्रिय तकनीक है जो आपको desktop browser का उपयोग करके कंप्यूटर से iOS डिवाइस पर वायरलेस तरीके से फ़ाइलें ट्रांसफर करने की अनुमति देती है। इस सुविधा का उपयोग करने के लिए आपका डिवाइस और कंप्यूटर एक ही Wi-Fi नेटवर्क से जुड़े होने चाहिए। "फ़ाइलें" → "Computer" → "Connect using Wi-Fi" खोलें और server सक्षम करें। इसके बाद एक desktop browser खोलें और एप्लिकेशन से URL दर्ज करें। आप अपने कंप्यूटर से खुले वेब पेज पर फ़ाइलें drag-n-drop कर सकते हैं और वे डिवाइस पर दिखाई देंगी।<br>

{{% details title="Evervideo में Wi-Fi Drive सुविधा का उपयोग कैसे करें?" closed="true" %}}

**desktop browser का उपयोग करके वायरलेस ट्रांसफर**<br>
1. ऐप लॉन्च करें: Evervideo खोलें।<br>
2. Wi-Fi के माध्यम से कनेक्ट करें: फ़ाइलें → "Computer" → "Connect via Wi-Fi" पर जाएं।<br>
3. (वैकल्पिक) सुरक्षा जोड़ें: यदि आवश्यक हो तो username और password दर्ज करें।<br>
4. Wi-Fi Drive शुरू करें: "Start Wi-Fi Drive" टैप करें और दिया गया URL कॉपी करें।<br>
5. एक browser (Safari, Chrome, Opera, Firefox, आदि) खोलें और URL दर्ज करें।<br>
6. फ़ाइलें upload, download, rename, या delete करने के लिए built-in file manager का उपयोग करें। आप अपने iPhone पर सीधे फ़ाइलें drag और drop कर सकते हैं।<br>
7. जब हो जाए, अपने iPhone पर "Stop Wi-Fi Drive" टैप करें।<br><br>

नोट: सर्वोत्तम प्रदर्शन के लिए सुनिश्चित करें कि JavaScript सक्षम है और आप नवीनतम browser version का उपयोग कर रहे हैं।<br><br>

**Mac Finder का उपयोग करके फ़ाइलें ट्रांसफर करें**<br>
1. Finder में, "Go" → "Connect to Server…" क्लिक करें।<br>
2. Evervideo ऐप में दिखाया गया server URL दर्ज करें।<br>
3. "Connect" क्लिक करें और अपने iPhone पर फ़ाइलें किसी भी network drive की तरह manage करें।<br><br>

**Windows File Explorer का उपयोग करके फ़ाइलें ट्रांसफर करें**<br>
1. "This PC" पर राइट-क्लिक करें → "Map Network Drive…"।<br>
2. "Folder" फ़ील्ड में Evervideo ऐप का server URL दर्ज करें।<br>
3. एक drive letter चुनें, "Finish" क्लिक करें, और अपने iPhone की फ़ाइलें सीधे ब्राउज़ करें।<br><br>

Evervideo आपके द्वारा देखे गए हर वीडियो की प्लेबैक position को track करता है, इसलिए वे वीडियो जिन्हें आपने स्पष्ट रूप से bookmark नहीं किया वे भी ठीक वहीं से resume किए जा सकते हैं जहां आपने रोका था। **सेटिंग्स → Media Library → Recents → Change List Size** के माध्यम से Recents list कितनी entries रखती है यह बदलें। आप अपने watch history का बैकअप लेने के लिए सूची को M3U, CSV, या TXT में export भी कर सकते हैं, या fresh start के लिए **Delete List** से इसे clear कर सकते हैं।

{{% details title="मेरा cloud video buffering या stuttering क्यों हो रहा है?" closed="true" %}}
**Buffering लगभग हमेशा slow network, बड़े file sizes, या कम player buffer settings के कारण होती है — Preloading Time बढ़ाएं और एक faster network पर switch करें या file download करें।**<br><br>

कुछ व्यावहारिक tips:<br>

- High-bitrate 4K या HEVC content के लिए **सेटिंग्स → प्लेयर → File Loading → Preloading Time** में **buffer बढ़ाएं**।<br>
- यदि आप cellular पर हैं तो **Wi-Fi पर switch करें**, या अपने router के करीब जाएं।<br>
- H.264 और HEVC के लिए **सेटिंग्स → प्लेयर → वीडियो** में **Hardware Decode सक्षम करें** ताकि CPU bottleneck न बने।<br>
- यदि source आपकी viewing speed से धीमा है तो **offline playback के लिए file download करें** — बड़ी 4K files easily cellular bandwidth से अधिक हो सकती हैं।<br>
- यदि कोई connection expire हो गया हो तो Files टैब में **अपने cloud account को re-authorize करें**।

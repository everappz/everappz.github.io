- **快速排序** — 在专辑、艺人、播放列表和文件夹中,几秒钟内就能找到任意一首曲目。
- **多种配色方案** — 选择与仪表盘或车内灯光相搭配的主题。
- **重新设计的「正在播放」屏幕** — 更大的封面、更清晰的控制按钮,以及一指即可触发的新播放操作。

如果你在驾车时从云端播放音乐 — Dropbox、Google Drive、OneDrive、iCloud Drive、[Plex](#media-servers)、[Jellyfin](#media-servers) 等 — 全新的 CarPlay 体验会让一切就像车原生自带的一样。

## 超过 10 项新的云端、NAS 和服务器集成

Evermusic 8.6 拓展了「音乐库」的定义。无论你的音频文件是在你信任的云端、家里的 NAS,还是自托管的媒体服务器,Evermusic 现在都能直接进行流式播放 — 无需同步,也无需导出。

### 注重隐私的云端:Internxt 与 Proton Drive

如果你看重端到端加密和零知识存储,那么注重隐私的云端中两个最受认可的名字现在已被原生集成到 Evermusic:

- **Internxt** — 西班牙的开源、后量子加密、符合 GDPR 的云端。提供免费套餐。
- **Proton Drive** — 由 Proton Mail 与 Proton VPN 团队打造、总部位于瑞士的端到端加密存储。提供免费套餐,以及面向更大音乐库的付费方案。

只需连接一次,你的音乐就会通过加密通道传输 — Evermusic 永远看不到你的明文数据,服务提供商的服务器同样如此。

### 自托管存储:QNAP、Nextcloud、Amazon S3

适合自行运维基础设施的用户:

- **QNAP** — 通过原生 API 连接 QNAP NAS,通过本地 Wi-Fi 或远程访问获得快速、稳定的播放体验。
- **Nextcloud** — 连接任意自托管或托管的 Nextcloud 实例。如果你已经因为隐私从 Google Drive 或 Dropbox 迁出,这是绝佳选择。
- **Amazon S3** — 将 Evermusic 指向任意 S3 桶(或 Backblaze B2、Wasabi、MinIO、Cloudflare R2 等 S3 兼容存储),即可直接播放你的收藏。

### <a id="media-servers"></a>媒体服务器:Plex、Subsonic、Navidrome、Jellyfin、Emby

这是面向自托管音乐爱好者最大的一项更新。Evermusic 8.6 把你的 iPhone、iPad 或 Mac 变成最受欢迎的开源与免费增值媒体服务器的一流客户端:

- **Plex** — Plex Media Server **免费** 下载并运行。**Plex Pass** 订阅可选,可解锁移动同步、硬件转码和直播电视等功能。Evermusic 同时支持免费和 Plex Pass 库。
- **Subsonic** — 自托管音乐流媒体服务器的鼻祖。官方 Subsonic 服务器为 **付费**(30 天试用后每月 1 美元),但 Evermusic 也支持几十种 Subsonic 兼容服务器的 API。
- **Navidrome** — 用 Go 编写的现代、轻量级音乐服务器,**完全免费且开源**。实现了 Subsonic API。可在树莓派、NAS 或任意 Linux 机器上运行。强烈推荐用于数十万首曲目以内的音乐库。
- **Jellyfin** — **完全免费且开源** 的媒体服务器(Emby 的社区分支)。处理音乐、电影、电视、书籍和有声书。无账号、无遥测、无订阅。
- **Emby** — **免费增值** 媒体服务器。核心服务器免费;**Emby Premiere** 是一次性或年度购买,可解锁移动应用、离线同步、Cinema 模式等。Evermusic 同时支持免费和 Premiere 库。

无论你运行哪种服务器,Evermusic 都能播放你的整个库 — 包括专辑、艺人、播放列表、流派和嵌入封面 — 同时具备离线缓存、无缝(gapless)与交叉淡入(crossfade)播放、10 段均衡器、AirPlay、Chromecast 和 **CarPlay**。播放历史由你的服务器管理;Evermusic 会尊重它。

### 全新的网络协议:FTP、SFTP、NFS

对于使用自定义服务器、家庭实验室或没有精致移动应用的通用 NAS 的用户,Evermusic 8.6 新增了三种经典协议:

- **SFTP(SSH 文件传输协议)** — 这是 **从你自己的服务器进行安全远程播放** 的正确选择。SFTP 基于 SSH 运行,因此整个传输(身份验证和音频数据)都被加密。如果你拥有 VPS、独立服务器或家中带有 SSH 访问权限的 Linux 机器,可以将一个音乐文件夹放到上面,在公共互联网上进行播放,而不会暴露其他任何东西。支持密码和密钥认证。
- **FTP** — 长期以来确立的文件传输标准。如果你的 **家用 NAS**(较旧的群晖、ASUS、D-Link、TerraMaster 或通用机箱)提供了 FTP 服务但没有原生 API 集成,这就很有用。最好在本地网络中使用。
- **NFS(网络文件系统)** — Linux 和大多数 NAS 设备上的事实标准共享协议。NFS 共享在家庭实验室和小型企业网络中很常见;Evermusic 现在可以挂载并直接从中进行播放。在相同硬件上,NFS 的开销比 SMB 更小。

> **提示:** SFTP 是适合在公共互联网上进行流式播放的协议。FTP 和 NFS 更适合本地网络使用 — 除非你将其封装在 VPN 中,否则不要将其暴露在公共互联网。

## Wi-Fi Drive:全新界面、更快上传

[**Wi-Fi Drive**](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/) 是 Evermusic 内置的功能,可以 **从电脑无线传输音乐到你的 iPhone 或 iPad,无需 iTunes、无需数据线、无需云端账户**。两台设备需要在同一个 Wi-Fi 网络中。在 iOS 应用中启动服务器后,你可以:

- 在任意桌面浏览器(Safari、Chrome、Firefox、Edge)中打开 URL,把音乐文件拖到页面上,即可直接上传到设备;或
- 通过 **Mac Finder**(「连接到服务器…」)或 **Windows File Explorer**(映射网络驱动器)使用 WebDAV 将设备挂载为网络共享。

这是在不依赖第三方服务的情况下,把大型本地音乐库搬到手机上的最快方式。请查看 [分步指南](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/)。

在 Evermusic 8.6 中,Wi-Fi Drive 获得了:

- **全新用户界面** — 更整洁,更易于一眼看清。
- **选择模式** — 选择多个文件或文件夹,批量进行操作。
- **改进的上传队列** — 进度反馈更清晰,对网络波动的容忍度更高。
- **整体更好的性能** — 大型音乐库传输更快。

## 全新的主屏幕同步歌词小组件

最受期待的功能之一终于来了:**可以固定在 iPhone、iPad 或 Mac 主屏幕的同步歌词小组件**。

当你播放带有时间对齐歌词的曲目时,小组件会实时显示该曲目的 **当前一行**,并随着音乐推进自动切换。点击小组件即可在 Evermusic 中打开完整歌词视图。

它能做什么:

- 为正在播放的曲目显示 **同步、带时间戳的歌词**。
- 随着曲目推进自动更新 — 无需手动刷新或滚动。
- 支持嵌入在音频文件中的 LRC 歌词,以及在应用中获取的歌词。
- 提供 **多种小组件尺寸**,你可以选择主屏幕上显示多少文本。
- 支持 **iPhone**、**iPad** 和 **Mac**(在支持的位置可用于主屏幕和通知中心)。

如果你喜欢在长时间收听时让手机放在桌面、支架或锁屏上跟随歌词,这个小组件再合适不过。它与有声书和播客类内容尤其搭配,边听边读有助于理解。

要启用,长按主屏幕,点击 **编辑 > 添加小组件**,搜索 **Evermusic**,然后选择你想要尺寸的 **歌词** 小组件。

Evermusic 8.6 围绕三个理念构建:

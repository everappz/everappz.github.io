## 超過 10 項全新的雲端、NAS 與伺服器整合

Evermusic 8.6 擴大了「音樂資料庫」的定義。無論你的音訊檔案是放在你信任的雲端、家中的 NAS,或是自架的媒體伺服器,Evermusic 現在都能直接串流播放 — 無需同步,也無需匯出。

### 重視隱私的雲端:Internxt 與 Proton Drive

如果你重視端對端加密與零知識儲存,兩個最受推崇的隱私導向雲端如今已成為 Evermusic 的原生選項:

- **Internxt** — 來自西班牙的開源、後量子加密、符合 GDPR 的雲端服務。提供免費方案。
- **Proton Drive** — 由 Proton Mail 與 Proton VPN 團隊打造、總部位於瑞士的端對端加密儲存空間。提供免費方案,以及供更大資料庫使用的付費方案。

只要連線一次,你的音樂就會走加密通道 — Evermusic 看不到你的明文資料,服務商伺服器同樣也看不到。

### 自架儲存:QNAP、Nextcloud、Amazon S3

適合自行管理基礎架構的使用者:

- **QNAP** — 透過原生 API 連線至 QNAP NAS 裝置,無論是本地 Wi-Fi 還是遠端存取,都能享有快速且穩定的播放。
- **Nextcloud** — 連線到任何自架或代管的 Nextcloud 執行個體。如果你出於隱私考量已從 Google Drive 或 Dropbox 移轉,這非常適合你。
- **Amazon S3** — 將 Evermusic 指向任何 S3 儲存桶(或 Backblaze B2、Wasabi、MinIO、Cloudflare R2 等 S3 相容儲存),即可直接串流你的收藏。

### <a id="media-servers"></a>媒體伺服器:Plex、Subsonic、Navidrome、Jellyfin、Emby

對於自架音樂愛好者來說,這是最重要的一項更新。Evermusic 8.6 把你的 iPhone、iPad 或 Mac 變成最受歡迎的開源與免費增值媒體伺服器的一流用戶端:

- **Plex** — Plex Media Server 可 **免費** 下載並執行。**Plex Pass** 為可選付費訂閱,可解鎖行動同步、硬體轉碼與直播電視等功能。Evermusic 同時支援免費與 Plex Pass 資料庫。
- **Subsonic** — 自架音樂串流伺服器的鼻祖。官方 Subsonic 伺服器需 **付費**(30 天試用後每月 1 美元),但 Evermusic 也支援數十種採用 Subsonic API 的相容伺服器。
- **Navidrome** — 以 Go 撰寫的現代化、輕量級音樂伺服器,**完全免費且開源**。實作 Subsonic API。可在 Raspberry Pi、NAS 或任何 Linux 機器上執行。針對數十萬首以內的曲目資料庫,我們強力推薦。
- **Jellyfin** — **完全免費且開源** 的媒體伺服器(Emby 的社群分支)。可處理音樂、電影、電視、書籍與有聲書。沒有帳號、沒有遙測、沒有訂閱。
- **Emby** — **免費增值** 的媒體伺服器。核心伺服器免費;**Emby Premiere** 為一次性或年度付費,可解鎖行動 App、離線同步、Cinema 模式等。Evermusic 同時支援免費與 Premiere 資料庫。

無論你執行哪一種伺服器,Evermusic 都能串流你的整個資料庫 — 包含專輯、藝術家、播放清單、類型與內嵌封面 — 並提供離線快取、無縫(gapless)與交叉淡入(crossfade)播放、10 段等化器、AirPlay、Chromecast 與 **CarPlay**。播放紀錄由你的伺服器管理,Evermusic 會予以尊重。

### 全新的網路協定:FTP、SFTP、NFS

對於使用自訂伺服器、家庭實驗室或缺乏精緻行動 App 的通用 NAS 的使用者,Evermusic 8.6 新增了三種經典協定:

- **SFTP(SSH 檔案傳輸協定)** — 這是 **從自己的伺服器進行安全遠端串流** 的正確選擇。SFTP 在 SSH 之上執行,因此整個傳輸(身分驗證與音訊資料)都會被加密。如果你擁有 VPS、專屬伺服器,或家中具備 SSH 存取權的 Linux 機器,可以將音樂資料夾放在那裡,透過公開網際網路串流,而不必暴露任何其他內容。支援密碼及金鑰驗證。
- **FTP** — 長期以來確立的檔案傳輸標準。如果你的 **家用 NAS**(較舊的 Synology、ASUS、D-Link、TerraMaster 或通用機殼)提供 FTP 伺服器但沒有原生 API 整合,這就很有用。建議在區域網路中使用。
- **NFS(網路檔案系統)** — Linux 與大多數 NAS 裝置上事實上的分享協定。NFS 共用在家庭實驗室與小型企業網路中相當常見;Evermusic 現在可以掛載並直接串流。在相同硬體上,NFS 的負擔比 SMB 更小。

> **提示:** 想透過開放網際網路串流時,SFTP 是最佳選擇。FTP 與 NFS 比較適合在區域網路內使用 — 除非以 VPN 包覆,否則別把它們暴露在網際網路。

## Wi-Fi Drive:全新介面、上傳更快

[**Wi-Fi Drive**](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/) 是 Evermusic 內建的功能,可以 **在不需要 iTunes、傳輸線或雲端帳號的情況下,透過 Wi-Fi 將音樂從電腦傳輸到 iPhone 或 iPad**。兩台裝置必須位於相同的 Wi-Fi 網路。在 iOS App 中啟動伺服器後,你可以:

- 在任何桌面瀏覽器(Safari、Chrome、Firefox、Edge)中開啟網址,把音樂檔拖到頁面上,即可直接上傳到裝置;或
- 透過 **Mac Finder**(「連線至伺服器…」)或 **Windows File Explorer**(對應網路磁碟)使用 WebDAV 將裝置掛載為網路共用。

這是不依賴第三方服務,將大型本地音樂資料庫搬到手機上的最快方式。請查看 [逐步指南](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/)。

在 Evermusic 8.6 中,Wi-Fi Drive 加入了:

- **全新使用者介面** — 更整潔,更易於一目了然。
- **選擇模式** — 選取多個檔案或資料夾,執行批次操作。
- **改進的上傳佇列** — 進度回饋更清楚,更能承受網路波動。
- **整體效能更佳** — 大型資料庫的傳輸更快。

## 主畫面全新同步歌詞小工具

最受期待的功能之一終於到來:**可固定在 iPhone、iPad 或 Mac 主畫面的同步歌詞小工具**。

當你播放具有時間對齊歌詞的曲目時,小工具會即時顯示該曲目的 **目前一行**,並隨著音樂自動切換。輕點小工具即可在 Evermusic 中開啟完整歌詞檢視。

它能做什麼:

- 為目前播放的曲目顯示 **同步且帶有時間戳的歌詞**。
- 隨著歌曲推進自動更新 — 不需要手動重新整理或捲動。
- 支援音訊檔案中內嵌的 LRC 歌詞,以及在 App 內取得的歌詞。
- 提供 **多種小工具尺寸**,可選擇主畫面要顯示多少文字。
- 可在 **iPhone**、**iPad** 與 **Mac** 上使用(在支援的位置可加入主畫面與通知中心)。

如果你喜歡在較長的聆聽時間中,讓手機放在桌上、立架或鎖定畫面上跟著歌詞閱讀,這個小工具非常適合。它與有聲書與類似 podcast 的內容尤其搭配,邊聽邊讀有助於理解。

要啟用它,長按主畫面,點選 **編輯 > 加入小工具**,搜尋 **Evermusic**,然後選擇你想要尺寸的 **歌詞** 小工具。

### 中國伺服器更穩定

在這次更新中,我們投入大量精力強化 **百度網盤** 與 **阿里雲盤** 的播放穩定性。如果你之前在 Evermusic 中使用過這些服務,你會發現:

- 在較慢或地理位置較遠的連線上,串流更穩定。
- 長時間播放期間,連線中斷次數降低。
- 網路短暫斷線時,重試與續傳邏輯更聰明。

這兩個雲端服務是中國最受歡迎的消費者儲存服務,而 Evermusic 是少數對兩者都提供一流原生支援的 iOS 音樂播放器之一。

### 全新的主畫面小工具與更聰明的更新

除了全新的歌詞小工具,既有的主畫面小工具 — 正在播放、播放佇列、最近播放 — 也藉由 **更好的更新邏輯** 進行細修,可以保持同步而不會額外耗電。同時也新增了讓使用者更能掌控顯示內容的全新小工具版面。

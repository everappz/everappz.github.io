## Зовсім новий досвід CarPlay

Ми перебудували Evermusic для **Apple CarPlay** з нуля. Результат — швидше, плавніше та значно простіше у використанні за кермом.

- **Швидке сортування** — знаходьте будь-який трек за секунди в альбомах, виконавцях, плейлистах і теках.
- **Кілька кольорових схем** — оберіть тему, що пасує до приладової панелі чи внутрішнього освітлення авто.
- **Перероблений екран «Зараз грає»** — більша обкладинка, чіткіші елементи керування й нові дії відтворення в один дотик.
- **Повна черга відтворення з одного погляду** — бачте, що далі, не залишаючи поточного екрана.
- **Швидкий алфавітний індекс прокручування** — переходьте величезними бібліотеками (10 000+ треків) без нескінченної прокрутки.
- **Плавніше завантаження вмісту** — більше жодних зависань під час відкривання великих папок, хмарних каталогів чи бібліотек медіасерверів.

Якщо ви слухаєте музику з хмари за кермом — Dropbox, Google Drive, OneDrive, iCloud Drive, [Plex](#media-servers), [Jellyfin](#media-servers) та інших — новий CarPlay робить увесь сценарій відчутно «рідним» для авто.

## Понад 10 нових інтеграцій із хмарами, NAS та серверами

Evermusic 8.6 розширює поняття вашої «музичної бібліотеки». Хай де лежать ваші файли — у хмарі, якій ви довіряєте, на NAS удома чи на самостійно піднятому медіасервері — Evermusic тепер може транслювати з них напряму, без синхронізації та експорту.

### Хмари, орієнтовані на приватність: Internxt і Proton Drive

Якщо вам важливі наскрізне шифрування й сховище без знання провайдера, два найшанованіших імені приватної хмари тепер підтримуються в Evermusic нативно:

- **Internxt** — іспанська хмара з відкритим кодом, постквантовим шифруванням і відповідністю GDPR. Доступний безкоштовний рівень.
- **Proton Drive** — наскрізно зашифроване сховище від творців Proton Mail і Proton VPN, із базою у Швейцарії. Є безкоштовний рівень і платні плани для більших бібліотек.

Підключіться один раз — і ваша музика йтиме зашифрованим тунелем. Evermusic ніколи не бачить ваших даних у відкритому вигляді, як і сервер провайдера.

### Самостійне сховище: QNAP, Nextcloud, Amazon S3

Для користувачів, які тримають власну інфраструктуру:

- **QNAP** — нативне з'єднання за API з пристроями QNAP NAS для швидкого та надійного відтворення локальною Wi-Fi або віддалено.
- **Nextcloud** — підключіться до будь-якої Nextcloud — самостійно піднятої або керованої. Чудово, якщо ви вже мігрували з Google Drive чи Dropbox з міркувань приватності.
- **Amazon S3** — спрямуйте Evermusic на будь-який S3-bucket (або S3-сумісне сховище: Backblaze B2, Wasabi, MinIO, Cloudflare R2) і транслюйте свою колекцію напряму.

### <a id="media-servers"></a>Медіасервери: Plex, Subsonic, Navidrome, Jellyfin, Emby

Це найбільше доповнення для прихильників self-hosted музики. Evermusic 8.6 перетворює ваш iPhone, iPad чи Mac на клієнт першого класу для найпопулярніших відкритих та freemium медіасерверів:

- **Plex** — Plex Media Server **безкоштовний** для завантаження та запуску. Підписка **Plex Pass** є опціональною й відкриває такі можливості, як мобільну синхронізацію, апаратне транскодування та живе ТБ. Evermusic працює і з безкоштовними бібліотеками, і з Plex Pass.
- **Subsonic** — оригінальний self-hosted сервер музичного стрімінгу. Офіційний сервер Subsonic **платний** (1 $/міс після 30-денного пробного періоду), але Evermusic також спілкується API Subsonic з десятками сумісних серверів.
- **Navidrome** — сучасний легкий музичний сервер, **повністю безкоштовний і з відкритим кодом**, написаний мовою Go. Реалізує API Subsonic. Працює на Raspberry Pi, NAS чи будь-якій машині з Linux. Дуже рекомендуємо для бібліотек до кількох сотень тисяч треків.
- **Jellyfin** — **повністю безкоштовний і з відкритим кодом** медіасервер (форк Emby спільнотою). Працює з музикою, фільмами, ТБ, книжками й аудіокнигами. Без облікових записів, без телеметрії, без підписок.
- **Emby** — медіасервер за моделлю **freemium**. Базовий сервер безкоштовний; **Emby Premiere** — разова або щорічна покупка, що відкриває мобільні застосунки, офлайн-синхронізацію, режим Cinema тощо. Evermusic підключається до бібліотек і безкоштовного, і Premiere рівня.

Який би сервер ви не використовували, Evermusic транслює всю вашу бібліотеку — альбоми, виконавців, плейлисти, жанри й вбудовані обкладинки — з офлайн-кешем, безперервним відтворенням і кросфейдом, 10-смуговим еквалайзером, AirPlay, Chromecast і **CarPlay**. Сервер веде історію прослуховування; Evermusic поважає її.

### Нові мережеві протоколи: FTP, SFTP, NFS

Для користувачів із самописними серверами, домашніми лабораторіями чи універсальними NAS, які не мають вилизаного мобільного застосунку, Evermusic 8.6 додає три класичні протоколи:

- **SFTP (SSH File Transfer Protocol)** — правильна відповідь для **безпечного віддаленого стрімінгу з власного сервера**. SFTP працює поверх SSH, тож уся передача (і автентифікація, і аудіодані) шифрується. Якщо у вас є VPS, виділений сервер чи Linux-машина вдома з доступом по SSH, можна покласти туди папку з музикою й транслювати через відкритий інтернет, нічого більше не відкриваючи. Підтримується автентифікація паролем і ключем.
- **FTP** — давній стандарт передавання файлів. Корисно, якщо ваш **домашній NAS** (старий Synology, ASUS, D-Link, TerraMaster чи універсальні бокси) надає FTP-сервер, але без рідної API-інтеграції. Найкраще використовувати в локальній мережі.
- **NFS (Network File System)** — фактичний стандартний протокол спільного доступу в Linux і на більшості NAS. NFS-шарінги типові для homelab і мереж малих компаній; Evermusic тепер може монтувати їх і транслювати безпосередньо. На тому самому обладнанні менше накладних витрат, ніж у SMB.

> **Порада:** SFTP — це протокол, який варто обирати для стрімінгу через відкритий інтернет. FTP та NFS краще тримати в локальній мережі — не виставляйте їх в інтернет, якщо тільки не загорнули у VPN.

## Wi-Fi Drive: новий інтерфейс і швидші завантаження

[**Wi-Fi Drive**](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/) — це вбудована функція Evermusic для **бездротового перенесення музики з комп'ютера на iPhone або iPad — без iTunes, без кабелів, без хмарного облікового запису**. Обидва пристрої мають бути в одній мережі Wi-Fi. Ви запускаєте сервер у застосунку iOS і потім:

- відкриваєте URL у будь-якому настільному браузері (Safari, Chrome, Firefox, Edge), перетягуєте файли музики на сторінку — і вони завантажуються прямо на пристрій, або
- монтуєте пристрій як мережевий ресурс через **Mac Finder** («Підключити до сервера…») або **Windows File Explorer** (Підключити мережевий диск) за допомогою WebDAV.

Це найшвидший спосіб перенести велику локальну музичну бібліотеку на телефон без сторонніх сервісів. Дивіться [покрокову інструкцію тут](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/).

В Evermusic 8.6 Wi-Fi Drive отримує:

- **Новий інтерфейс** — чистіший і легший для побіжного читання.
- **Режим вибору** — обирайте кілька файлів чи тек і виконуйте дії пакетно.
- **Покращена черга завантажень** — кращий зворотний зв'язок про прогрес і стійкість до перебоїв мережі.
- **Краща загальна продуктивність** — швидші передачі для великих бібліотек.

## Новий віджет синхронізованих текстів пісень на Головному екрані

Одна з найочікуваніших можливостей нарешті тут: **віджет синхронізованих текстів, який можна закріпити на Головному екрані iPhone, iPad або Mac**.

Коли ви відтворюєте трек із текстом, прив'язаним до часу, віджет показує **поточний рядок** пісні в реальному часі — і рядок просувається разом із музикою. Торкніться віджета, щоб відкрити Evermusic у режимі повного перегляду тексту.

Що він робить:

- Показує **синхронізовані тексти з часовими мітками** для трека, що зараз відтворюється.
- Оновлюється відповідно до перебігу пісні — без ручного оновлення та прокручування.
- Працює з вбудованими в аудіофайли LRC-текстами та з текстами, отриманими в застосунку.
- Доступний у **кількох розмірах віджета**, щоб ви обрали, скільки тексту вмістити на Головному екрані.
- Працює на **iPhone**, **iPad** та **Mac** (Головний екран і Центр сповіщень там, де підтримується).

Віджет ідеально підходить, коли ви любите стежити за словами, поки телефон лежить на столі, у тримачі або на екрані блокування під час тривалого прослуховування. Особливо добре він пасує до аудіокниг і подкаст-подібного контенту, де читання разом із прослуховуванням допомагає розумінню.

Щоб увімкнути, утримуйте Головний екран, торкніться **Редагувати > Додати віджет**, знайдіть **Evermusic** і виберіть віджет **Тексти** в потрібному розмірі.

## Інші покращення

### Оновлення дизайну Liquid Glass

Інтерфейс Evermusic 8.6 оновлено для нового матеріалу **Liquid Glass** від Apple у всьому застосунку — напівпрозорі поверхні, плавніші анімації та відточені елементи керування, які природно вписуються в iOS, iPadOS та macOS.

### Краща надійність для китайських серверів

У цьому релізі ми приділили багато уваги стабільності відтворення для **Baidu Netdisk (百度网盘)** і **Aliyun Drive (阿里云盘)**. Якщо ви вже користувалися цими сервісами в Evermusic, ви помітите:

- Швидший перегляд каталогів у бібліотеках із тисячами файлів.
- Стабільніший стрімінг на повільних або географічно віддалених з'єднаннях.
- Менше обривів зв'язку під час тривалих сесій відтворення.
- Покращену логіку повторних спроб і відновлення, коли мережа коротко зникає.

Ці дві хмари — найпопулярніші сервіси зберігання для споживачів у Китаї, а Evermusic — один із небагатьох iOS-плеєрів із першокласною нативною підтримкою обох.

### Оновлені бібліотеки з'єднань

Ми оновили внутрішні бібліотеки, які Evermusic використовує для спілкування з **WebDAV**, **SMB**, **DLNA**, **Dropbox**, **Google Drive**, **OneDrive** та іншими сервісами. Простіше кажучи: менше відмов з'єднання у крайових випадках, краща підтримка нових версій серверів і вища продуктивність загалом.

### Нові віджети Головного екрана та розумніші оновлення

Окрім нового віджета текстів, наявні віджети Головного екрана — Зараз грає, Черга відтворення, Нещодавно прослухані — переглянуто з **кращою логікою оновлення**, тож вони залишаються синхронізованими, не з'їдаючи зайвої батареї. Також з'явилися нові розмітки віджетів для тих, хто хоче більше контролювати, що видно з одного погляду.

### Виправлення вад і шліфування

- Усунуто проблеми з відтворенням на деяких медіасерверах і самостійно піднятих конфігураціях.
- Виправлення локалізації в кількох мовах.
- Багато менших покращень за прямими відгуками з оглядів App Store та листів на support@everappz.com.

## Чому це оновлення важливе

Evermusic 8.6 побудовано навколо трьох ідей:

1. **Ваша музика, де б ви її не зберігали.** Хай ваша бібліотека лежить у iCloud Drive, у хмарі з акцентом на приватність на кшталт Proton Drive чи Internxt, на медіасервері Plex чи Jellyfin, на домашньому NAS чи Raspberry Pi із Navidrome — Evermusic тепер підключається нативно й дає однаковий досвід відтворення скрізь.
2. **Краще в авто.** CarPlay — це екран, який багато користувачів бачать найчастіше, і перебудований досвід це відображає.

Так. Починаючи з Evermusic 8.6 ви можете підключитися до Plex Media Server і транслювати всю свою музичну бібліотеку. Plex Media Server працює безкоштовно; Plex Pass — опціональний. Evermusic підтримує і безкоштовні налаштування, і Plex Pass.
{{% /details %}}

{{% details title="Чи підтримуються Jellyfin або Navidrome у Evermusic?" closed="true" %}}
Так. І Jellyfin, і Navidrome повністю підтримуються у Evermusic 8.6. Jellyfin — це безкоштовний медіасервер з відкритим кодом. Navidrome — безкоштовний музичний сервер з відкритим кодом, що реалізує API Subsonic. Evermusic нативно підключається до обох.
{{% /details %}}

{{% details title="Plex, Jellyfin, Emby, Navidrome і Subsonic — безкоштовні?" closed="true" %}}
- **Plex** — сервер безкоштовний; Plex Pass — опціональне платне розширення.
- **Jellyfin** — повністю безкоштовний і з відкритим кодом.
- **Emby** — сервер безкоштовний; Emby Premiere платне і відкриває мобільну та офлайн-синхронізацію.
- **Navidrome** — повністю безкоштовний і з відкритим кодом.
- **Subsonic** — офіційний сервер коштує 1 $ на місяць після 30-денного пробного періоду, проте його API відкритий, і багато безкоштовних серверів (включно з Navidrome) його реалізують.
{{% /details %}}

{{% details title="Чи можу я транслювати з домашнього NAS через SFTP, FTP або NFS?" closed="true" %}}
Так. Evermusic 8.6 додає SFTP, FTP та NFS як рідні типи з'єднання. SFTP — рекомендований варіант для трансляції з власного сервера через відкритий інтернет, бо весь трафік шифрується через SSH. FTP та NFS краще використовувати в локальній мережі або за VPN.
{{% /details %}}

{{% details title="Як підключити Evermusic до власного сервера через SFTP?" closed="true" %}}
Відкрийте Evermusic, перейдіть на вкладку З'єднання, оберіть SFTP та введіть ім'я хоста або IP сервера, порт (зазвичай 22), ім'я користувача й пароль або приватний SSH-ключ. Evermusic покаже ваші віддалені теки й транслюватиме аудіофайли напряму з наскрізним шифруванням.
{{% /details %}}

{{% details title="Чи підтримує Evermusic Internxt і Proton Drive?" closed="true" %}}
Так. Обидві хмари, що зосереджені на приватності, підтримуються з Evermusic 8.6. Вони приєднуються до Mega та інших privacy-first сервісів, уже доступних у застосунку.
{{% /details %}}

{{% details title="Що таке Wi-Fi Drive у Evermusic?" closed="true" %}}
Wi-Fi Drive — це вбудована функція Evermusic для бездротового перенесення файлів. Вона дозволяє завантажувати музику з комп'ютера на iPhone або iPad через локальну мережу Wi-Fi — без iTunes, без кабелів, без хмарного акаунта. Можна використовувати будь-який настільний браузер або WebDAV-клієнт, як-от Mac Finder чи Windows File Explorer. Дивіться [повний посібник із Wi-Fi Drive](/docs/howto/how-to-transfer-files-wirelessly-from-a-computer-to-an-iphone-using-wifi-drive/).
{{% /details %}}

{{% details title="Як працює новий віджет текстів?" closed="true" %}}
Віджет текстів показує синхронізовані за часом рядки на Головному екрані iPhone, iPad або Mac для трека, що відтворюється. Відображуваний рядок автоматично рухається разом із піснею. Додайте його, утримуючи Головний екран, торкнувшись Редагувати > Додати віджет, знайшовши Evermusic і обравши віджет Тексти.
{{% /details %}}

## 처음부터 다시 만든 CarPlay

Flacbox의 **Apple CarPlay**를 처음부터 다시 디자인하여 운전 중 청취가 더 안전하고 쉽게 느껴지도록 했습니다. 곡을 찾는 방법부터 재생을 제어하는 방법까지 모든 세부 사항이 차량 내 사용 경험에 맞게 다시 조정되었습니다.

- **빠른 정렬** — 끝없는 스크롤 없이 앨범, 아티스트, 재생 목록, 폴더 전반에서 어떤 곡도 순식간에 찾을 수 있습니다.
- **차량에 어울리는 색상 테마** — 낮이든 밤이든 대시보드 또는 실내 조명에 어울리는 테마를 선택하세요.
- **새롭게 디자인된 재생 중 화면** — 더 큰 아트워크, 더 명확한 컨트롤, 한 번의 탭으로 사용 가능한 새로운 재생 동작.
- **재생 대기열 전체를 한눈에** — 현재 화면을 떠나지 않고 다음에 무엇이 나올지 확인할 수 있습니다.
- **대용량 라이브러리를 위한 알파벳 인덱스** — 한 번의 터치로 수만 곡의 FLAC, DSD, ALAC, APE 트랙을 가로지를 수 있습니다.
- **더 빠른 로딩, 더 이상의 멈춤 없음** — 큰 폴더, 클라우드 디렉터리, 미디어 서버 라이브러리를 여는 것이 즉시처럼 느껴집니다.

운전 중에 무손실 오디오를 스트리밍한다면 — Dropbox, Google Drive, OneDrive, iCloud Drive, [Plex](#media-servers), [Jellyfin](#media-servers), Subsonic 또는 다른 어떤 소스에서든 — 다시 만든 CarPlay 경험은 모든 흐름을 차량에 원래부터 있던 것처럼 자연스럽게 만들어 줍니다.

## 음악을 연결하는 10가지 이상의 새로운 방법

Flacbox 7.4는 ‘음악 라이브러리’로 간주되는 것의 의미를 확장합니다. 고해상도 파일이 신뢰하는 클라우드, 집의 NAS 또는 자체 호스팅 스트리밍 서버에 있다면, Flacbox는 이제 그곳에서 직접 재생합니다 — 동기화도, 내보내기도, 포맷 변환도 없습니다.

### 프라이빗 클라우드: Internxt와 Proton Drive

엔드 투 엔드 암호화와 제로 지식 스토리지가 중요하다면, 가장 인정받는 프라이버시 우선 클라우드 두 곳이 이제 Flacbox에 네이티브로 들어왔습니다.

- **Internxt** — 오픈 소스이자 포스트 양자 암호화, GDPR을 준수하는 스페인 클라우드. 무료 티어 제공.
- **Proton Drive** — Proton Mail과 Proton VPN 제작자가 만든 엔드 투 엔드 암호화 스토리지(스위스 본사). 무료 티어 제공, 더 큰 라이브러리를 위한 유료 요금제도 있습니다.

한 번 연결하면 FLAC, DSD, ALAC 파일이 암호화된 터널을 통해 스트리밍됩니다 — Flacbox는 데이터를 평문으로 보지 않고, 제공자 서버 역시 마찬가지입니다.

### 개인 서버: QNAP, Nextcloud, Amazon S3

자체 인프라를 운영하는 청취자를 위해.

- **QNAP** — QNAP NAS 장치에 네이티브 API 연결로 로컬 Wi-Fi 또는 원격 접속을 통해 빠르고 안정적인 재생을 제공합니다. 고비트레이트 FLAC과 DSD를 재인코딩 없이 직접 스트리밍합니다.
- **Nextcloud** — 자체 호스팅이든 관리형이든 모든 Nextcloud 인스턴스에 연결할 수 있습니다. 프라이버시를 이유로 이미 Google Drive나 Dropbox에서 이전했다면 훌륭한 선택지입니다.
- **Amazon S3** — Flacbox를 모든 S3 버킷(또는 Backblaze B2, Wasabi, MinIO, Cloudflare R2 같은 S3 호환 스토리지)에 연결하고 컬렉션을 직접 스트리밍하세요.

### <a id="media-servers"></a>스트리밍 서버: Plex, Subsonic, Navidrome, Jellyfin, Emby

자체 호스팅 음악 팬에게 가장 큰 변화입니다. Flacbox 7.4는 iPhone이나 Mac을 가장 인기 있는 오픈 소스 및 프리미엄 미디어 서버를 위한 일류 고해상도 클라이언트로 만듭니다.

- **Plex** — Plex Media Server는 다운로드와 실행이 **무료**입니다. **Plex Pass** 구독은 선택 사항이며 모바일 동기화, 하드웨어 트랜스코딩 등의 부가 기능을 잠금 해제합니다. Flacbox는 무료 라이브러리와 Plex Pass 라이브러리 모두에서 작동합니다.
- **Subsonic** — 자체 호스팅 음악 스트리밍 서버의 원조입니다. 공식 Subsonic 서버는 **유료**(30일 체험 후 월 1달러)이며, Flacbox는 수십 종의 호환 서버와도 Subsonic API로 통신합니다.
- **Navidrome** — Go로 작성된 현대적이고 가벼운 음악 서버로, **완전 무료이며 오픈 소스**입니다. Subsonic API를 구현합니다. Raspberry Pi, NAS 또는 모든 Linux 머신에서 실행됩니다. 수십만 곡 규모의 무손실 컬렉션까지 강력 추천합니다.
- **Jellyfin** — **완전 무료이며 오픈 소스인** 미디어 서버(Emby의 커뮤니티 포크)입니다. 음악, 영화, TV, 오디오북을 다룹니다. 계정도, 텔레메트리도, 구독도 없습니다.
- **Emby** — **프리미엄(Freemium)** 미디어 서버입니다. 코어 서버는 무료이며, **Emby Premiere**는 일회성 또는 연간 결제로 모바일 앱, 오프라인 동기화 등을 잠금 해제합니다. Flacbox는 무료 라이브러리와 Premiere 라이브러리 모두에 연결됩니다.

어떤 서버를 사용하든 Flacbox는 전체 컬렉션 — 앨범, 아티스트, 재생 목록, 장르, 임베디드 아트워크 — 을 USB DAC에 비트 퍼펙트 출력, 10밴드 이퀄라이저, 크로스페이드와 갭리스 재생, AirPlay, Chromecast 그리고 다시 만든 **CarPlay** 경험과 함께 스트리밍합니다. 듣기 기록은 서버가 관리하며 Flacbox는 이를 존중합니다.

### 새로운 전송 방법: FTP, SFTP, NFS

맞춤형 서버, 홈랩, 정교한 모바일 앱이 없는 일반 NAS를 사용하는 청취자를 위해 Flacbox 7.4는 세 가지 클래식 네트워크 프로토콜을 추가합니다.

- **SFTP(SSH File Transfer Protocol)** — **자체 서버에서 안전하게 원격 스트리밍**할 때 가장 적합한 답입니다. SFTP는 SSH 위에서 동작하므로 전송 전체(인증과 오디오 데이터)가 암호화됩니다. SSH 접근이 가능한 VPS, 전용 서버 또는 집의 Linux 머신이 있다면 FLAC이나 DSD 폴더를 올려두고 다른 것은 노출하지 않은 채 공개 인터넷에서 스트리밍할 수 있습니다. 비밀번호 및 키 기반 인증을 모두 지원합니다.
- **FTP** — 오랫동안 자리 잡은 파일 전송 표준입니다. 오래된 Synology, ASUS, D-Link, TerraMaster, 일반 박스 같은 **가정용 NAS**가 FTP 서버는 제공하지만 네이티브 API 연동이 없는 경우에 유용합니다. 로컬 네트워크 내 사용을 권장합니다.
- **NFS(Network File System)** — Linux와 대부분의 NAS 장치에서 사실상 표준이 된 공유 프로토콜입니다. NFS 공유는 홈랩과 소규모 비즈니스 네트워크에서 흔히 쓰이며, Flacbox는 이를 마운트해 비트 퍼펙트 오디오를 직접 스트리밍합니다. 같은 하드웨어에서 SMB보다 오버헤드가 적습니다.

> **팁:** 공개 인터넷에서 스트리밍하려면 SFTP가 적합한 프로토콜입니다. FTP와 NFS는 로컬 네트워크 안에서 사용하는 것이 가장 좋습니다 — VPN으로 감싸지 않는 한 공개 인터넷에 노출하지 마세요.

## 그 외 개선

### Liquid Glass와 어울리는 새로운 디자인

Flacbox 7.4의 인터페이스는 앱 전반에 걸쳐 Apple의 새로운 **Liquid Glass** 머티리얼에 맞게 업데이트되었습니다 — 반투명 표면, 더 부드러운 애니메이션, iOS 26과 macOS 26에 자연스럽게 어울리는 정제된 컨트롤. 라이브러리, 재생 중, 이퀄라이저, 설정 화면이 모두 새로운 시스템 미학에 맞게 다시 조정되었습니다.

### 더 강력한 네트워크 라이브러리

Flacbox가 **WebDAV**, **SMB**, **DLNA**, **Dropbox**, **Google Drive**, **OneDrive**, **iCloud Drive**, **MEGA** 및 다른 서비스와 통신하는 데 사용하는 내부 라이브러리를 새로 고쳤습니다. 즉, 엣지 케이스의 연결 실패가 줄어들고, 최신 서버 버전에 대한 지원이 향상되며, 느리거나 지리적으로 먼 연결에서 고비트레이트 오디오를 스트리밍할 때 안정성이 개선됩니다.

### 일부 서버의 재생 문제 수정

일부 자체 호스팅 서버에서의 재생 문제를 추적해 수정했습니다 — 큰 FLAC 및 DSD 파일을 탐색한 후 멈추는 문제, 매우 긴 파일 목록이 있는 라이브러리의 느린 시작 시간 등이 포함됩니다. 스트리밍이 처음부터 끝까지 더 깔끔하게 느껴질 것입니다.

### 더 나은 새로 고침의 새로운 홈 화면 위젯

홈 화면 위젯 — 재생 중, 재생 대기열, 최근 재생 — 이 더 깔끔한 레이아웃과 더 똑똑한 새로 고침 정책으로 재설계되었습니다. 추가 배터리 소모 없이 앱과 동기화 상태를 유지하며, 몇 가지 새로운 위젯 크기로 한눈에 보이는 정보에 대한 더 많은 제어를 제공합니다.

## 이번 업데이트가 의미하는 것

Flacbox 7.4는 두 가지 아이디어를 중심으로 만들어졌습니다.

1. **어디에 보관하든, 당신의 고해상도 음악.** 무손실 라이브러리가 iCloud Drive, Proton Drive나 Internxt 같은 프라이버시 중심 클라우드, Plex나 Jellyfin과 같은 미디어 서버, 집의 NAS, 또는 Navidrome을 실행하는 Raspberry Pi 어디에 있든 — Flacbox는 이제 네이티브로 연결되어 어디서나 동일한 비트 퍼펙트 재생 경험을 제공합니다.

## 10+ 种连接音乐的新方式

Flacbox 7.4 拓展了「音乐库」的定义。如果你的 hi-res 文件存放在你信任的云端、家中 NAS,或自托管的流媒体服务器,Flacbox 现在可以直接从中流播放 — 无需同步、无需导出、无需格式转换。

### 注重隐私的云端:Internxt 与 Proton Drive

如果端到端加密与零知识存储对你很重要,那么注重隐私的云端中两个最受尊敬的名字现在已在 Flacbox 中原生支持:

- **Internxt** — 西班牙的开源、后量子加密、符合 GDPR 的云端。提供免费套餐。
- **Proton Drive** — 由 Proton Mail 与 Proton VPN 团队打造、总部位于瑞士的端到端加密存储。提供免费套餐,以及面向更大音乐库的付费方案。

只需连接一次,你的 FLAC、DSD 或 ALAC 文件就会通过加密通道传输 — Flacbox 永远看不到你的明文数据,服务提供商的服务器同样如此。

### 个人服务器:QNAP、Nextcloud、Amazon S3

适合自行运维基础设施的听众:

- **QNAP** — 通过原生 API 连接 QNAP NAS 设备,通过本地 Wi-Fi 或远程访问获得快速、可靠的播放。直接流播放高比特率 FLAC 和 DSD,无需重新编码。
- **Nextcloud** — 连接任意自托管或托管的 Nextcloud 实例。如果你已经因为隐私原因从 Google Drive 或 Dropbox 迁出,这是绝佳选择。
- **Amazon S3** — 将 Flacbox 指向任意 S3 存储桶(或 S3 兼容存储,如 Backblaze B2、Wasabi、MinIO、Cloudflare R2),即可直接流播放你的收藏。

### <a id="media-servers"></a>流媒体服务器:Plex、Subsonic、Navidrome、Jellyfin、Emby

这是面向自托管音乐爱好者最大的一项更新。Flacbox 7.4 把你的 iPhone 或 Mac 变成最受欢迎的开源与免费增值媒体服务器的一流 hi-res 客户端:

- **Plex** — Plex Media Server **免费** 下载并运行。**Plex Pass** 订阅可选,可解锁移动同步、硬件转码及其他附加功能。Flacbox 同时支持免费和 Plex Pass 资料库。
- **Subsonic** — 自托管音乐流媒体服务器的鼻祖。官方 Subsonic 服务器为 **付费**(30 天试用后每月 1 美元),Flacbox 也支持几十种 Subsonic 兼容服务器的 API。
- **Navidrome** — 用 Go 编写的现代、轻量级音乐服务器,**完全免费且开源**。实现了 Subsonic API。可在树莓派、NAS 或任意 Linux 机器上运行。强烈推荐用于数十万首曲目以内的无损收藏。
- **Jellyfin** — **完全免费且开源** 的媒体服务器(Emby 的社区分支)。处理音乐、电影、电视和有声书。无账户、无遥测、无订阅。
- **Emby** — **免费增值** 媒体服务器。核心服务器免费;**Emby Premiere** 是一次性或年度购买,可解锁移动应用、离线同步等。Flacbox 同时支持免费和 Premiere 资料库。

无论你运行哪种服务器,Flacbox 都能流播放你的整个收藏 — 包括专辑、艺人、播放列表、流派和嵌入封面 — 同时提供 bit-perfect 输出到 USB DAC、10 段均衡器、crossfade 与 gapless 播放、AirPlay、Chromecast 以及重建后的 **CarPlay** 体验。你的服务器保留收听历史;Flacbox 会尊重它。

### 全新传输方式:FTP、SFTP、NFS

对于使用自定义服务器、家庭实验室或没有精致移动应用的通用 NAS 的听众,Flacbox 7.4 新增了三种经典网络协议:

- **SFTP(SSH 文件传输协议)** — 这是 **从你自己的服务器进行安全远程流播放** 的正确选择。SFTP 基于 SSH 运行,因此整个传输(身份验证和音频数据)都被加密。如果你拥有 VPS、独立服务器,或家中带有 SSH 访问的 Linux 机器,可以把 FLAC 或 DSD 文件夹放上去,通过公共互联网流播放,而不会暴露其他任何东西。支持密码和密钥认证。
- **FTP** — 长期以来确立的文件传输标准。如果你的 **家用 NAS**(较旧的群晖、ASUS、D-Link、TerraMaster 或通用机箱)提供 FTP 服务但没有原生 API 集成,这就很有用。最好在本地网络中使用。
- **NFS(网络文件系统)** — Linux 和大多数 NAS 设备上的事实标准共享协议。NFS 共享在家庭实验室和小型企业网络中很常见;Flacbox 现在可以挂载并直接流播放 bit-perfect 音频。在相同硬件上,NFS 的开销比 SMB 更小。

> **提示:** SFTP 是适合在公共互联网上进行流播放的协议。FTP 和 NFS 更适合本地网络使用 — 除非你将其封装在 VPN 中,否则不要将其暴露在公共互联网。

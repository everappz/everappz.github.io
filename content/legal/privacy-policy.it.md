## Dati utente Google

Alcune delle nostre applicazioni hanno la capacità di connettersi al tuo account Google Drive, permettendoti di gestire file, modificare i tag audio dei file multimediali e riprodurre contenuti multimediali presenti nel tuo account Google Drive.

Una volta connesso il tuo account Google Drive all'applicazione, questa utilizza un token di accesso OAuth e il [Google Drive SDK](https://developers.google.com/drive/api/guides/about-sdk) ufficiale per gestire le richieste al server Google Drive. Non memorizziamo nessuna delle tue informazioni personali quando connetti il tuo account Google Drive. Tutte le tue informazioni personali e i dati di connessione, incluso il token di accesso, sono memorizzati in modo sicuro localmente sul tuo dispositivo nel Keychain, un archivio di sistema sicuro.

Hai il controllo del tuo accesso. Se desideri revocare l'accesso al tuo account Google Drive, puoi farlo in qualsiasi momento aprendo la pagina delle impostazioni dell'account nel tuo browser web. Segui i passaggi descritti qui: [Come disconnettere un'app di terze parti dal tuo account Google](/docs/howto/how-to-disconnect-third-party-app-from-your-google-account). Inoltre, nelle impostazioni dell'applicazione, puoi disconnettere il tuo account Google Drive e tutti i dati di connessione, incluso il token di accesso, verranno rimossi dal tuo dispositivo.

Puoi comodamente scaricare i tuoi file personali dall'account Google Drive connesso al tuo dispositivo, e questi file verranno posizionati nella directory Documenti dell'App. Puoi rimuovere questi file in qualsiasi momento utilizzando il file manager integrato nell'applicazione. Le nostre applicazioni non condividono alcuna informazione dal tuo account Google.

Il nostro utilizzo e trattamento dei dati ottenuti dalle API di Google sono conformi alla [Politica sui dati utente dei servizi API di Google](https://developers.google.com/terms/api-services-user-data-policy), inclusi i requisiti di utilizzo limitato.

## Annunci Google Mobile

Alcune delle nostre applicazioni utilizzano il Google Mobile Ads SDK per la monetizzazione. Per migliorare le prestazioni di AdMob, il Google Mobile Ads SDK può raccogliere determinate informazioni dalle app, tra cui:

- Indirizzo IP, che può essere utilizzato per stimare la posizione generale di un dispositivo.
- Log di crash non relativi all'utente, che possono essere utilizzati per diagnosticare problemi e migliorare l'SDK.
- Le informazioni diagnostiche possono anche essere utilizzate per scopi pubblicitari e di analisi.
- Dati sulle prestazioni associati all'utente come il tempo di avvio dell'app, il tasso di blocco o il consumo energetico, che possono essere utilizzati per valutare il comportamento dell'utente, comprendere l'efficacia delle funzionalità esistenti del prodotto e pianificare nuove funzionalità.
- I dati sulle prestazioni possono anche essere utilizzati per la visualizzazione di annunci, inclusa la condivisione con altre entità che visualizzano annunci.
- Un ID dispositivo come l'identificatore pubblicitario del dispositivo o altri identificatori di dispositivo legati all'app, che possono essere utilizzati per scopi pubblicitari e di analisi di terze parti.
- Dati pubblicitari, come gli annunci che l'utente ha visto, possono essere utilizzati per alimentare le funzionalità di analisi e pubblicità.
- Altre interazioni utente-prodotto come i tocchi di avvio dell'app e le informazioni di interazione, come le visualizzazioni di video, possono essere utilizzate per migliorare le prestazioni pubblicitarie.

Site Overview: everappz.com is a software publisher website built with Hugo (Hextra theme), hosted on GitHub Pages. The site showcases 4 iOS/Mac apps (Evermusic, Flacbox, Evertag, Evervideo) with 14M+ combined downloads. It features 47 how-to guides, 20 blog posts, 4 user guides, and comprehensive documentation.

Key Strengths: Solid static-site performance, extensive helpful content library (47 how-to articles), strong niche authority in iOS audio apps, established press coverage from 24+ publications.

Critical Gaps: Zero structured data (JSON-LD), no sitemap in robots.txt, unoptimized meta titles across all product pages, and llms.txt not enabled for AI search visibility.

2. Technical SEO 65/100

Strengths

  • HTTPS enabled site-wide
  • Canonical tags auto-generated on all pages
  • meta robots: index, follow in production builds
  • Open Graph and Twitter Card meta tags present
  • CSS minified, concatenated, and fingerprinted with SRI integrity
  • Clean URL structure with trailing slashes
  • FlexSearch enabled for on-site search

Issues

Critical No sitemap reference in robots.txt
robots.txt contains only User-agent: * with no Sitemap: directive. Search engines may not discover the XML sitemap automatically.
Fix: Add Sitemap: https://everappz.com/sitemap.xml to robots.txt, or configure Hugo's enableRobotsTXT with a custom template that includes the sitemap URL.
High Sitemap: 571 URLs with identical lastmod dates
All URLs have 2026-04-09T20:38:41+02:00. Google devalues lastmod signals when dates are uniform. Hugo's enableGitInfo: true should provide per-page dates — the issue may be that all content was regenerated at once without individual git history.
Fix: Ensure each page's lastmod reflects its actual last modification date via Git commit history.
High ~400 tag/taxonomy pages in sitemap
Tag pages provide minimal unique content and consume crawl budget. The sitemap has ~400 tag URLs out of 571 total (70%).
Fix: Exclude tag pages from the sitemap, or add noindex to low-value taxonomy pages. Reduce sitemap to ~170 meaningful content URLs.
Medium Homepage title is just “Everappz”
The <title> tag pattern for the homepage is simply the site title with no keywords or value proposition.
Fix: Update to something like “Everappz – iOS & Mac Apps for Music, Video & File Management”
Medium schema.html partial referenced but doesn’t exist
head.html:28 calls partial "schema.html" but the file is missing from the theme. Hugo silently ignores this — no structured data is output.
Fix: Create a layouts/_partials/schema.html file with JSON-LD structured data (see Section 5).

3. Content Quality 55/100

Strengths

  • 47 how-to guides covering specific user queries (NAS streaming, FLAC playback, CarPlay music, USB flash drives, etc.)
  • 20 blog posts with product announcements, comparisons, and technical articles
  • 4 comprehensive user guides per product app
  • Blog displays author, date, reading time, tags, and categories
  • Strong topical authority in iOS audio/video app niche
  • Content covers long-tail queries that attract organic traffic

Issues

High Product page meta titles lack keywords
Titles like “Evermusic – Everappz” miss target search terms entirely. Users searching for “offline music player iPhone” or “FLAC player iOS” won’t see relevant titles in SERPs.
Fix: Update to descriptive titles: “Evermusic: Free Offline Music Player for iPhone & Mac”, “Flacbox: Hi-Res Audio Player for iPhone & Mac (FLAC, DSD, APE)”
High Product page H1s are single words
H1 tags show only the product name (“Evermusic”, “Flacbox”) with no descriptive keywords.
Fix: Add keyword-rich H1s that include product category and platform.
Medium No author profile pages
Blog posts show author names but there are no linked author bio pages with credentials, social profiles, or expertise signals (E-E-A-T gap).
Fix: Create author profile pages with bios, credentials, and links to social profiles.
Medium Older blog content needs refresh
Several blog posts date to 2014–2016 (“Evermusic 3 Million Downloads”, etc.) and reference outdated versions. Stale content can weigh down overall site quality signals.

6. Performance (Core Web Vitals) 80/100

Assessment: As a Hugo-generated static site, everappz.com benefits from excellent baseline performance. Pages are pre-rendered HTML with no server-side processing, resulting in fast TTFB. CSS is minified and fingerprinted.

Strengths

  • Static HTML pages — no server-side rendering delay
  • CSS concatenated, minified, and served with SRI integrity hashes
  • Google Analytics uses preconnect for faster loading
  • Critical JS (theme switching) is inlined to prevent flash of unstyled content
  • No heavy JavaScript frameworks

Issues

Medium No explicit image lazy loading
Product pages include multiple screenshots that may load eagerly, impacting LCP. Add loading="lazy" to below-fold images.
Medium No font preloading detected
Web fonts may cause layout shift (CLS). Add <link rel="preload"> for critical font files.
Low No real CWV field data available
Configure a Google API key to access CrUX and PageSpeed Insights for real-user performance data.

7. AI Search Readiness (GEO) 30/100

Generative Engine Optimization (GEO) measures how well the site is positioned for AI-powered search (Google AI Overviews, ChatGPT, Perplexity, Bing Copilot).

High llms.txt returns 404
The Hextra theme includes a llms.txt template at themes/hextra/layouts/llms.txt, but it is not enabled in hugo.yaml. The home page outputs are ["HTML", "RSS"] and need "llms" added.
Fix: Change hugo.yaml line 12 from home: ["HTML", "RSS"] to home: ["HTML", "RSS", "llms"]
High No explicit AI crawler directives in robots.txt
No rules for GPTBot, Google-Extended, CCBot, Anthropic-AI, or other AI crawlers. The current User-agent: * allows all by default, but explicit directives demonstrate intentional AI engagement.
Fix: Add explicit allow rules for key AI crawlers, or block specific ones if desired.
Medium Content lacks citation-ready formatting
How-to guides would benefit from concise, quotable summary paragraphs at the top of each article. AI systems extract and cite well-structured passages more readily.

8. Images 45/100

Medium Alt text appears missing or generic on product screenshots
Product pages include app screenshots but alt text attributes are not optimized for search or accessibility.
Medium No WebP/AVIF optimization detected
Hugo supports image processing pipelines. Consider converting images to modern formats for better performance.
Low No default og:image configured
Site.Params.images is not set. Social shares may lack preview images. Add a default OG image in hugo.yaml params.
